## 一、实体关系说明

### 1.1 JobApplication 与 MockInterviewSession 的关系

```
┌──────────────────────┐         ┌──────────────────────────┐
│   JobApplication     │         │   MockInterviewSession    │
│   (岗位申请)          │         │   (模拟面试会话)           │
├──────────────────────┤         ├──────────────────────────┤
│ • id                 │<────┐   │ • id                     │
│ • company_id         │     │   │ • session_id             │
│ • job_title          │     │   │ • application_id  ───────┼───┐ (外键)
│ • status             │     │   │ • company                │   │

│ • resume_id          │     │   │ • resume_id              │   │

                             │   └──────────────────────────┘   │
```

### 3.4 创建面试会话 API

**⚠️ 关键**: applicationId 是必传参数，后端会根据它获取公司信息和职位名称

**⚠️ 后端实际接口**: `POST /api/mock-interview/start`

**⚠️ 认证要求**: 需要在请求头携带 Token，后端从 Token 中获取 userId

```typescript
// src/services/interviewApi.ts

export interface StartInterviewRequest {
  applicationId: number;
  // ⚠️ userId 已移除 - 后端从 Sa-Token 获取
}

export interface StartInterviewResponse {
  sessionId: string;
  companyId?: number;
  companyName?: string;
  jobTitle?: string;
  seniorityLevel?: string;
  status: 'IDLE' | 'WELCOME' | 'TECHNICAL_QA' | 'SCENARIO' | 'EVALUATION' | 'FINISHED';
  currentRound: number;
  totalRounds: number;
  createdAt: string;
}

export async function startInterview(params: StartInterviewRequest): Promise<StartInterviewResponse> {
  // Axios 拦截器会自动添加 Authorization: Bearer {token}
  const response = await apiClient.post('/mock-interview/start', params);
  return response.data;
}
```

### 3.5 前端调用逻辑（含认证检查）

```typescript
// 在 CompanyManagementPage.tsx 中

import { useUserStore } from '@/store/userStore';
import { useNavigate } from 'react-router-dom';
import { toast } from 'sonner';

const handleStartInterview = async () => {
  // ⚠️ 新增：检查登录状态
  const { isLoggedIn } = useUserStore.getState();
  const navigate = useNavigate();

    return;

    navigate(`/interview/${result.sessionId}`);

  } catch (error) {
    toast.error('创建面试失败', {
      description: error.message
    });
  }
};
```

```typescript
// 在 LoginPage.tsx 中，登录成功后

const handleLoginSuccess = (response: LoginResponse) => {

  useUserStore.getState().login(response.userInfo);
```

### 4.2 InterviewPage 组件结构

```tsx
// src/pages/InterviewPage.tsx

function InterviewPage() {
  const { sessionId } = useParams<{ sessionId: string }>();
  const {
    currentSession,
    messages,
    loading,
    sendMessage,
    loadSession,
    finishInterview
  } = useInterviewStore();

  // 页面加载时获取会话信息
  useEffect(() => {
    if (sessionId) {
      loadSession(sessionId);
    }
  }, [sessionId]);

  if (loading || !currentSession) {
    return <InterviewPageSkeleton />;
  }

  return (
    <div className="interview-page full-screen">
      {/* 顶部导航栏 */}
      <InterviewHeader
        company={currentSession.company}
        jobTitle={currentSession.jobTitle}
        sessionId={sessionId}
        onFinish={finishInterview}
      />

      <div className="interview-content">
        {/* 左侧：对话区域 */}
        <InterviewChat
          messages={messages}
          onSendMessage={sendMessage}
        />

        {/* 右侧：进度面板 */}
        <InterviewSidebar
          session={currentSession}
          stats={calculateStats(messages)}
        />
      </div>
    </div>
  );
}
```

```typescript
// src/store/interviewStore.ts

interface InterviewStore {

  sendMessage: (sessionId: string, content: string, roundNumber: number) => Promise<void>;

const sendMessage = async (sessionId: string, content: string, roundNumber: number) => {
  try {
    // 1. 立即添加用户消息到 UI
    const userMessage: InterviewMessage = {
      id: Date.now(),
      role: 'CANDIDATE',  // ⚠️ 后端使用的角色名
      content,
      roundNumber,
      timestamp: new Date().toISOString()
    };
    setMessages(prev => [...prev, userMessage]);

    // 2. 调用后端 API
    const response = await interviewApi.sendMessage({
      content,
      roundNumber
    }, sessionId);

    // 3. 添加 AI 回复到 UI (后端返回的是面试官问题)
    const aiMessage: InterviewMessage = {
      id: response.data.id,
      role: response.data.role,  // INTERVIEWER
      content: response.data.content,
      roundNumber: response.data.roundNumber,
      timestamp: response.data.createdAt
    };
    setMessages(prev => [...prev, aiMessage]);

  } catch (error) {
    // 4. 发送失败，显示错误提示
    toast.error('发送消息失败');
  }
};

// interviewApi.ts
export async function sendMessage(
  params: { content: string; roundNumber: number },
  sessionId: string
): Promise<ApiResponse<InterviewMessage>> {
  return apiClient.post(`/mock-interview/sessions/${sessionId}/message`, params);
}
```

#### 12.2.1 authUtil 工具类

```typescript
// src/utils/authUtil.ts

const TOKEN_KEY = 'satoken';
const USER_INFO_KEY = 'user_info';

export const authUtil = {
  // 存储 Token
  setToken: (token: string) => {
    localStorage.setItem(TOKEN_KEY, token);
  },

  // 获取 Token
  getToken: (): string | null => {
    return localStorage.getItem(TOKEN_KEY);
  },

  // 清除 Token
  clearToken: () => {
    localStorage.removeItem(TOKEN_KEY);
    localStorage.removeItem(USER_INFO_KEY);
  },

  // 存储用户信息
  setUserInfo: (userInfo: UserInfo) => {
    localStorage.setItem(USER_INFO_KEY, JSON.stringify(userInfo));
  },

  // 获取用户信息
  getUserInfo: (): UserInfo | null => {
    const data = localStorage.getItem(USER_INFO_KEY);
    return data ? JSON.parse(data) : null;
  },

  // 检查是否登录
  isLoggedIn: (): boolean => {
    return !!authUtil.getToken();
  }
};
```

```typescript
    set({ isLoggedIn: true, userInfo });
```

#### 12.2.3 API 拦截器配置

```typescript
// src/services/apiClient.ts

import axios from 'axios';
import { authUtil } from '@/utils/authUtil';
import { useUserStore } from '@/store/userStore';
import { toast } from 'sonner';

export const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:8080/api/data',
  timeout: 30000
});

// 请求拦截器：自动注入 Token
apiClient.interceptors.request.use(
  (config) => {
    const token = authUtil.getToken();
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => {
    return Promise.reject(error);
  }
);

// 响应拦截器：处理 401
apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      // Token 过期，清除本地数据
      useUserStore.getState().logout();

      // 保存当前路径，登录后返回
      const currentPath = window.location.pathname;
      if (currentPath !== '/login' && currentPath !== '/register') {
        sessionStorage.setItem('redirect_after_login', currentPath);
      }

      toast.error('登录已过期，请重新登录');
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);
```

```java
public class UserContext {
    private static final ThreadLocal<UserInfoHolder> CONTEXT_HOLDER = new ThreadLocal<>();

    public static Long getCurrentUserId() {
        UserInfoHolder holder = getCurrentUser();
        if (holder == null || holder.getUserId() == null) {
            throw new NotLoginException("未登录或登录已过期");
        }
        return holder.getUserId();
    }

    public static String getCurrentUsername() {
        UserInfoHolder holder = getCurrentUser();
        return holder != null ? holder.getUsername() : null;
    }

    public static void setCurrentUser(UserInfoHolder holder) {
        CONTEXT_HOLDER.set(holder);
    }

    public static void clear() {
        CONTEXT_HOLDER.remove();
    }
}
```

## 十三、常见问题

### Q1: 为什么点击"模拟面试"会跳转到登录页？

**A**: 模拟面试功能需要用户登录才能使用：
1. 前端检查 `useUserStore.getState().isLoggedIn`
2. 如果未登录，保存当前路径到 `sessionStorage`
3. 跳转到登录页 `/login`
4. 登录成功后自动返回原页面继续面试

```typescript
if (!isLoggedIn) {
  sessionStorage.setItem('redirect_after_login', window.location.pathname);
  navigate('/login');
  return;
}
```

### Q2: Token 存储在哪里？如何自动刷新？

**A**:
- **存储位置**: `localStorage`，key 为 `satoken`
- **自动注入**: Axios 请求拦截器自动添加 `Authorization: Bearer {token}`
- **Token 过期**: 响应拦截器捕获 401，清除本地数据并跳转登录

```typescript
// 自动注入
apiClient.interceptors.request.use((config) => {
  const token = localStorage.getItem('satoken');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 401 处理
apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('satoken');
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);
```

### Q3: 模拟面试和 JobApplication 是什么关系？

**A**: 一对多关系：
- 一个 `JobApplication` 可以有多个 `MockInterviewSession`
- 用户可以对同一个岗位申请重复进行模拟面试
- 每次面试都是独立的 `sessionId`，但都关联到同一个 `applicationId`

```
JobApplication (id=123, 字节跳动-Java后端)
    ├── MockInterviewSession (session-001, 第一次面试)
    ├── MockInterviewSession (session-002, 第二次面试)
    └── MockInterviewSession (session-003, 第三次面试)
```

### Q4: 后端如何知道当前登录用户是谁？

**A**: 通过 Sa-Token + UserContext 机制：
1. 前端在请求头携带 `Authorization: Bearer {token}`
2. `LoginInterceptor` 拦截请求，验证 Token
3. 从 Sa-Token 获取 `userId` 并设置到 `UserContext` (ThreadLocal)
4. Controller 通过 `UserContext.getCurrentUserId()` 获取

```java
// Controller 不再从请求获取 userId
@PostMapping("/start")
public ApiResponse<MockInterviewSession> startInterview(@RequestBody StartInterviewRequest request) {
    Long userId = UserContext.getCurrentUserId();  // 从 ThreadLocal 获取
    // ...
}
```

### Q5: 如何从不同入口进入面试？

**A**: 有三种方式：
1. 从职位详情页点击"模拟面试"按钮（需登录）
2. 从面试列表页点击"继续面试"（需登录）
3. 从面试列表页点击"再次面试"创建新会话（需登录）

所有入口都会先检查登录状态，未登录会跳转到登录页。

### Q6: 面试页面和 AI 聊天助手有什么区别？

**A**: 完全独立：
- **路由不同**: 面试是 `/interview/:sessionId`，聊天是内嵌组件
- **状态隔离**: 使用不同的 Store (interviewStore vs chatStore)
- **API 不同**: 面试用 `/api/mock-interview/*`，聊天用 `/api/chat/*` ⚠️
- **Agent 不同**: 面试有3个协同 Agent，聊天是单个通用 Agent
- **认证要求**: 面试 API 需要登录，聊天 API 可选登录

### Q7: 如何处理页面刷新？

**A**: 使用 `useSessionRestore` hook：
- 从 URL 获取 `sessionId`
- 调用 `loadSession(sessionId)` 重新加载会话数据
- 加载历史消息并恢复状态
- 如果 Token 过期，会被拦截器捕获并跳转登录

### Q8: 面试过程中如何防止意外离开？

**A**: 使用 `useLeaveConfirmation` hook：
- 监听 `beforeunload` 事件
- 显示浏览器原生确认对话框
- 仅在面试未完成时触发

### Q9: 登录后为什么没有自动返回面试页面？

**A**: 检查以下配置：
1. 登录页是否正确读取 `sessionStorage.redirect_after_login`
2. 登录成功后是否正确调用 `navigate(redirectPath)`
3. 确保重定向路径以 `/` 开头

```typescript
const redirectPath = sessionStorage.getItem('redirect_after_login');
if (redirectPath) {
  sessionStorage.removeItem('redirect_after_login');
  navigate(redirectPath);  // 必须是 navigate('/interview/xxx') 格式
}
```

### Q10: 如何在开发环境测试登录功能？

**A**:
1. 启动后端，确保 Sa-Token 配置正确
2. 使用 Postman 测试登录接口：
   ```
   POST http://localhost:8080/api/auth/login
   Body: {"username": "test", "password": "123456"}
   ```
3. 获取 Token 后，在前端 localStorage 手动设置：
   ```javascript
   localStorage.setItem('satoken', 'your-token-here')
   ```
4. 刷新页面，userStore 会自动更新登录状态
