## 一、实体关系说明

### 1.1 JobApplication 与 MockInterviewSession 的关系

```
┌──────────────────────┐         ┌──────────────────────────┐
│   JobApplication     │         │   MockInterviewSession    │
│   (岗位申请)          │         │   (模拟面试会话)           │
├──────────────────────┤         ├──────────────────────────┤
│ • id                 │<────┐   │ • id                     │
│ • company_id         │     │   │ • session_id             │
│ • job_title          │     │   │ • application_id  ───────┼───┐ (外键)
│ • status             │     │   │ • company                │   │
│ • application_date   │     │   │ • job_title              │   │
│ • resume_id          │     │   │ • resume_id              │   │
└──────────────────────┘     │   │ • status                 │   │
                             │   └──────────────────────────┘   │
                             │                                  │
┌──────────────────────┐     │   ┌──────────────────────────┐   │
│   Company            │     │   │   InterviewMessages       │   │
│   (公司信息)          │     │   │   (面试消息)              │   │
├──────────────────────┤     │   ├──────────────────────────┤   │
│ • id                 │────┼──>│ • id                     │   │
│ • name               │     │   │ • session_id      ───────┼───┘
│ • industry           │     │   │ • role                   │
│ • size               │     │   │ • content                │
└──────────────────────┘     │   └──────────────────────────┘
                             │
    关键关系说明：
    1. MockInterviewSession.application_id → JobApplication.id
    2. 从 JobApplication 获取：company、jobTitle、resumeId
    3. 一次 JobApplication 可以有多个 MockInterviewSession（可重复面试）
```

### 1.2 前端数据流

```
用户在职位列表/详情页
    ↓
选中一个 JobApplication (id=123)
    ↓
点击"模拟面试"按钮 (传递 applicationId=123)
    ↓
后端创建 MockInterviewSession (关联 application_id=123)
    ↓
从 JobApplication 获取公司名、职位名
    ↓
面试页面显示：字节跳动 - Java后端工程师
```

### 3.4 创建面试会话 API

**⚠️ 关键**: applicationId 是必传参数，后端会根据它获取公司信息和职位名称

**⚠️ 后端实际接口**: `POST /api/mock-interview/start`

```typescript
// src/services/interviewApi.ts

export interface StartInterviewRequest {
  applicationId: number;
  userId?: number;  // 可选，后端会从 session 获取
}

export interface StartInterviewResponse {
  sessionId: string;
  companyId?: number;
  companyName?: string;
  jobTitle?: string;
  seniorityLevel?: string;
  status: 'IDLE' | 'WELCOME' | 'TECHNICAL_QA' | 'SCENARIO' | 'EVALUATION' | 'FINISHED';
  currentRound: number;
  totalRounds: number;
  createdAt: string;
}

export async function startInterview(params: StartInterviewRequest): Promise<StartInterviewResponse> {
  // ⚠️ 正确的 API 路径
  const response = await apiClient.post('/mock-interview/start', params);
  return response.data;
}
```

### 3.5 前端调用逻辑

};

### 4.2 InterviewPage 组件结构

```tsx
// src/pages/InterviewPage.tsx

function InterviewPage() {
  const { sessionId } = useParams<{ sessionId: string }>();
  const {
    currentSession,
    messages,
    loading,
    sendMessage,
    loadSession,
    finishInterview
  } = useInterviewStore();

  // 页面加载时获取会话信息
  useEffect(() => {
    if (sessionId) {
      loadSession(sessionId);
    }
  }, [sessionId]);

  if (loading || !currentSession) {
    return <InterviewPageSkeleton />;
  }

  return (
    <div className="interview-page full-screen">
      {/* 顶部导航栏 */}
      <InterviewHeader
        company={currentSession.company}
        jobTitle={currentSession.jobTitle}
        sessionId={sessionId}
        onFinish={finishInterview}
      />

      <div className="interview-content">
        {/* 左侧：对话区域 */}
        <InterviewChat
          messages={messages}
          onSendMessage={sendMessage}
        />

        {/* 右侧：进度面板 */}
        <InterviewSidebar
          session={currentSession}
          stats={calculateStats(messages)}
        />
      </div>
    </div>
  );
}
```

### 5.2 发送消息代码示例

```typescript
// src/store/interviewStore.ts

interface InterviewStore {
  // 发送面试消息
  sendMessage: (sessionId: string, content: string, roundNumber: number) => Promise<void>;
}

const sendMessage = async (sessionId: string, content: string, roundNumber: number) => {
  try {
    // 1. 立即添加用户消息到 UI
    const userMessage: InterviewMessage = {
      id: Date.now(),
      role: 'CANDIDATE',  // ⚠️ 后端使用的角色名
      content,
      roundNumber,
      timestamp: new Date().toISOString()
    };
    setMessages(prev => [...prev, userMessage]);

    // 2. 调用后端 API
    const response = await interviewApi.sendMessage({
      content,
      roundNumber
    }, sessionId);

    // 3. 添加 AI 回复到 UI (后端返回的是面试官问题)
    const aiMessage: InterviewMessage = {
      id: response.data.id,
      role: response.data.role,  // INTERVIEWER
      content: response.data.content,
      roundNumber: response.data.roundNumber,
      timestamp: response.data.createdAt
    };
    setMessages(prev => [...prev, aiMessage]);

  } catch (error) {
    // 4. 发送失败，显示错误提示
    toast.error('发送消息失败');
  }
};

// interviewApi.ts
export async function sendMessage(
  params: { content: string; roundNumber: number },
  sessionId: string
): Promise<ApiResponse<InterviewMessage>> {
  return apiClient.post(`/mock-interview/sessions/${sessionId}/message`, params);
}
```

## 十一、API 快速对照表

**⚠️ 重要**: 所有模拟面试 API 基础路径为 `/api/mock-interview/*`

| 功能 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 创建会话 | POST | `/api/mock-interview/start` | 传入 applicationId |
| 获取会话详情 | GET | `/api/mock-interview/sessions/{sessionId}` | 返回完整会话信息 |
| 发送消息 | POST | `/api/mock-interview/sessions/{sessionId}/message` | 返回面试官问题 |
| 获取消息列表 | GET | `/api/mock-interview/sessions/{sessionId}/messages` | 返回所有历史消息 |
| 结束面试 | POST | `/api/mock-interview/sessions/{sessionId}/finish` | 持久化所有记忆 |
| 获取评分列表 | GET | `/api/mock-interview/sessions/{sessionId}/evaluations` | 返回各轮评分 |
| 评估单轮 | POST | `/api/mock-interview/sessions/{sessionId}/evaluate` | 评估用户回答 |

**消息角色对照**:
- `INTERVIEWER`: 面试官（AI 提问）
- `CANDIDATE`: 候选人（用户回答）

## 十二、常见问题

### Q1: 模拟面试和 JobApplication 是什么关系？

**A**: 一对多关系：
- 一个 `JobApplication` 可以有多个 `MockInterviewSession`
- 用户可以对同一个岗位申请重复进行模拟面试
- 每次面试都是独立的 `sessionId`，但都关联到同一个 `applicationId`

```
JobApplication (id=123, 字节跳动-Java后端)
    ├── MockInterviewSession (session-001, 第一次面试)
    ├── MockInterviewSession (session-002, 第二次面试)
    └── MockInterviewSession (session-003, 第三次面试)
```

### Q2: 如何从不同入口进入面试？

**A**: 有三种方式：
1. 从职位详情页点击"模拟面试"按钮
2. 从面试列表页点击"继续面试"
3. 从面试列表页点击"再次面试"创建新会话

### Q3: 面试页面和 AI 聊天助手有什么区别？

**A**: 完全独立：
- **路由不同**: 面试是 `/interview/:sessionId`，聊天是内嵌组件
- **状态隔离**: 使用不同的 Store (interviewStore vs chatStore)
- **API 不同**: 面试用 `/api/mock-interview/*`，聊天用 `/api/chat/*` ⚠️
- **Agent 不同**: 面试有3个协同 Agent，聊天是单个通用 Agent

### Q4: 如何处理页面刷新？

**A**: 使用 `useSessionRestore` hook：
- 从 URL 获取 `sessionId`
- 调用 `loadSession(sessionId)` 重新加载会话数据
- 加载历史消息并恢复状态

### Q5: 面试过程中如何防止意外离开？

**A**: 使用 `useLeaveConfirmation` hook：
- 监听 `beforeunload` 事件
- 显示浏览器原生确认对话框
- 仅在面试未完成时触发
