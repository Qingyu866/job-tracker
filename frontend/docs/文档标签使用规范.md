**创建日期**: 2026-03-17
**密级**: 内部公开

## 一、文档结构规范

### 1.1 文档头部标签

所有前端设计文档必须包含以下头部标签：

```markdown
# 文档标题

**对应后端文档**: `docs/功能设计/XXX设计_YYYYMMDD.md`（可选）
**创建日期**: YYYY-MM-DD
**密级**: 内部公开
**版本**: vX.Y
```

#### 标签说明

| 标签 | 必需 | 说明 |
|------|------|------|
| 对应后端文档 | 条件 | 如有对应后端文档则必填，格式为相对路径 |
| 创建日期 | ✅ | 文档首次创建日期 |
| 密级 | ✅ | 默认为"内部公开"，可选"机密"、"绝密" |
| 版本 | ✅ | 遵循语义化版本，如 v1.0、v1.1 |

---

### 1.2 目录结构

文档超过 5 个章节时，必须在头部标签后添加目录：

```markdown
## 目录

1. [章节名称](#一章节名称)
2. [章节名称](#二章节名称)
...
```

#### 目录锚点格式

- 中文章节名使用 URL 编码
- 例如：`#一设计概述` 对应 `## 一、设计概述`
- 工具推荐：使用 VS Code 的 "Markdown All in One" 插件自动生成

---

## 二、章节编号规范

### 2.1 一级章节

使用中文数字 + 顿号：

```markdown
## 一、设计概述
## 二、Token 管理
## 三、API 拦截器
```

### 2.2 二级章节

使用阿拉伯数字 + 点号：

```markdown
### 1.1 认证流程
### 1.2 核心功能
### 2.1 authUtil 工具类
```

### 2.3 三级章节

使用阿拉伯数字 + 括号：

```markdown
#### 2.1.1 登录状态检查
#### 2.1.2 Token 自动注入
```

### 2.4 编号层级限制

- 最多使用 4 级标题
- 超过 4 级时，使用列表或粗体文本代替

---

## 三、文档尾部标签

### 3.1 变更记录

所有文档必须在末尾包含变更记录表格：

```markdown
## 文档变更记录

| 版本 | 日期 | 变更内容 | 作者 |
|------|------|----------|------|
| v1.0 | YYYY-MM-DD | 初始版本，完成XXX设计 | 作者名 |
| v1.1 | YYYY-MM-DD | 新增XXX功能说明 | 作者名 |
```

### 3.2 文档状态

变更记录后标注文档状态：

```markdown
**文档状态**: 已完成 | 编写中 | 已废弃
```

### 3.3 相关文档

列出与本文档相关的其他文档：

```markdown
**相关文档**:
- `docs/功能设计/XXX设计_YYYYMMDD.md` - 简要说明
- `frontend/docs/YYY设计_YYYYMMDD.md` - 简要说明
```

## 四、内容标签规范

### 4.1 代码块标签

代码块必须标注语言类型：

```markdown
​```typescript
// TypeScript 代码
​```

​```tsx
// React 组件代码
​```

​```java
// Java 后端代码
​```

​```bash
# Shell 命令
​```
```

### 4.2 重要提示标签

使用统一的提示格式：

```markdown
**⚠️ 重要**: 说明内容

**⚠️ 注意**: 说明内容

**✅ 推荐**: 说明内容

**❌ 不推荐**: 说明内容
```

### 4.3 文件路径标签

引用文件路径时使用行内代码：

```markdown
文件位置：`src/components/chat/ChatInput.tsx`
新增文件：`src/store/userStore.ts`
```

### 4.4 API 接口标签

API 接口说明格式：

```markdown
**接口**: `POST /api/auth/login`

**认证要求**: 需要登录，请求头需携带 Token

**请求参数**:
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| username | string | ✅ | 用户名 |
| password | string | ✅ | 密码 |

**响应示例**:
​```json
{
  "code": 200,
  "message": "登录成功",
  "data": {...}
}
​```
```

## 五、表格规范

### 5.1 功能表格

```markdown
| 功能 | 描述 | 优先级 | 复杂度 |
|------|------|--------|--------|
| 登录 | 用户名密码登录 | P0 | 中 |
| 注册 | 新用户注册 | P0 | 低 |
```

### 5.2 文件清单表格

```markdown
| 文件路径 | 类型 | 说明 |
|----------|------|------|
| `src/pages/LoginPage.tsx` | 新增 | 登录页面 |
| `src/store/userStore.ts` | 新增 | 用户状态管理 |
| `src/App.tsx` | 修改 | 添加路由守卫 |
```

## 七、命名规范

### 7.1 文件命名

```
功能名称_YYYYMMDD.md

示例：
- 前端用户认证功能设计_20260317.md
- OCR图片识别前端设计_20260317.md
- 模拟面试系统前端设计_20260317.md
```

### 7.2 组件命名

- 组件文件：PascalCase，如 `LoginPage.tsx`
- 工具文件：camelCase，如 `authUtil.ts`
- Store 文件：camelCase + Store，如 `userStore.ts`
- API 文件：camelCase + Api，如 `authApi.ts`

## 九、检查清单

创建或更新文档时，请检查以下项目：

### 头部检查

- [ ] 标题清晰明确
- [ ] 包含创建日期
- [ ] 包含密级标识
- [ ] 包含版本号
- [ ] 如有对应后端文档，已添加链接

### 目录检查

- [ ] 超过 5 个章节时包含目录
- [ ] 目录链接正确可跳转
- [ ] 章节编号连续无重复

### 内容检查

- [ ] 章节编号符合规范
- [ ] 代码块标注语言类型
- [ ] 文件路径使用行内代码
- [ ] 表格格式正确

### 尾部检查

- [ ] 包含变更记录
- [ ] 文档状态明确
- [ ] 相关文档链接完整
