## 一、功能需求分析

### 1.1 OCR 图片识别模块

#### 关键功能点

| 功能 | 描述 | 优先级 | 复杂度 |
|------|------|--------|--------|
| 图片上传 | 在聊天中上传图片附件，支持拖拽、点击选择 | P0 | 中 |
| 图片预览 | 上传前预览图片，支持删除操作 | P0 | 低 |
| OCR 识别状态 | 显示识别中/成功/失败状态 | P0 | 低 |
| 识别结果展示 | 展示识别的文本内容、简历数据、JD数据 | P1 | 中 |
| 简历创建 | 基于识别结果创建简历记录 | P1 | 中 |
| JD 技能提取 | 从 JD 图片提取技能标签 | P1 | 中 |

#### 交互流程

```
用户选择图片
    ↓
前端验证（大小≤10MB、类型JPG/PNG/PDF）
    ↓
转换为 Base64 / 上传到服务器
    ↓
添加到消息输入框预览区
    ↓
用户点击发送
    ↓
通过 WebSocket 发送（包含图片信息）
    ↓
后端接收并调用 OCR
    ↓
返回识别结果给 AI
    ↓
AI 根据识别结果回复
```

#### 视觉设计要求

- 图片预览区：80x80px 缩略图，圆角 8px
- OCR 状态标签：底部半透明背景，字体 10px
- 颜色状态：识别中(蓝色)、成功(绿色)、失败(红色)

### 1.2 模拟面试系统模块

#### 关键功能点

| 功能 | 描述 | 优先级 | 复杂度 |
|------|------|--------|--------|
| 独立面试页面 | 全屏专用面试界面，与聊天助手分离 | P0 | 高 |
| 发起面试 | 从职位详情页进入，选择简历后开始 | P0 | 中 |
| 面试对话 | 与三 AI 面试官实时对话 | P0 | 高 |
| 进度跟踪 | 显示当前轮次、已考察技能、待考察技能 | P1 | 中 |
| 结束面试 | 生成面试报告 | P1 | 高 |
| 历史记录 | 查看过去的面试会话 | P2 | 中 |
| 面试报告 | 总分、评分明细、真实性分析、改进建议 | P1 | 高 |

#### 交互流程

```
发起面试流程：
职位详情页 → 点击"模拟面试" → 简历选择对话框 → 确认信息 → 进入面试页面

面试进行流程：
主面试官提问 → 用户回答 → 副面试官分析 → 主面试官新问题 → 循环直到结束

结束流程：
用户点击结束/达到轮数 → 生成报告 → 显示评分和建议
```

#### 视觉设计要求

- 面试页面布局：左侧对话区(1fr) + 右侧进度面板(320px)
- 进度面板：白色背景，圆角 12px
- 分数卡片：渐变背景(135deg, #667eea 0%, #764ba2 100%)
- 技能标签：已考察(secondary)、待考察(outline)

### 1.3 三面试官独立记忆存储模块

#### 关键功能点

| 功能 | 描述 | 优先级 | 复杂度 |
|------|------|--------|--------|
| 会话列表页面 | 显示所有面试会话记录 | P0 | 中 |
| 会话状态隔离 | 每个会话独立状态管理 | P0 | 中 |
| 会话恢复 | 刷新后恢复到上次会话 | P1 | 低 |
| 离开确认 | 防止意外离开未完成面试 | P1 | 低 |
| 多标签页同步 | 可选，标签页间状态同步 | P2 | 中 |

#### 交互流程

```
进入面试方式：
1. 从职位详情页发起 → 选择简历 → 开始面试
2. 从会话列表继续 → 直接进入面试页面
3. 查看历史报告 → 跳转到报告页面
```

#### 视觉设计要求

- 会话卡片：白色背景，圆角 12px，阴影 paper-md
- 进行中状态：显示进度条和剩余时间
- 已完成状态：显示评分和完成时间

### 3.4 优先级排序原则

1. **业务价值优先**：P0 功能直接影响核心用户体验
2. **依赖关系优先**：被依赖的组件优先开发
3. **风险优先**：复杂度高的任务提前开发，留出缓冲时间
4. **迭代交付**：每个阶段都有可交付成果

### 4.2 组件抽象原则

#### 原则 1：单一职责

每个组件只负责一个功能，避免过度耦合：

```tsx

function OcrStatusBadge({ status }: { status: OcrStatus }) {
  return <Badge variant={statusVariant[status]}>{statusText[status]}</Badge>;

// ❌ 不好的做法：职责过多
function OcrStatusBadge({ status, onRetry, onConfirm, showActions }) {
  // 混合了状态展示和操作逻辑
}
```

#### 原则 2：组合优于继承

使用组合模式构建复杂组件：


#### 原则 3：Props 接口一致性

相似组件使用一致的 Props 接口：

```tsx

interface ListItemProps {
  data: T;
  selected?: boolean;
```

#### 原则 4：受控与非受控模式

支持两种模式，提供灵活性：

```tsx
// 支持受控和非受控模式
interface InputProps {
  value?: string;           // 受控模式
  defaultValue?: string;    // 非受控模式
  onChange?: (value: string) => void;
}

function Input({ value, defaultValue, onChange }: InputProps) {
  const [internalValue, setInternalValue] = useState(defaultValue ?? '');
  const isControlled = value !== undefined;
  const currentValue = isControlled ? value : internalValue;
  
  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const newValue = e.target.value;
    if (!isControlled) {
      setInternalValue(newValue);
    }
    onChange?.(newValue);
  };
  
  return <input value={currentValue} onChange={handleChange} />;
}
```

```typescript
function useInterviewAccess(sessionId: string | undefined) {
  const { user } = useAuthStore();
  const { session } = useInterviewStore();
  
  const canAccess = useMemo(() => {
    if (!user || !session) return false;
    return session.userId === user.id;
```
