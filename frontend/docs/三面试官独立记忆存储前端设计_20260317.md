## 一、设计概述

### 1.1 前端影响说明

**重要**：三面试官独立记忆存储主要是**后端架构优化**，对前端的影响较小。

前端主要确保：
1. 正确处理不同的面试会话（sessionId 隔离）
2. 维护每个会话的状态独立性
3. 支持会话列表查看和历史记录

### 1.2 模拟面试是独立页面

```
模拟面试 ≠ AI 聊天助手

模拟面试是独立的全屏页面：
- 路由：/interview/:sessionId
- 独立的 WebSocket 连接（如果使用实时通信）
- 独立的状态管理
- 独立的 UI 组件
```

## 四、用户体验优化

### 4.1 会话恢复

```typescript
// 用户刷新页面后，恢复到上次所在的会话
function useSessionRestore() {
  useEffect(() => {
    // 从 localStorage 读取上次活跃的 sessionId
    const lastSessionId = localStorage.getItem('last_interview_session_id');

    if (lastSessionId) {
      // 检查会话是否还存在
      const session = getSession(lastSessionId);
      if (session && session.state !== 'FINISHED') {
        navigateToInterview(lastSessionId);
      }
    }
  }, []);

  // 保存当前活跃会话
  useEffect(() => {
    if (activeSessionId) {
      localStorage.setItem('last_interview_session_id', activeSessionId);
    }
  }, [activeSessionId]);
}
```

### 4.2 离开确认

```tsx
// 防止用户意外离开未完成的面试
interface InterviewHeaderProps {
  session: MockInterviewSession;
}

function InterviewHeader({ session }: Props) {
  const isUnfinished = session.state !== 'FINISHED';

  useEffect(() => {
    if (!isUnfinished) return;

    const handleBeforeUnload = (e: BeforeUnloadEvent) => {
      e.preventDefault();
      e.returnValue = '面试进行中，确定要离开吗？';
    };

    window.addEventListener('beforeunload', handleBeforeUnload);

    return () => {
      window.removeEventListener('beforeunload', handleBeforeUnload);
    };
  }, [session, isUnfinished]);

  return (
    <div className="interview-header">
      <h1>{session.companyName} - {session.jobTitle}</h1>
      <Button variant="outline" onClick={() => navigate('/interviews')}>
        返回列表
      </Button>
    </div>
  );
}
```

```typescript
const channel = new BroadcastChannel('interview_sessions');

// 发送会话更新
function broadcastSessionUpdate(session: MockInterviewSession) {
  channel.postMessage({
    type: 'SESSION_UPDATED',
    sessionId: session.sessionId,
    data: session
  });
}

// 接收其他标签页的更新
channel.onmessage = (event) => {
```

### 6.1 页面路由结构

```
/interviews
```

### 6.2 与 AI 聊天助手的区别

| 特性 | AI 聊天助手 | 模拟面试 |
|------|-----------|---------|
| 路由 | 内嵌组件 | 独立页面 `/interview/*` |
| WebSocket | `/api/ws/chat` | 独立连接或 HTTP 轮询 |
| 状态管理 | chatStore | interviewStore |
| 进入方式 | 点击聊天图标 | 从职位详情页或会话列表 |

### 6.3 用户流程

```
新用户流程：
1. 在职位详情页点击"模拟面试"
2. 选择要使用的简历
3. 进入全屏面试页面
4. 与三 AI 面试官对话
5. 完成面试，查看报告

老用户流程：
1. 访问会话列表页 /interviews
2. 选择历史会话查看报告
3. 或点击"继续面试"恢复未完成的会话
4. 或点击"再次面试"发起新面试
```
