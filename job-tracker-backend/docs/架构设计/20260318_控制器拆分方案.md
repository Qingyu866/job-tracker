# 控制器拆分方案

## 📊 当前控制器分析

### 控制器概览

| 控制器 | 行数 | 职责数量 | 是否需要拆分 |
|--------|------|----------|--------------|
| DataController | 1076 | 7 | ✅ **急需拆分** |
| MockInterviewController | 297 | 1 | ❌ 单一职责 |
| ChatController | 175 | 1 | ❌ 单一职责 |
| OcrController | 230 | 1 | ❌ 单一职责 |
| ResumeController | 165 | 1 | ❌ 单一职责 |
| StatusController | 204 | 1 | ❌ 单一职责 |
| SkillTagController | 114 | 1 | ❌ 单一职责 |
| AuthController | 83 | 1 | ❌ 单一职责 |

### 问题诊断

**DataController** 存在严重问题：
- 📏 **代码行数过多**: 1076行，远远超过建议的300行限制
- 🎯 **职责过多**: 包含7个不同的业务领域
- 🔧 **维护困难**: 修改一个功能可能影响其他功能
- 📦 **依赖注入过多**: 注入了7个不同的Service
- 🚦 **缺乏内聚**: 不同功能的代码混在一起

---

## 🎯 拆分方案

### 方案概览

将 **DataController** 拆分为 7 个独立的控制器：

```
DataController (1076行)
    ↓ 拆分为 ↓
├── ApplicationController (求职申请)
├── InterviewController (面试记录)
├── CompanyController (公司信息)
├── ApplicationLogController (申请日志)
├── StatisticsController (统计数据)
├── ExportController (数据导出)
└── SearchController (搜索功能)
```

---

## ⚠️ 注意事项

### API 路径兼容性

**问题**: 前端已使用 `/api/data/*` 路径

**解决方案**:

**选项 1**: 保持路径不变（推荐）
- 新控制器仍使用 `/data/*` 前缀
- 只是拆分成多个文件
- 前端无需修改

**选项 2**: 使用新的RESTful路径
- 新控制器使用标准RESTful路径
- 前端需要更新所有API调用
- 提供迁移期兼容层

### 依赖注入优化

拆分后，每个控制器只注入自己需要的 Service：

```java
// 拆分前
public class DataController {
    private final ApplicationService applicationService;
    private final InterviewService interviewService;
    private final CompanyService companyService;
    private final ApplicationLogService applicationLogService;
    private final ExportService exportService;
    private final StatusTransitionService statusTransitionService;
    private final InterviewManagementService interviewManagementService;
}

// 拆分后
public class ApplicationController {
    private final ApplicationService applicationService;
    private final StatusTransitionService statusTransitionService;
    private final ApplicationLogService applicationLogService;
}
```

### 代码复用

如果多个控制器需要相同的工具方法，可以：

1. 创建 `ControllerHelper` 类
2. 使用继承创建 `BaseController`
3. 使用 Spring 的 `@ControllerAdvice`
