## 问题现象

### 初次修复尝试（失败）

在 System Message 中添加了输出格式要求：
```java
# ⚠️ 输出格式要求
- **只返回纯 JSON 对象，不要使用 Markdown 代码块**
- 不要包含 ```json 或 ``` 标记
- 直接输出 JSON，例如：{"action": "NEXT_QUESTION", "nextTopic": "...", ...}
```

**结果**：LLM（google/gemma-3-4b）**仍然返回 Markdown 代码块**：
```json
{
  "action": "NEXT_QUESTION",
  "next_topic": "服务注册与发现机制",
  ...
}
```

**错误**：
```
JsonParseException: Unexpected character ('`' (code 96))
```

---

## 根本原因

### LLM 的训练数据习惯

**问题**：
1. LLM 的训练数据中，大量的 JSON 都被包裹在 Markdown 代码块中
2. 这是 LLM 学会的"好习惯"：使用代码块提高可读性
3. 即使 System Message 中说明了要求，LLM 仍然倾向于使用 Markdown

**temperature: 0.7 的影响**：
- temperature = 0.7 意味着较高的随机性
- LLM 不太会严格遵循指令
- 更容易按照"习惯"输出

---

## 最终修复方案

### 1. 强化 System Message 指令

**修改前**（温和的提示）：
```java
# ⚠️ 输出格式要求
- **只返回纯 JSON 对象，不要使用 Markdown 代码块**
- 不要包含 ```json 或 ``` 标记
- 直接输出 JSON，例如：{"action": "NEXT_QUESTION", ...}
```

**修改后**（强烈的指令 + 负向示例）：
```java
# ⚠️️ 输出格式要求（必须遵守）
**严禁使用 Markdown 代码块！**
- ❌ 错误示例：```json {...}```
- ✅ 正确示例：{"action": "NEXT_QUESTION", ...}
- 直接输出纯 JSON，不要有任何标记
- 不要包含 ```json 或 ``` 标记
- 开头直接是 `{`，结尾直接是 `}`
```

**关键改进**：
1. 使用"**必须遵守**"强调
2. 使用"**严禁**"强化语气
3. 提供负向示例（❌ 错误示例）
4. 提供正向示例（✅ 正确示例）
5. 明确说明"开头直接是 `{`，结尾直接是 `}`"

---

### 2. 降低 temperature

**修改前**：

```

**修改后**：

```

**效果**：

- ✅ 减少"创造性"输出，更倾向于遵循规则

## temperature 参数说明

### temperature 范围与效果

| temperature | 效果 | 适用场景 |
|------------|------|---------|
| 0.0-0.2 | 几乎完全确定性，严格遵循指令 | 需要精确格式输出的场景 |
| 0.3-0.5 | 较低随机性，倾向于遵循指令 | 一般的对话和问答 |
| 0.6-0.8 | 较高随机性，创造性输出 | 创意写作、头脑风暴 |
| 0.9-1.0 | 最高随机性，非常创造性 | 创意生成、故事创作 |

**我们的场景**：
- 需要**精确的 JSON 格式**输出
- 不能有任何格式错误
- **最适合：temperature = 0.0-0.2**

private String cleanMarkdownCodeBlocks(String text) {

## 如果仍然失败

### 降级方案：在代码中清理 Markdown 标记

**实现位置**：`MockInterviewController.java`

```java
private String cleanMarkdownCodeBlocks(String text) {
    if (text == null || text.isEmpty()) {
        return text;
    }

    // 移除 Markdown 代码块标记
    String cleaned = text
        .replaceAll("```json\\s*", "")  // 移除开头的 ```json
        .replaceAll("```\\s*$", "")  // 移除结尾的 ```
        .trim();

    log.debug("清理 Markdown 标记前长度: {}, 清理后长度: {}",
            text.length(), cleaned.length());

    return cleaned;
}

// 使用
try {
    EvaluationResult result = agents.evaluator().evaluate(context);
} catch (OutputParsingException e) {
    log.warn("Agent 返回格式错误，尝试清理 Markdown 标记", e);

    // 提取原始内容
    String rawContent = e.getMessage();
    if (rawContent != null && rawContent.contains("```json")) {
        String cleaned = cleanMarkdownCodeBlocks(rawContent);

        // 手动解析
        ObjectMapper mapper = new ObjectMapper();
        EvaluationResult result = mapper.readValue(cleaned, EvaluationResult.class);
        // 使用 result
    }
}
```
