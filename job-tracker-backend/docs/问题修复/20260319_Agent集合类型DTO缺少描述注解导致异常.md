## 根本原因

### LangChain4j 的类型描述要求

**问题**：
1. LangChain4j 使用 `PojoCollectionOutputParser` 解析集合类型（`List<T>`）
2. 在生成输出格式指令时，需要为集合元素类型 `T` 生成 JSON Schema
3. **如果 `T` 是复杂类型（包含枚举），必须为枚举值添加 `@Description` 注解**
4. 否则 `formatInstructions()` 方法会抛出 `IllegalStateException: null`

### 具体原因

**缺少 `@Description` 注解的类**：

1. **`SkillCredibility.ExaggerationLevel` 枚举**
   ```java
   public enum ExaggerationLevel {
       NONE,      // ❌ 缺少 @Description
       SLIGHT,    // ❌ 缺少 @Description
       MODERATE,  // ❌ 缺少 @Description
       SEVERE     // ❌ 缺少 @Description
   }
   ```

2. **`ImprovementSuggestion.Priority` 枚举**
   ```java
   public enum Priority {
       HIGH,      // ❌ 缺少 @Description
       MEDIUM,    // ❌ 缺少 @Description
       LOW        // ❌ 缺少 @Description
   }
   ```

3. **`CredibilityAssessment.MatchLevel` 枚举**
   ```java
   public enum MatchLevel {
       PERFECT_MATCH,           // ❌ 缺少 @Description
       MOSTLY_MATCH,            // ❌ 缺少 @Description
       PARTIALLY_EXAGGERATED,   // ❌ 缺少 @Description
       SEVERELY_EXAGGERATED,    // ❌ 缺少 @Description
       EXCEEDS_EXPECTATIONS     // ❌ 缺少 @Description
   }
   ```

4. **`NextStepDecision` 的所有枚举**
   - `Action` 枚举
   - `TopicSource` 枚举
   - `QuestionType` 枚举

## 为什么需要 @Description 注解？

### LangChain4j 的工作原理

**1. 生成 JSON Schema**：
```
PojoCollectionOutputParser.formatInstructions()
    ↓
为 List<SkillCredibility> 生成 Schema
    ↓
为 SkillCredibility 类生成 Schema
    ↓
为 ExaggerationLevel 枚举生成 Schema
    ↓
需要每个枚举值的描述
```

**2. 如果缺少描述**：
```
formatInstructions() 检测到枚举没有 @Description
    ↓
无法生成完整的 JSON Schema
    ↓
抛出 IllegalStateException: null
```

**3. 如果有描述**：
```
formatInstructions() 读取 @Description 注解
    ↓
生成完整的 JSON Schema
    ↓
将 Schema 添加到 System Message
    ↓
LLM 根据 Schema 返回正确的 JSON
```

## 最佳实践

### 1. 所有 Agent 返回的 DTO 都应该添加注解

**规则**：
- ✅ 所有字段：添加 `@Description` 注解
- ✅ 所有枚举值：添加 `@Description` 注解
- ✅ 复杂类型：使用嵌套的 `@Description`

**示例**：
```java
@Data
public class MyDto {
    @Description("用户姓名")
    private String name;

    @Description("用户年龄")
    private Integer age;

    @Description("用户状态")
    private UserStatus status;

    public enum UserStatus {
        @Description("活跃用户")
        ACTIVE,

        @Description("已禁用用户")
        DISABLED
    }
}
```

---

### 2. 描述要清晰明确

**好的描述**：
```java
@Description("夸大程度评分，范围0.0-1.0。0.0-0.2表示完全匹配，0.3-0.5表示轻微夸大")
private Double exaggerationScore;
```

**不好的描述**：
```java
@Description("分数")  // ❌ 太模糊
private Double exaggerationScore;
```

---

### 3. 枚举值描述要说明含义

**好的描述**：
```java
@Description("严重夸大：实际表现远低于简历声称，完全不符合")
SEVERE
```

**不好的描述**：
```java
@Description("严重")  // ❌ 没有说明具体含义
SEVERE
```

## 相关问题

### 问题 1: 为什么其他 DTO 没有报错？

**答案**：
- `EvaluationResult` 不是集合类型，直接返回单个对象
- `NextStepDecision` 不是集合类型
- `CredibilityScoreResult` 不是集合类型

**只有集合类型**（`List<T>`）才会触发 `PojoCollectionOutputParser.formatInstructions()`，而这个方法对枚举描述要求严格。

---

### 问题 2: 为什么在运行时才报错？

**答案**：
- Java 的泛型擦除导致类型信息在运行时丢失
- LangChain4j 通过反射获取类型信息
- `formatInstructions()` 在运行时动态生成 Schema
- 只有在调用 Agent 方法时才会执行 `formatInstructions()`

2. **描述要详细**
