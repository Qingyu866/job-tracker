# 聊天记忆隔离问题分析与解决方案

## 📋 问题描述

### 现象

当用户开启新的对话（新的 sessionId）时，AI 仍然能记住之前对话的内容，不同会话之间没有隔离。

### 复现步骤

1. 使用 sessionId="5e252d2d-c2c6-0645-7be3-4bb4857f0586" 进行对话
2. 使用新的 sessionId="session-1773554134127" 开启新对话
3. AI 仍然记得之前的对话内容

### 用户反馈

> "当我开启了一个新的对话那么这边的对话记忆应该是创建一个新的，而不是全局共同使用一个对话记忆！！"

## 🔍 问题根因分析

### 架构流程

```
WebSocket 消息处理流程：
1. ChatWebSocketHandler.handleTextMessage()
2. chatHistoryService.saveUserMessage(sessionKey, content)
3. jobAgent.chat(content, currentDate, currentTime, dayOfWeek)
4. chatHistoryService.saveAssistantMessage(sessionKey, aiResponse)
```

### 核心问题

**JobAgent 是单例，无法区分不同会话的记忆**

```java
@Bean
public JobAgent jobAgent(...) {
    return AiServices.builder(JobAgent.class)
        .chatMemoryProvider(chatMemoryProvider)  // ← SafeTurnBasedChatMemoryProvider 单例
        .build();
}
```

**`SafeTurnBasedChatMemoryProvider` 虽然为每个 memoryId 创建独立记忆，但 `jobAgent.chat()` 方法没有接收 memoryId 参数！**

```java
String chat(@UserMessage String userMessage, ...);
// 没有 memoryId 参数！！
```

**关键代码位置**：

| 文件 | 问题 |
|------|------|
| `LangChain4jConfig.java` | JobAgent 是单例，无法传递 memoryId |
| `JobAgent.java` | chat() 方法没有 memoryId 参数 |
| `ChatWebSocketHandler.java` | 无法指定使用哪个记忆 |

### 项目现有实现

项目中已有自定义的 `SafeTurnBasedChatMemory` 和 `SafeTurnBasedChatMemoryProvider`：

- **SafeTurnBasedChatMemory**：基于轮次的记忆清理（不会打断对话轮次）
- **SafeTurnBasedChatMemoryProvider**：为每个 memoryId 创建独立记忆
- 这些实现是正确的，但 JobAgent 无法利用它们

## 💡 解决方案

### 方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **方案1：动态 Agent + 已有 SafeTurnBasedChatMemory** | 使用已有实现，真正的会话隔离 | 需要改动调用方式 | ⭐⭐⭐⭐⭐ |
| 方案2：原型 JobAgent | 不需要大改动 | 资源浪费，每次创建新 Agent | ⭐⭐ |
| 方案3：ThreadLocal | 简单 | 并发安全问题 | ⭐ |

### 推荐方案：动态 Agent + 已有 SafeTurnBasedChatMemory

**核心思路**：
1. 创建 `JobAgentFactory`（动态创建 JobAgent 实例，直接使用 `SafeTurnBasedChatMemoryProvider`）
2. 修改 `ChatWebSocketHandler` 使用工厂模式
3. **使用项目中已有的 `SafeTurnBasedChatMemory`，不要使用官方的 `MessageWindowChatMemory`**

**重要说明**：
- 官方的 `MessageWindowChatMemory` 有轮次打断问题
- 项目已自定义 `SafeTurnBasedChatMemory` 解决此问题
- 必须继续使用 `SafeTurnBasedChatMemory`，不能用回官方实现

## 🛠️ 实施步骤

### 步骤 1：创建 JobAgentFactory

动态创建 JobAgent 实例，**直接使用 SafeTurnBasedChatMemoryProvider（它内部已有缓存）：**

```java
@Component
@RequiredArgsConstructor
public class JobAgentFactory {
    private final StreamingChatModel streamingChatModel;
    private final OpenAiChatModel chatModel;
    private final SafeTurnBasedChatMemoryProvider chatMemoryProvider;  // ← 直接注入
    // 工具类...

    public JobAgent createAgent(String sessionId) {
        // ← 直接使用 provider，它内部已经有缓存管理了
        ChatMemory memory = chatMemoryProvider.get(sessionId);

        return AiServices.builder(JobAgent.class)
            .streamingChatModel(streamingChatModel)
            .chatModel(chatModel)
            .chatMemory(memory)  // 使用独立的 ChatMemory
            .tools(...)
            .build();
    }

    // 清理会话
    public void clearSession(String sessionId) {
        chatMemoryProvider.clear(sessionId);
    }

    // 获取活跃会话数
    public int getActiveSessionCount() {
        return chatMemoryProvider.activeSessionCount();
    }
}
```

**注意**：不需要额外的 `ChatMemoryManager`，因为 `SafeTurnBasedChatMemoryProvider` 内部已经有 `ConcurrentHashMap` 来管理缓存。

### 步骤 2：修改 ChatWebSocketHandler

```java
@Component
@RequiredArgsConstructor
public class JobAgentFactory {
    private final StreamingChatModel streamingChatModel;
    private final ChatMemoryManager chatMemoryManager;
    // 工具类...

    public JobAgent createAgent(String sessionId) {
        ChatMemory memory = chatMemoryManager.getMemory(sessionId);

        return AiServices.builder(JobAgent.class)
            .streamingChatModel(streamingChatModel)
            .chatMemory(memory)  // 使用独立的 ChatMemory
            .tools(...)
            .build();
    }
}
```

### 步骤 3：修改 ChatWebSocketHandler

```java
// 原来（单例，共享记忆）
@Autowired
private JobAgent jobAgent;
String response = jobAgent.chat(content, ...);

// 新方案（动态创建，独立记忆）
@Autowired
private JobAgentFactory jobAgentFactory;
JobAgent agent = jobAgentFactory.createAgent(sessionKey);
String response = agent.chat(content, ...);
```

### 步骤 4：清理过期记忆

定期清理长时间未使用的会话记忆：

```java
@Scheduled(fixedRate = 3600000)  // 每小时
public void cleanupExpiredMemories() {
    // 清理超过 1 小时未使用的记忆
}
```

## ⚠️ 注意事项

### 技术约束

1. **必须使用 `SafeTurnBasedChatMemory`**：不要使用官方的 `MessageWindowChatMemory`
2. **保持轮次完整性**：SafeTurnBasedChatMemory 按轮次清理，不会打断对话

### 性能影响

- **优点**：真正的会话隔离，用户体验更好
- **缺点**：每次请求需要创建新的 JobAgent 实例
- **优化**：可以考虑缓存 JobAgent 实例

### 内存管理

- **风险**：大量会话可能导致内存泄漏
- **解决**：实现定期清理过期记忆的机制
- **监控**：添加会话数量监控，设置上限

### 兼容性

- **数据库**：不需要修改，chat_sessions 表保持不变
- **前端**：不需要修改，仍然使用相同的 sessionId
- **测试**：需要更新测试用例
