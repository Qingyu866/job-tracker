# AI 工具参数简化优化方案

## 问题分析

### 问题概述
当前 AI 工具中存在参数过多的问题，导致 AI 不敢或不愿意使用这些工具。用户反馈："有些工具的字段太多导致 AI 不敢使用这个 tool"。

### 受影响的工具

| 工具名称 | 参数数量 | 必填参数 | 严重程度 |
|---------|---------|---------|---------|
| `createApplication` | 10 | 2 | 🔴 高 |
| `createCompany` | 6 | 1 | 🟡 中 |
| `createInterview` | 6 | 2 | 🟡 中 |

### 根本原因
1. **认知负担**：参数数量越多，AI 决策成本越高
2. **参数焦虑**：即使标记为可选，AI 也可能担心遗漏重要信息
3. **用户习惯**：用户最常使用的是基础功能，不需要那么多可选参数

---

## 解决方案

### 核心策略：直接简化现有工具

**不新增工具**，直接修改现有工具的方法签名和参数：
- 移除低频使用的可选参数
- 保留核心必填参数
- 对于需要补充信息的场景，引导用户使用 update 工具

### 优势
1. 不增加工具数量，不增加 AI 的选择成本
2. 工具名称保持不变，无需学习新工具
3. 大幅降低使用门槛

## 详细设计

### 1. ApplicationCommandTools.createApplication

#### 修改前（10个参数）
```java
@Tool("""
    创建新的求职申请

    必填参数：
    - companyName: 公司名称
    - jobTitle: 职位名称

    可选参数：
    - jobType: 工作类型（全职/兼职/实习）
    - workLocation: 工作地点
    - salaryMin: 最低薪资
    - salaryMax: 最高薪资
    - status: 申请状态（默认愿望清单）
    - priority: 优先级（1-10，默认5）
    - notes: 备注
    """)
public ToolResult createApplication(
    String companyName,
    String jobTitle,
    String jobType,
    String workLocation,
    BigDecimal salaryMin,
    BigDecimal salaryMax,
    String status,
    Integer priority,
    String notes
)
```

#### 修改后（2个参数）
```java
@Tool("""
    创建新的求职申请（快速创建，仅需基本信息）

    适用场景：
    - 用户说"帮我创建字节跳动的申请"
    - 用户说"添加一个阿里巴巴前端工程师的申请"

    必填参数：
    - companyName: 公司名称
    - jobTitle: 职位名称

    说明：
    - 创建后默认状态为"愿望清单"
    - 其他信息（薪资、地点等）可通过更新工具补充
    - 返回：创建结果，包含新申请ID
    """)
public ToolResult createApplication(
    String companyName,
    String jobTitle
)
```

#### 移除的参数及处理方式
| 移除参数 | 处理方式 |
|---------|---------|
| `jobType` | 不设置，留空 |
| `workLocation` | 不设置，留空 |
| `salaryMin` | 不设置，留空 |
| `salaryMax` | 不设置，留空 |
| `status` | **默认为 WISHLIST（愿望清单）** |
| `priority` | **默认为 5（中等优先级）** |
| `notes` | 不设置，留空 |

## 实现细节

### 代码修改示例

#### ApplicationCommandTools.java

```java
// 修改前
public ToolResult createApplication(
    String companyName,
    String jobTitle,
    String jobType,
    String workLocation,
    BigDecimal salaryMin,
    BigDecimal salaryMax,
    String status,
    Integer priority,
    String notes
) {
    // 原有实现...
}

// 修改后
public ToolResult createApplication(String companyName, String jobTitle) {
    // 1. 验证必填参数
    if (StringUtils.isBlank(companyName) || StringUtils.isBlank(jobTitle)) {
        return ToolResult.error(ToolConstants.ERR_PARAM_MISSING, "公司名称和职位不能为空");
    }

    // 2. 查找或创建公司
    Company company = toolHelper.smartMatchCompany(companyName);

    // 3. 创建申请（使用默认值）
    JobApplication application = JobApplication.builder()
        .companyId(company.getId())
        .jobTitle(jobTitle)
        .status(ApplicationStatus.WISHLIST.getCode())  // 默认愿望清单
        .priority(ToolConstants.DEFAULT_PRIORITY)       // 默认优先级5
        .applicationDate(LocalDate.now())
        .build();

    jobApplicationMapper.insert(application);

    return ToolResult.success("申请创建成功，ID: " + application.getId(), application);
}
```

## 工具描述优化原则

### 优化后的描述模板

```java
@Tool("""
    [功能概述]

    适用场景：
    - 用户示例1
    - 用户示例2

    必填参数：
    - 参数名: 参数说明

    说明：
    - 默认行为说明
    - 补充信息的方式（如需）
    - 返回：返回值说明
    """)
```

### 优化要点
1. **场景前置**：先说明使用场景，让 AI 快速理解
2. **示例驱动**：提供用户实际使用的例子
3. **默认值透明**：明确说明默认值是什么
4. **补充引导**：告知如何补充缺失信息

## 风险评估

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| 用户需要更多信息 | 低 | 引导使用 update 工具补充 |
| 默认值不符合预期 | 低 | 默认值选择最常用、最安全的选项 |
| 破坏向后兼容 | 无 | 方法签名改变，但无外部直接调用 |

---

## 总结

本方案通过**直接简化现有工具**的方式，在不增加工具数量的前提下，大幅降低了 AI 使用工具的门槛。移除低频参数并设置合理默认值，预计可提升 50% 的工具调用成功率。
