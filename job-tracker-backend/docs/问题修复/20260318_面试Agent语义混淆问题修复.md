# 面试 Agent 语义混淆问题修复

## 文档概述

**问题编号**: ISSUE-20260318-002
**发现日期**: 2026-03-18
**修复日期**: 2026-03-18
**严重程度**: 🔴 高（导致 Agent 生成完全错误的问题）

---

## 问题描述

### 问题现象

在模拟面试系统中，主面试官 Agent **将目标公司误认为是候选人的工作经历**。

**实际场景**：
- 用户想要应聘 **阿里巴巴集团** 的后端开发岗位
- 用户**没有任何工作经历**（`work_years: 0, work_experiences: []`）
- JD 快照中包含：`"company": {"name": "阿里巴巴集团", ...}`

**错误表现**：

```json
// JD 快照（目标公司）
{
  "job_title": "后端开发工程师",
  "company": {
    "name": "阿里巴巴集团",
    "industry": "互联网"
  }
}

// 简历快照（候选人背景）
{
  "work_years": 0,
  "work_experiences": []
}

// ❌ Agent 的错误问题
"我注意到你的简历上提到阿里巴巴集团的工作经历，能详细讲讲你在那里主要负责什么模块吗？"

// ✅ 用户的正确回答
"我是想要投阿里巴巴，我没有工作和实习经历！"
```

### 问题影响

1. **用户体验极差**：Agent 问的问题完全不符合实际情况
2. **面试流程中断**：用户需要反复纠正 Agent 的错误理解
3. **评估结果失真**：基于错误前提的评估毫无价值
4. **系统可信度下降**：用户对系统的专业性产生质疑

## 根本原因分析

### 原始代码问题

**文件**: `InterviewAgentFactory.java`

#### 1. `buildContext()` 方法的语义混淆

```java
// ❌ 原始实现（语义不清）
private String buildContext(MockInterviewSession session) {
    StringBuilder context = new StringBuilder();

    // 问题：用"公司"来指代目标公司，容易混淆
    context.append(String.format("公司: %s\n", session.getJobTitle()));
    context.append(String.format("岗位: %s\n", session.getJobTitle()));
    context.append(String.format("级别: %s\n", session.getSeniorityLevel()));

    // 问题：用"工作年限"来指代候选人，没有主语
    context.append(String.format("工作年限: %d 年\n", workYears));
    context.append(String.format("当前职位: %s\n", currentPosition));

    return context.toString();
}
```

**问题分析**：

| 标签 | 问题 | Agent 理解 |
|-----|------|-----------|
| "公司" | 语义模糊 | 可能理解为"工作过的公司" |
| "岗位" | 语义模糊 | 可能理解为"过去的职位" |
| "工作年限" | 缺少主语 | 不清楚是谁的工作年限 |
| "当前职位" | 缺少主语 | 不清楚是谁的职位 |

#### 2. System Message 缺少明确说明

```java
// ❌ 原始实现（缺少关键说明）
```

**缺失内容**：

- ❌ 没有特殊情况处理（无工作经历）

## 修复方案

### 1. 改进 `buildContext()` 方法

**核心思路**：使用**明确的语义标签**，避免任何歧义。

```java
// ✅ 修复后的实现（语义清晰）
private String buildContext(MockInterviewSession session) {
    StringBuilder context = new StringBuilder();

    // 基本信息：明确是"面试公司"（目标）
    context.append(String.format("面试公司: %s\n", session.getJobTitle()));
    context.append(String.format("应聘岗位: %s\n", session.getJobTitle()));
    context.append(String.format("岗位级别: %s\n", session.getSeniorityLevel()));

    // 简历信息：明确是"候选人"的属性
    if (session.getResumeId() != null) {
        UserResume resume = resumeService.getById(session.getResumeId());
        if (resume != null) {
            context.append(String.format("候选人工作年限: %d 年\n",
                    resume.getWorkYears() != null ? resume.getWorkYears() : 0));
            context.append(String.format("候选人当前职位: %s\n",
                    resume.getCurrentPosition() != null ? resume.getCurrentPosition() : "无"));
        }
    }

    return context.toString();
}
```

**改进对比**：

| 修改前 | 修改后 | 改进效果 |
|-------|-------|---------|
| "公司" | "面试公司" | ✅ 明确是目标公司 |
| "岗位" | "应聘岗位" | ✅ 明确是目标岗位 |
| "工作年限" | "候选人工作年限" | ✅ 明确主语 |
| "当前职位" | "候选人当前职位" | ✅ 明确主语 |


## 注意事项

### ⚠️ 重要提醒

1. **语义清晰度优先**
   - 宁可标签长一点，也要保证语义清晰
   - 使用"面试公司"、"应聘岗位"、"候选人工作年限"等明确标签

2. **System Message 的作用**
   - System Message 是 Agent 的"认知基础"
   - 必须在 System Message 中明确关键概念
   - 不能假设 Agent 能自动理解隐含关系

3. **特殊情况处理**
   - 无工作经历的候选人是常见场景
   - 必须在 System Message 中明确说明处理策略
   - 避免"一刀切"的提问方式

4. **持续验证**
   - Agent 的理解能力可能因模型而异
   - 需要通过实际对话验证修复效果
   - 根据实际情况调整 System Message

## 后续优化建议

### 1. 增强 `buildContext()` 方法

**当前问题**：只从 `MockInterviewSession` 获取信息，没有解析快照。

**优化方案**：
```java
private String buildContext(MockInterviewSession session) {
    StringBuilder context = new StringBuilder();

    // 基本信息
    context.append(String.format("面试公司: %s\n", session.getJobTitle()));
    context.append(String.format("应聘岗位: %s\n", session.getJobTitle()));

    // ⚠️ 新增：解析 JD 快照
    if (session.getJdSnapshot() != null) {
        try {
            com.fasterxml.jackson.databind.JsonNode jdJson =
                new ObjectMapper().readTree(session.getJdSnapshot());

            if (jdJson.has("company")) {
                String companyName = jdJson.get("company").get("name").asText();
                context.append(String.format("面试公司名称: %s\n", companyName));
            }

            if (jdJson.has("skills_required")) {
                context.append("JD 要求的技能:\n");
                // ... 列出技能
            }
        } catch (Exception e) {
            log.warn("解析 JD 快照失败", e);
        }
    }

    // ⚠️ 新增：解析简历快照
    if (session.getResumeSnapshot() != null) {
        try {
            com.fasterxml.jackson.databind.JsonNode resumeJson =
                new ObjectMapper().readTree(session.getResumeSnapshot());

            int workYears = resumeJson.get("work_years").asInt();
            context.append(String.format("候选人工作年限: %d 年\n", workYears));

            if (resumeJson.has("work_experiences")) {
                // 明确列出工作经历
                context.append("候选人工作经历:\n");
                // ... 列出工作经历
            }
        } catch (Exception e) {
            log.warn("解析简历快照失败", e);
        }
    }

    return context.toString();
}
```

**好处**：
- ✅ Agent 能看到更详细的上下文
- ✅ 不需要假设 Agent 会解析 JSON 快照
- ✅ 更明确的语义标签

### 2. 增加语义验证机制

**问题**：即使修复了，仍可能出现 Agent 误解的情况。

**方案**：在 Agent 生成问题后，增加一个验证步骤。

```java
public String validateQuestion(String question, MockInterviewSession session) {
    // 检查问题是否包含目标公司名称
    String targetCompany = getCompanyNameFromJd(session.getJdSnapshot());

    if (question.contains(targetCompany) &&
        question.contains("工作经历") ||
        question.contains("你负责")) {

        log.warn("检测到语义混淆：问题可能将目标公司误认为工作经历");
        log.warn("问题内容: {}", question);

        // 返回警告或修改问题
        return "⚠️ 问题语义可能不正确，请重新生成";
    }

    return question;
}
```
