# Tool Call Records 修复说明（最终方案）

## 问题描述

`tool_call_records` 表一直为空，虽然表结构、实体类、Mapper 和服务方法都已存在。

## 根本原因

LangChain4j 的 `AiServices` 直接调用工具方法，没有提供工具调用的回调机制。

## ❌ 为什么 AOP 方案不可行

Spring AOP 使用 CGLIB 创建代理类（如 `TimeTools$SpringCGLIB$0`），而 LangChain4j 需要直接扫描原始类的 `@Tool` 注解。

错误信息：
```
Object 'com.jobtracker.agent.tools.shared.TimeTools$SpringCGLIB$0' does not have any methods annotated with @Tool
```

**结论**：Spring AOP 与 LangChain4j 的工具系统不兼容。

## ✅ 最终解决方案：手动记录

在每个工具方法中手动调用 `ToolCallTrace.record()` 来记录工具调用。

### 实现步骤

1. **创建 `ToolCallTrace.java`**
   - 提供静态方法 `record()` 和 `recordFailure()`
   - 自动获取当前 AI 消息 ID
   - 异步保存到数据库

2. **在工具方法中添加记录调用**
   ```java
   public ToolResult getApplicationById(Long applicationId) {
       long startTime = System.currentTimeMillis();
       try {
           // 业务逻辑
           ToolResult result = doSomething();
           // 记录成功
           ToolCallTrace.record("getApplicationById", applicationId, result, startTime);
           return result;
       } catch (Exception e) {
           // 记录失败
           ToolCallTrace.recordFailure("getApplicationById", applicationId, e, startTime);
           throw e;
       }
   }
   ```

### 示例代码

已在 `ApplicationQueryTools.java` 的 `getApplicationById()` 方法中实现。

## 代码模板

```java
@Tool("工具描述")
public ToolResult toolMethod(ParamType param) {
    long startTime = System.currentTimeMillis();
    log.info("AI调用：工具方法 param={}", param);

    ToolResult result;
    try {
        // 参数验证
        if (param == null) {
            result = ToolResult.error("ERR_PARAM_MISSING", "参数不能为空");
        } else {
            // 业务逻辑
            Object data = doBusinessLogic(param);
            result = ToolResult.success("操作成功", data);
        }

        // 记录成功
        ToolCallTrace.record("toolMethod", param, result, startTime);
        return result;
    } catch (Exception e) {
        // 记录失败
        ToolCallTrace.recordFailure("toolMethod", param, e, startTime);
        throw e;
    }
}
```

## 记录的数据内容

每条工具调用记录包含：
- `id`: 主键
- `message_id`: 关联的 AI 消息 ID
- `tool_name`: 工具名称（如 "getApplicationById"）
- `tool_input`: 输入参数（JSON）
- `tool_output`: 输出结果（JSON）
- `status`: SUCCESS 或 FAILURE
- `error_message`: 错误信息（失败时）
- `execution_time_ms`: 执行耗时（毫秒）
- `created_at`: 创建时间

## 注意事项

1. **性能影响**: 每次工具调用会增加 1 次 INSERT 操作，但影响很小
2. **数据量**: 频繁使用会产生大量记录，建议定期清理历史数据
3. **消息 ID 关联**: 通过查询最新 ASSISTANT 消息获取，可能在高并发下有问题
4. **异步优化**: 目前是同步保存，如需优化可改为异步（使用 `@Async`）
