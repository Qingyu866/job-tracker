## 一、核心概念

### 1.1 OCR 是什么

OCR（Optical Character Recognition，光学字符识别）是指将图片中的文字转换为可编辑文本的技术。

```
输入图片 → OCR 引擎 → 识别结果
  JPG/PNG    智谱 API    JSON 文本
```

### 1.2 为什么需要 OCR

| 对比项 | Gemma3-4b 视觉能力 | 专业 OCR 服务 |
|--------|-------------------|---------------|
| 文字识别准确率 | 70-80% | 95%+ |
| 表格识别 | ❌ | ✅ |
| 版面分析 | ❌ | ✅ |

### 1.3 智能降级策略

```
用户上传图片
    │
    ├─ 有网络 → 调用智谱 OCR API
    │              │
    │              ├─ 成功 → 返回结果
    │              └─ 失败 → 降级到本地 Gemma3-4b
    │
    └─ 无网络 → 直接使用本地 Gemma3-4b
```

### 2.1 整体架构


## 三、数据模型

### 3.1 OCR 调用记录表

```sql
CREATE TABLE ocr_call_records (
    record_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    session_id VARCHAR(100),
    user_id BIGINT NOT NULL,

    -- 请求信息
    image_url VARCHAR(500),
    image_type VARCHAR(20),      -- RESUME/JD/GENERAL
    ocr_provider VARCHAR(50),

    -- 响应信息
    ocr_status VARCHAR(20),       -- SUCCESS/FAILED/FALLBACK
    recognized_text TEXT,
    confidence_score DECIMAL(3,2),
    processing_time_ms INT,

    -- 错误信息
    error_code VARCHAR(50),
    error_message VARCHAR(500),

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 四、AI 工具设计

### 4.1 三个 OCR 工具

#### recognizeImage - 通用文字识别
```java
@Tool("[识别] 识别图片中的文字内容")
ToolResult recognizeImage(String imageData)
```
- **场景**: 用户说"识别一下这张图片"
- **参数**: Base64 编码的图片数据
- **返回**: 识别的文本内容

#### recognizeResume - 简历识别
```java
@Tool("[识别] 识别简历图片并解析结构化信息")
ToolResult recognizeResume(String imageData)
```
- **场景**: 用户上传简历图片
- **功能**: 识别并自动创建 `user_resumes` 记录

#### recognizeJD - JD 识别
```java
@Tool("[识别] 识别 JD 图片并提取技能要求")
ToolResult recognizeJD(String imageData, Long applicationId)
```
- **场景**: 用户上传 JD 截图
- **功能**: 提取技能标签并匹配 `skill_tags`

### 4.2 工具描述模板

```
@Tool("""
    [识别] 识别图片中的文字内容

    适用场景：
    - 用户说"识别一下这张图片"
    - 用户说"这是什么内容"
    - 用户上传图片并询问内容

    参数：
    - imageData: Base64编码的图片数据

    说明：
    - 优先使用智谱 OCR 云服务
    - 云服务失败时降级到本地模型
    - 返回：识别的文本内容
    """)
```

## 六、实现要点

### 6.1 智谱 OCR 调用

```java
// 1. 构建multipart请求
MultipartBody.Builder builder = new MultipartBody.Builder()
    .setType(MultipartBody.FORM)
    .addFormDataPart("image", base64Image);

// 2. 设置认证头
Request request = new Request.Builder()
    .url(apiUrl)
    .post(builder.build())
    .addHeader("Authorization", "Bearer " + apiKey)
    .build();

// 3. 发送请求并解析响应
Response response = client.newCall(request).execute();
ZhipuOcrResponse ocrResponse = parseResponse(response.body().string());
```

### 6.2 降级逻辑

```java
public OcrResult recognize(byte[] imageData, OcrOptions options) {
    // 1. 尝试智谱 OCR
    try {
        OcrResult result = zhipuOcrClient.recognize(imageData, options);
        if (result.getStatus() == OcrStatus.SUCCESS) {
            return result;
        }
    } catch (Exception e) {
        log.warn("智谱 OCR 调用失败", e);
    }

    // 2. 降级到本地模型
    return fallbackToLocalModel(imageData, options);
}
```

### 6.3 置信度判断

```java
if (result.getConfidence() < 0.6) {
    // 置信度过低，标记为降级
    result.setStatus(OcrStatus.FALLBACK);
}
```

## 八、常见问题

### Q1: 为什么不直接用 Gemma3-4b？

**A**: Gemma3-4b 的文字识别准确率只有 70-80%，且不支持表格识别和版面分析。专业 OCR 服务准确率可达 95%+。

### Q2: 降级策略什么时候触发？

**A**:
1. 网络不可用时
2. 智谱 API 调用失败时
3. 响应超时时（30秒）
4. 识别置信度过低时

### Q3: 如何控制成本？

**A**:
1. 设置用户配额（如每月 20 次）
2. 简单图片优先用本地模型
3. 缓存相同图片的识别结果
