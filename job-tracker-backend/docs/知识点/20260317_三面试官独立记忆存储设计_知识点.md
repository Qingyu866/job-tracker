## 一、核心概念

### 1.1 sessionId 定义

**⚠️ 重要**：本设计中的 `sessionId` 特指**模拟面试系统**的会话ID

```
sessionId = MockInterviewSession.sessionId
           来自 mock_interview_sessions 表
           标识一次独立的模拟面试过程
```

**与其他系统的区别**：

| 系统 | 表/实体 | ID 字段 | 用途 |
|------|---------|---------|------|
| 模拟面试系统 | `mock_interview_sessions` | `session_id` | 标识一次模拟面试会话 |
| 聊天系统 | `chat_sessions` | `session_key` | 标识一次 AI 聊天对话 |

### 1.2 两层隔离架构

#### 第一层：会话间隔离

不同模拟面试会话的 Agent 记忆完全独立：
```
session-byt-001 (字节跳动面试) ≠ session-tx-002 (腾讯面试)
```

#### 第二层：Agent 间隔离

同一会话内三个 Agent 的记忆完全独立：
```
主面试官记忆 ≠ 副面试官记忆 ≠ 评审专家记忆
```

## 二、问题背景

### 2.1 当前架构问题

#### 问题 1：同会话内三个 Agent 共享记忆
```
三个 Agent 的"思考过程"混在一起
❌ 主面试官可能看到评审专家的评分逻辑
❌ 副面试官可能看到主面试官的提问策略
```

#### 问题 2：不同会话之间记忆串扰
```
全局共享 ChatMemoryProvider
❌ Session A 的面试官可能看到 Session B 的对话历史
❌ 不同用户的面试内容可能互相"泄露"
```

### 2.2 严重性

| 问题 | 严重性 | 影响 |
|------|--------|------|
| 同会话记忆污染 | 🔴 高 | AI 决策被干扰，不符合真实面试场景 |
| 跨会话记忆串扰 | 🔴🔴 严重 | 用户隐私泄露，面试内容混淆 |

### 3.2 隔离机制

| 隔离维度 | 隔离机制 | 技术实现 |

## 四、数据模型

### 4.1 interview_memories 表

```sql
CREATE TABLE interview_memories (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    session_id VARCHAR(64) NOT NULL UNIQUE COMMENT '模拟面试会话ID',

    -- 三个 Agent 的独立记忆（JSON 存储）
    main_interviewer_memory JSON COMMENT '主面试官记忆',
    vice_interviewer_memory JSON COMMENT '副面试官记忆',
    evaluator_memory JSON COMMENT '评审专家记忆',

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_session_id (session_id)
) COMMENT='面试 Agent 独立记忆存储（两层隔离）';
```

## 五、核心类设计

### 5.1 AgentType 枚举

```java
public enum AgentType {
    MAIN_INTERVIEWER("main", "主面试官"),
    VICE_INTERVIEWER("vice", "副面试官"),
    EVALUATOR("evaluator", "评审专家");
}
```

### 5.2 SessionMemories 容器

```java
@Data
public class SessionMemories {
    private final String sessionId;
    private ChatMemory mainInterviewerMemory;
    private ChatMemory viceInterviewerMemory;
    private ChatMemory evaluatorMemory;

    public ChatMemory getMemory(AgentType agentType) {
        return switch (agentType) {
            case MAIN_INTERVIEWER -> mainInterviewerMemory;
            case VICE_INTERVIEWER -> viceInterviewerMemory;
            case EVALUATOR -> evaluatorMemory;
        };
    }
}
```

### 5.3 InterviewMemoryProvider

```java
@Slf4j
@Component
@RequiredArgsConstructor
public class InterviewMemoryProvider {

    // L1 内存缓存：sessionId → SessionMemories
    private final Map<String, SessionMemories> memoryCache = new ConcurrentHashMap<>();

    /**
     * 获取指定会话和 Agent 类型的 ChatMemory
     * 关键方法：确保 (sessionId, agentType) 组合有独立的 ChatMemory 实例
     */
    public ChatMemory getChatMemory(String sessionId, AgentType agentType) {
        // 1. 获取或创建会话的 SessionMemories（会话隔离）
        SessionMemories sessionMemories = memoryCache.computeIfAbsent(
            sessionId,
            id -> loadOrCreateSessionMemories(id)
        );

        // 2. 获取或创建指定 Agent 的 ChatMemory（Agent 隔离）
        ChatMemory memory = sessionMemories.getMemory(agentType);
        if (memory == null) {
            memory = createChatMemory(sessionId, agentType);
            sessionMemories.setMemory(agentType, memory);
        }

        return memory;
    }

    /**
     * 持久化会话的所有 Agent 记忆
     */
    public void persistAll(String sessionId) {
        persistMemory(sessionId, AgentType.MAIN_INTERVIEWER);
        persistMemory(sessionId, AgentType.VICE_INTERVIEWER);
        persistMemory(sessionId, AgentType.EVALUATOR);
    }
}
```

## 六、使用方式

### 6.1 在 InterviewAgentFactory 中使用

```java
public InterviewAgents createAgents(MockInterviewSession session) {
    String sessionId = session.getSessionId();

    // 为每个 Agent 获取独立的 ChatMemory
    ChatMemory mainMemory = memoryProvider.getChatMemory(
        sessionId, AgentType.MAIN_INTERVIEWER
    );
    ChatMemory viceMemory = memoryProvider.getChatMemory(
        sessionId, AgentType.VICE_INTERVIEWER
    );
    ChatMemory evalMemory = memoryProvider.getChatMemory(
        sessionId, AgentType.EVALUATOR
    );

    // 创建 Agent（使用独立记忆）
    MainInterviewerAgent main = AiServices.builder(MainInterviewerAgent.class)
        .chatModel(chatModel)
        .chatMemory(mainMemory)  // 完全独立
        .systemMessage(buildMainSystemMessage())
        .build();

    // ... 创建其他 Agent

    return new InterviewAgents(sessionId, main, vice, eval);
}
```

## 七、记忆同步策略

| 场景 | 同步策略 | 说明 |
|------|----------|------|
| 每轮对话后 | 异步持久化 | 不阻塞对话，后台更新数据库 |
| 面试结束时 | 同步持久化 | 确保所有数据保存 |
| Agent 缓存清除 | 先持久化再清除 | 防止数据丢失 |
| 服务器重启 | 从数据库恢复 | Provider 启动时懒加载 |

## 八、隔离验证

### 8.1 验证两层隔离

```java
@Test
public void testMemoryIsolation() {
    String sessionA = "session-abc-123";
    String sessionB = "session-xyz-789";

    // 获取 Session A 的主面试官记忆
    ChatMemory sessionA_main = memoryProvider.getChatMemory(
        sessionA, AgentType.MAIN_INTERVIEWER
    );
    sessionA_main.add(UserMessage.from("Session A 的用户消息"));

    // 获取 Session B 的主面试官记忆
    ChatMemory sessionB_main = memoryProvider.getChatMemory(
        sessionB, AgentType.MAIN_INTERVIEWER
    );
    sessionB_main.add(UserMessage.from("Session B 的用户消息"));

    // 获取 Session A 的副面试官记忆
    ChatMemory sessionA_vice = memoryProvider.getChatMemory(
        sessionA, AgentType.VICE_INTERVIEWER
    );

    // 验证会话间隔离
    assertNotEquals(sessionA_main.messages(), sessionB_main.messages());

    // 验证 Agent 间隔离
    assertNotEquals(sessionA_main.messages(), sessionA_vice.messages());

    // 验证内存地址完全不同
    assertNotSame(sessionA_main, sessionB_main);
    assertNotSame(sessionA_main, sessionA_vice);
}
```

```

├─ L1 缓存：JVM 内存（ConcurrentHashMap）

    ├─ 面试结束时清除
    ├─ 超过 24 小时未活动自动清除
    └─ 内存超过阈值时 LRU 淘汰

JSON 字段大小：
├─ main_interviewer_memory: 10-50KB
├─ vice_interviewer_memory: 5-20KB
└─ evaluator_memory: 10-30KB
```

## 十一、常见问题

### Q1: sessionId 是什么？

**A**: sessionId 是模拟面试系统的会话ID（`MockInterviewSession.sessionId`），标识一次独立的模拟面试过程，不是聊天系统的 session_key。

### Q2: 为什么需要两层隔离？

**A**:
1. **会话间隔离**：用户可能同时准备多个公司的面试，不能让"字节跳动面试"的内容泄露到"腾讯面试"
2. **Agent 间隔离**：三个 Agent 各司其职，主面试官不应该看到评审专家的评分逻辑

### Q3: 记忆什么时候持久化？

**A**:
- 每轮对话后：可选，异步持久化
- 面试结束时：必须，同步持久化所有 Agent 记忆

### Q4: 服务器重启后记忆会丢失吗？

**A**: 不会。记忆已持久化到数据库的 JSON 字段中，重启后会自动从数据库恢复。
