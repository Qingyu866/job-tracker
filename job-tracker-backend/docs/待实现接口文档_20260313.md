## 二、删除策略说明

### 2.1 保护性删除策略

为避免误删重要数据，采用**保护性删除**策略：

#### 删除申请 (Application)
```
1. 检查是否有关联的面试记录
   ├── 有 → 返回错误提示："该申请下有 X 条面试记录，请先删除面试记录"
   └── 无 → 执行逻辑删除（deleted = 1）
```

#### 删除公司 (Company)
```
1. 检查是否有关联的申请记录
   ├── 有 → 返回错误提示："该公司下有 X 条申请记录，请先删除申请记录"
   └── 无 → 执行逻辑删除（deleted = 1）
```

#### 删除面试 (Interview)
```
直接执行逻辑删除（deleted = 1），无前置检查
```

### 2.2 逻辑删除优势

- 数据可恢复：通过数据库操作可恢复误删数据
- 保留历史：便于数据分析和审计
- 外键安全：不会破坏数据完整性
