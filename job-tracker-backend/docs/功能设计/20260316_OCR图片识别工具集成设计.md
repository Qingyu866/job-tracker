## 1. 问题分析

### 1.1 当前现状

**现有能力**：
- 后端已集成 `MultimodalJobAgent` 支持多模态输入
- 前端 `ChatInput.tsx` 支持图片上传
- 使用 LM Studio 的 Gemma3-4b 模型进行图片识别

**存在问题**：
```
用户场景1：上传简历图片
├─ Gemma3-4b 识别结果：文字识别准确率约 70-80%
├─ 问题：结构化信息丢失（如工作经历、技能清单等）
└─ 后果：AI 无法准确解析简历结构

用户场景2：上传 JD 截图
├─ Gemma3-4b 识别结果：能识别文字，但遗漏格式信息
├─ 问题：岗位要求、职责区分不清
└─ 后果：提取的技能标签不准确
```

### 1.2 为什么需要 OCR

| 对比项 | Gemma3-4b 视觉能力 | 专业 OCR 服务 |
|--------|-------------------|---------------|
| 文字识别准确率 | 70-80% | 95%+ |
| 表格识别 | ❌ 不支持 | ✅ 支持 |
| 版面分析 | ❌ 不支持 | ✅ 支持 |
| 手写识别 | ❌ 不支持 | ✅ 支持 |
| 扫描件处理 | ⚠️ 一般 | ✅ 优秀 |
| 成本 | 本地免费 | 云服务付费 |

### 1.3 核心使用场景

```
场景1：简历上传与解析
用户上传简历图片 → OCR识别 → 结构化解析 → 存入 user_resumes 表

场景2：JD 图片提取
用户上传JD截图 → OCR识别 → 提取技能要求 → 匹配 skill_tags

场景3：聊天中的图片附件
用户在聊天中发送图片 → OCR识别 → AI理解上下文 → 智能回复
```

## 3. 数据模型设计

### 3.1 OCR 调用记录表

```sql
CREATE TABLE ocr_call_records (
    record_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    session_id VARCHAR(100) COMMENT '关联的聊天会话',
    user_id BIGINT NOT NULL COMMENT '用户ID',

    -- 请求信息
    image_url VARCHAR(500) COMMENT '图片URL（OSS存储）',
    image_type VARCHAR(20) COMMENT '图片类型：RESUME/JD/GENERAL',
    ocr_provider VARCHAR(50) DEFAULT 'ZHIPU' COMMENT 'OCR提供商',
    request_size_bytes INT COMMENT '请求图片大小（字节）',

    -- 响应信息
    ocr_status VARCHAR(20) COMMENT '识别状态：SUCCESS/FAILED/FALLBACK',
    recognized_text TEXT COMMENT '识别的文本内容',
    confidence_score DECIMAL(3,2) COMMENT '置信度（0.00-1.00）',
    processing_time_ms INT COMMENT '处理耗时（毫秒）',

    -- 错误信息
    error_code VARCHAR(50) COMMENT '错误代码',
    error_message VARCHAR(500) COMMENT '错误信息',

    -- 元数据
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_session (session_id),
    INDEX idx_user (user_id),
    INDEX idx_created (created_at)
) COMMENT 'OCR调用记录表';
```

## 5. 配置文件

### 5.1 application.yml 新增配置

```yaml
# OCR 配置
ocr:
  # 启用 OCR 功能
  enabled: true

  # 默认 OCR 提供商
  default-provider: ZHIPU  # ZHIPU / LOCAL

  # 智谱 OCR 配置
  zhipu:
    api-url: https://open.bigmodel.cn/api/paas/v4/ocr
    api-key: ${ZHIPU_API_KEY}  # 从环境变量读取
    timeout: 30000  # 30秒超时
    max-retries: 1  # 失败重试次数

  # 本地降级配置
  fallback:
    enable-local-model: true  # 启用本地模型降级
    local-model-confidence: 0.6  # 本地模型最低置信度

  # 图片限制
  limits:
    max-image-size: 10485760  # 10MB
    supported-formats: jpg,jpeg,png,pdf
```
