### 3.2 用户提示词（用于方法参数传递）

对于用户提示词，有两种方式：

#### 方式 1：直接在代码中构建（推荐用于简单场景）

```java
private String buildQuestionContextFromPlan(...) {
    return String.format("""
        # 任务
        请根据以下计划生成一个面试问题。

        # 技能: %s
        # 难度: %d
        ...
        """, skillName, difficulty);
}
```

#### 方式 2：从文件加载（推荐用于复杂场景）

**user/interview/ask-question-template.txt**：
```text
# 任务
请根据以下计划生成一个面试问题。

# 技能: {{skill_name}}
# 难度: {{difficulty}}
...
```

**加载方法**：
```java
@Service
public class PromptLoaderService {
    public String loadPrompt(String path) {
        Resource resource = new ClassPathResource("prompts/" + path);
        return StreamUtils.copyToString(
            resource.getInputStream(),
          StandardCharsets.UTF_8
        );
    }

    public String render(String template, Map<String, Object> variables) {
        String result = template;
        for (Map.Entry<String, Object> entry : variables.entrySet()) {
            result = result.replace(
                "{{" + entry.getKey() + "}}",
                entry.getValue().toString()
            );
        }
        return result;
    }
}
```

### 4.2 简化 InterviewAgentFactory

由于 System Message 已经通过 `@SystemMessage(fromResource = ...)` 指定，`InterviewAgentFactory` 不再需要手动构建 System Message：

```java
@Component
@RequiredArgsConstructor
public class InterviewAgentFactory {

    private final OpenAiChatModel chatModel;
    private final SkillTagService skillTagService;
    private final UserResumeService resumeService;
    private final InterviewMemoryProvider memoryProvider;

    private final Map<String, InterviewAgents> agentsCache = new ConcurrentHashMap<>();

    public InterviewAgents createAgents(MockInterviewSession session) {
        String sessionId = session.getSessionId();

        if (agentsCache.containsKey(sessionId)) {
            return agentsCache.get(sessionId);
        }

        // 为每个 Agent 创建独立的 ChatMemory
        ChatMemory mainMemory = memoryProvider.getChatMemory(sessionId, AgentType.MAIN_INTERVIEWER);
        ChatMemory viceMemory = memoryProvider.getChatMemory(sessionId, AgentType.VICE_INTERVIEWER);
        ChatMemory evalMemory = memoryProvider.getChatMemory(sessionId, AgentType.EVALUATOR);

        // ⭐ 获取快照（用于传递给方法参数）
        String resumeSnapshot = session.getResumeSnapshot();
        String jdSnapshot = session.getJdSnapshot();
        String basicContext = buildContext(session);

        // ⚠️ 注意：不再需要 systemMessage() 方法
        // System Message 通过 @SystemMessage(fromResource = ...) 自动加载

        MainInterviewerAgent mainInterviewer = AiServices.builder(MainInterviewerAgent.class)
                .chatModel(chatModel)
                .chatMemory(mainMemory)
                // System Message 已在接口中通过 @SystemMessage 指定
                .build();

        ViceInterviewerAgent viceInterviewer = AiServices.builder(ViceInterviewerAgent.class)
                .chatModel(chatModel)
                .chatMemory(viceMemory)
                .build();

        ExpertEvaluatorAgent evaluator = AiServices.builder(ExpertEvaluatorAgent.class)
                .chatModel(chatModel)
                .chatMemory(evalMemory)
                .build();

        SkillGeneratorAgent skillGenerator = AiServices.builder(SkillGeneratorAgent.class)
                .chatModel(chatModel)
                .build();

        InterviewAgents agents = new InterviewAgents(
                sessionId,
                mainInterviewer,
                viceInterviewer,
                evaluator,
                skillGenerator,
                resumeSnapshot,  // 新增：保存快照供方法调用使用
                jdSnapshot,
                basicContext
        );

        agentsCache.put(sessionId, sessionId);
        log.info("创建面试 Agent 成功（使用文件化提示词），会话ID: {}", sessionId);

        return agents;
    }

    private String buildContext(MockInterviewSession session) {
        // ... 构建基本上下文
    }

    /**
     * Agent 组（包含快照信息）
     */
    public record InterviewAgents(
        String sessionId,
        MainInterviewerAgent mainInterviewer,
        ViceInterviewerAgent viceInterviewer,
        ExpertEvaluatorAgent evaluator,
        SkillGeneratorAgent skillGenerator,
        String resumeSnapshot,      // 新增
        String jdSnapshot,           // 新增
        String basicContext          // 新增
    ) {}
}
```

## 五、对比：LangChain4j 原生 vs 自定义方案

| 特性 | LangChain4j 原生 | 自定义 PromptService |
|-----|-----------------|---------------------|
| 代码量 | 少（直接用注解） | 多（需要实现服务） |
| 维护性 | 高（文件 + 注解） | 中（需要管理服务代码） |
| 灵活性 | 中（受限于 LangChain4j） | 高（完全自定义） |
| 性能 | 好（框架优化） | 需要自己实现缓存 |
| 学习成本 | 低（熟悉注解即可） | 中（需要理解实现） |

### 6.1 系统提示词：使用 LangChain4j 原生

**理由**：
- ✅ 代码简洁
- ✅ 框架原生支持
- ✅ 自动变量替换
- ✅ 不需要自己实现加载逻辑

**实现**：
```java

    @UserMessage String context,
    @V("resume_snapshot") String resumeSnapshot,
    @V("jd_snapshot") String jdSnapshot
);
```

### 6.2 用户提示词：使用简单模板服务

**理由**：
- ✅ 用户提示词变化频繁
- ✅ 需要动态构建
- ✅ 复用性高

**实现**：
```java
@Service
public class UserPromptService {
    private final Map<String, String> cache = new ConcurrentHashMap<>();

    public String loadAndRender(String path, Map<String, Object> variables) {
        String template = cache.computeIfAbsent(path, this::loadFromFile);
        return render(template, variables);
    }
}
```

## 八、注意事项

### 8.1 路径问题
- `fromResource` 使用的是 **classpath 路径**
- 路径必须以 `/` 开头
- 文件必须在 `src/main/resources` 目录下

### 8.2 变量替换
- `@V` 注解的变量名必须与文件中的 `{{name}}` 匹配
- 变量值不能为 null（会报错）
- 需要处理空值情况

### 8.3 编码问题
- 文件必须使用 **UTF-8** 编码
- 注意 BOM（Byte Order Mark）

### 8.4 缓存问题
- LangChain4j 会缓存 System Message
- 修改文件后需要重启应用
