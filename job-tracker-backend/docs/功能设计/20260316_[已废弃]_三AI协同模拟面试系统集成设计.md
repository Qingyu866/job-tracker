### 2.2 三 Agent 职责定义

#### 2.2.1 主面试官 (MainInterviewerAgent)

```java
/**
 * 主面试官 Agent
 * <p>
 * 职责：与候选人进行自然对话，提出问题
 * </p>
 */
public interface MainInterviewerAgent {

    @SystemMessage("""
        # 角色设定
        你是一名资深技术面试官，正在对候选人进行模拟面试。

        # 约束条件
        1. **严禁评分**：不要提及分数、评价或"表现不错"等词汇
        2. **严禁重复**：不要询问已考察过的知识点（见{{covered_topics}}）
        3. **单一任务**：每次回复只提出一个问题，或针对上一个回答进行追问
        4. **上下文约束**：必须基于【下一考察点】({{next_topic}}) 进行提问
        5. **流式友好**：回复长度控制在 200 字以内

        # 当前状态
        - 面试阶段：{{current_phase}}
        - 候选人岗位：{{job_title}}
        - 已考察知识点：{{covered_topics}}
        - 下一考察点：{{next_topic}}

        # 输出要求
        直接输出提问内容，不要包含任何 JSON 标记或前缀解释。
        """)
    String askQuestion(@UserMessage String context);

    /**
     * 流式提问（推荐）
     */
    Flux<String> askQuestionStream(@UserMessage String context);
}
```

### 3.2 面试编排器 (InterviewOrchestrator)

```java
/**
 * 面试编排器
 * <p>
 * 核心职责：
 * 1. 协调三个 Agent 的调用
 * 2. 管理状态机流转
 * 3. 控制流式输出
 * 4. 并行处理评分
 * </p>
 */
@Slf4j
@Service
@RequiredArgsConstructor
public class InterviewOrchestrator {

    private final MainInterviewerAgentFactory mainAgentFactory;
    private final ViceInterviewerAgent viceAgent;
    private final ExpertEvaluatorAgent expertAgent;
    private final InterviewStateStore stateStore;
    private final InterviewTopicRepository topicRepository;
    private final InterviewEvaluationRepository evaluationRepository;

    /**
     * 处理用户回答（核心编排逻辑）
     *
     * @param sessionId 会话 ID
     * @param userAnswer 用户回答
     * @return 响应流（主面试官回复 + 评分更新）
     */
    public Flux<InterviewMessage> handleUserAnswer(String sessionId, String userAnswer) {
        // 1. 获取当前状态
        InterviewState currentState = stateStore.getState(sessionId);

        // 2. 并行触发（CompletableFuture）
        CompletableFuture<EvaluationResult> evalTask = CompletableFuture.supplyAsync(() -> {
            // 异步评分，不阻塞主流程
            return expertAgent.evaluateAsync(sessionId, userAnswer);
        });

        CompletableFuture<NextStepDecision> viceTask = CompletableFuture.supplyAsync(() -> {
            // 副面试官决定下一步
            return viceAgent.decideNextStep(sessionId, userAnswer);
        });

        // 3. 等待副面试官决策
        NextStepDecision decision = viceTask.join();

        // 4. 更新已考察知识点
        stateStore.addCoveredTopic(sessionId, decision.getNextTopic());

        // 5. 驱动主面试官生成新问题（流式）
        MainInterviewerAgent mainAgent = mainAgentFactory.getAgent(sessionId);
        Flux<String> questionStream = mainAgent.askQuestionStream(decision.buildContext());

        // 6. 构建响应流
        return Flux.concat(
            // 先发送状态更新
            Flux.just(InterviewMessage.stateUpdate(decision.getNextTopic())),

            // 再发送问题流
            questionStream.map(InterviewMessage::assistantMessage),

            // 最后发送评分（如果完成）
            Flux.from(evalTask)
                .map(eval -> InterviewMessage.evaluationUpdate(eval))
        );
    }
}
```

### 3.3 题库管理 (InterviewTopicRepository)

```java
/**
 * 面试题库仓储
 * <p>
 * 设计原则：
 * 1. 支持 JD 技能点映射
 * 2. 支持难度分级
 * 3. 支持去重检查
 * </p>
 */
@Slf4j
@Repository
@RequiredArgsConstructor
public class InterviewTopicRepository {

    private final RedisTemplate<String, InterviewTopic> redisTemplate;
    private final InterviewTopicMapper topicMapper;

    private static final String TOPIC_POOL_KEY = "interview:topic:pool:";
    private static final String COVERED_KEY = "interview:covered:";

    /**
     * 根据岗位 JD 获取题库
     */
    public List<InterviewTopic> getTopicsByJobTitle(String jobTitle) {
        String key = TOPIC_POOL_KEY + jobTitle;
        return redisTemplate.opsForList().range(key, 0, -1);
    }

    /**
     * 获取下一个未考察的知识点
     */
    public InterviewTopic getNextTopic(String sessionId, String jobTitle) {
        // 获取所有题库
        List<InterviewTopic> allTopics = getTopicsByJobTitle(jobTitle);

        // 获取已考察列表
        Set<String> covered = getCoveredTopics(sessionId);

        // 过滤未考察的
        return allTopics.stream()
            .filter(topic -> !covered.contains(topic.getId()))
            .findFirst()
            .orElse(null);
    }

    /**
     * 检查问题是否重复
     */
    public boolean isDuplicate(String sessionId, String questionText) {
        // 初期：简单关键词匹配
        Set<String> covered = getCoveredTopics(sessionId);
        return covered.stream()
            .anyMatch(topic -> questionText.contains(topic));
    }

    /**
     * 获取已考察知识点
     */
    public Set<String> getCoveredTopics(String sessionId) {
        String key = COVERED_KEY + sessionId;
        return redisTemplate.opsForSet().members(key);
    }
}
```

2. 对候选人的回答进行自然追问，挖掘技术深度
3. 保持温和、专业的面试风格

# 严格约束
1. **严禁评分**：不要提及分数、等级、表现等词汇
2. **严禁重复**：不要问已考察过的知识点
3. **单一问题**：每次只问一个问题
4. **长度控制**：回复控制在 200 字以内
5. **自然对话**：直接输出问题，不要加前缀

# 示例对话
❌ 错误：
"好的，你的回答不错。下一个问题，请讲讲 HashMap 的底层结构。"

✅ 正确：
"你提到了 HashMap 的数组结构，那它是如何解决哈希冲突的呢？"
```

# 工作流程
1. 提取用户回答中的技术关键词
2. 检查这些关键词是否已考察

```

```
