# 三面试官独立记忆存储设计

**文档日期**: 2026-03-17
**作者**: Job Tracker Team
**版本**: 1.0.0

## 重要说明：sessionId 定义

**⚠️ 本文档中的 `sessionId` 特指「模拟面试系统」的会话ID**

```
sessionId = MockInterviewSession.sessionId
           来自 mock_interview_sessions 表
           标识一次独立的模拟面试过程
```

**与其他系统的区别**：

| 系统 | 表/实体 | ID 字段 | 用途 |
|------|---------|---------|------|
| **模拟面试系统** | `mock_interview_sessions` | `session_id` | 标识一次模拟面试会话，包含三个面试官 Agent |
| **聊天系统** | `chat_sessions` | `session_key` | 标识一次 AI 聊天对话 |
| **WebSocket** | - | - | 实时通信连接 |

**关键理解**：
- 一个用户可以同时进行**多个独立的模拟面试**（例如：针对不同公司）
- 每个模拟面试会话有**自己独立的三位面试官 Agent**
- 不同模拟面试会话的 Agent 记忆**必须完全隔离**（用户 A 的字节跳动面试 ≠ 用户 A 的腾讯面试）

## 一、问题背景

### 1.1 当前架构问题

当前的模拟面试系统中存在两个层面的记忆污染问题：

#### 问题 1：同会话内三个 Agent 共享记忆
```
当前架构（共享记忆）:
┌─────────────────────────────────────────────────────────┐
│  单个模拟面试会话                                         │
│  MockInterviewSession (sessionId: "abc-123")           │
│  ├─ MainInterviewerAgent ───┐                          │
│  ├─ ViceInterviewerAgent ───┼──▶ Shared ChatMemory     │
│  └─ ExpertEvaluatorAgent ───┘                          │
│                                                          │
│  (例如：用户针对"字节跳动-后端工程师"职位的模拟面试)      │
└─────────────────────────────────────────────────────────┘

问题：
❌ 三个 Agent 的"思考过程"混在一起
❌ 主面试官可能看到评审专家的评分逻辑
❌ 副面试官可能看到主面试官的提问策略
```

#### 问题 2：不同模拟面试会话之间记忆串扰
```
当前架构（会话间串扰）:
┌─────────────────────────────────────────────────────────┐
│  全局共享 ChatMemoryProvider                            │
│  ├─ 面试会话 A ("abc-123") ────┐                        │
│  │  └─ 用户: "字节跳动-后端"                            │
│  ├─ 面试会话 B ("xyz-789") ────┼──▶ Shared ChatMemory   │
│  │  └─ 用户: "腾讯-前端"                                │
│  └─ 面试会话 C ("def-456") ────┘                        │
│     └─ 用户: "阿里-Java"                                │
└─────────────────────────────────────────────────────────┘

严重问题：
🔴 会话 A 的面试官可能看到会话 B 的对话历史
🔴 "字节跳动面试"的内容可能泄露到"腾讯面试"
🔴 一个用户无法同时针对多个公司准备面试
🔴 面试历史互相污染，无法准确评估
```

**场景示例**：
```
用户小明同时准备两家公司的面试：
1. 上午：针对"字节跳动"的模拟面试（会话ID: session-bytedance-001）
2. 下午：针对"腾讯"的模拟面试（会话ID: session-tencent-002）

如果记忆不隔离：
❌ 下午的腾讯面试官可能看到上午字节跳动的面试内容
❌ 两家的面试评分会互相干扰
❌ 无法独立分析针对不同公司的面试表现
```

### 1.2 核心问题分析

| 问题 | 描述 | 影响 | 严重性 |
|------|------|------|--------|
| **同会话记忆污染** | 三个 Agent 共享记忆，互相"看见"对方的思考 | AI 决策被干扰，不符合真实面试场景 | 🔴 高 |
| **跨会话记忆串扰** | 不同 sessionId 的会话共享 ChatMemory | 用户隐私泄露，面试内容混淆 | 🔴🔴 严重 |
| **状态管理混乱** | 面试状态存储在 MockInterviewSession 的 JSON 字段中 | 查询困难，无法追踪状态变化历史 | 🟡 中 |
| **无法独立追踪** | 无法单独分析某个 Agent 的决策逻辑 | 调试困难，无法评估各 Agent 表现 | 🟡 中 |
| **扩展性差** | 新增 Agent 需要修改共享逻辑 | 违反开闭原则 | 🟡 中 |

#### 2.1.2 隔离维度说明

| 隔离维度 | 隔离机制 | 技术实现 |
|----------|----------|----------|
| **模拟面试会话间隔离** | 不同 MockInterviewSession 的记忆完全独立 | 1. 数据库：`session_id` UNIQUE 约束<br>2. 内存：`ConcurrentHashMap<String, SessionMemories>`<br>   - Key = `MockInterviewSession.sessionId`<br>3. ChatMemory：每个会话创建独立实例 |
| **Agent 间隔离** | 同一模拟面试会话内三个 Agent 记忆独立 | 1. 数据库：三个独立的 JSON 字段<br>2. 内存：三个独立的 ChatMemory 对象<br>3. 持久化：独立加载和保存 |

**实际场景示例**：
```
用户小明的面试记录：
├─ 面试会话 1: session-byt-001 (字节跳动-后端)
│  ├─ MainInterviewer Memory (只包含字节跳动的提问历史)
│  ├─ ViceInterviewer Memory (只包含字节跳动的选题决策)
│  └─ Evaluator Memory (只包含字节跳动的评分记录)
│
└─ 面试会话 2: session-tx-002 (腾讯-前端)
   ├─ MainInterviewer Memory (只包含腾讯的提问历史)
   ├─ ViceInterviewer Memory (只包含腾讯的选题决策)
   └─ Evaluator Memory (只包含腾讯的评分记录)

两个会话的记忆完全独立，互不干扰。
```

### 2.2 数据库设计

#### 2.2.1 新增表：`interview_memories`

```sql
-- =====================================================
-- 面试记忆存储表
-- =====================================================
CREATE TABLE interview_memories (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '记忆ID',
    session_id VARCHAR(64) NOT NULL UNIQUE COMMENT '面试会话ID',

    -- 三个 Agent 的独立记忆（JSON 存储）
    main_interviewer_memory JSON COMMENT '主面试官记忆：提问历史、用户回答',
    vice_interviewer_memory JSON COMMENT '副面试官记忆：选题决策、考察进度',
    evaluator_memory JSON COMMENT '评审专家记忆：评分历史、真实性分析',

    -- 元数据
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    INDEX idx_session_id (session_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
COMMENT='面试 Agent 独立记忆存储';
```

#### 2.3.2 SessionMemories 数据结构

```java
package com.jobtracker.agent.memory;

import dev.langchain4j.memory.ChatMemory;
import lombok.Data;

/**
 * 单个会话的三个 Agent 记忆容器
 * <p>
 * 确保同一会话内的三个 Agent 记忆互不干扰
 * </p>
 */
@Data
public class SessionMemories {

    /**
     * 会话ID
     */
    private final String sessionId;

    /**
     * 主面试官的独立 ChatMemory
     */
    private ChatMemory mainInterviewerMemory;

    /**
     * 副面试官的独立 ChatMemory
     */
    private ChatMemory viceInterviewerMemory;

    /**
     * 评审专家的独立 ChatMemory
     */
    private ChatMemory evaluatorMemory;

    public SessionMemories(String sessionId) {
        this.sessionId = sessionId;
    }

    /**
     * 根据 AgentType 获取对应的 ChatMemory
     */
    public ChatMemory getMemory(AgentType agentType) {
        return switch (agentType) {
            case MAIN_INTERVIEWER -> mainInterviewerMemory;
            case VICE_INTERVIEWER -> viceInterviewerMemory;
            case EVALUATOR -> evaluatorMemory;
        };
    }

    /**
     * 设置对应 AgentType 的 ChatMemory
     */
    public void setMemory(AgentType agentType, ChatMemory memory) {
        switch (agentType) {
            case MAIN_INTERVIEWER -> mainInterviewerMemory = memory;
            case VICE_INTERVIEWER -> viceInterviewerMemory = memory;
            case EVALUATOR -> evaluatorMemory = memory;
        }
    }

    /**
     * 检查是否所有 Agent 的记忆都已创建
     */
    public boolean isComplete() {
        return mainInterviewerMemory != null
                && viceInterviewerMemory != null
                && evaluatorMemory != null;
    }
}
```

#### 2.3.3 InterviewMemoryProvider（核心）

```java
package com.jobtracker.agent.memory;

import com.fasterxml.jackson.core.JsonProcessingException;
import com.fasterxml.jackson.databind.ObjectMapper;
import com.jobtracker.entity.InterviewMemory;
import com.jobtracker.mapper.InterviewMemoryMapper;
import dev.langchain4j.data.message.ChatMessage;
import dev.langchain4j.data.message.AiMessage;
import dev.langchain4j.data.message.UserMessage;
import dev.langchain4j.memory.ChatMemory;
import dev.langchain4j.memory.chat.MessageWindowChatMemory;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Component;

import java.util.ArrayList;
import java.util.List;
import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;

/**
 * 面试记忆提供者
 * <p>
 * 实现两层隔离：
 * 1. 会话间隔离：不同 sessionId 的记忆完全独立
 * 2. Agent 间隔离：同一会话内三个 Agent 的记忆完全独立
 * </p>
 *
 * <p>隔离机制：</p>
 * <ul>
 *   <li>内存层：Map&lt;sessionId, SessionMemories&gt; 确保会话隔离</li>
 *   <li>Agent层：SessionMemories 内部包含三个独立的 ChatMemory</li>
 *   <li>持久化层：数据库中 session_id UNIQUE 约束确保物理隔离</li>
 * </ul>
 */
@Slf4j
@Component
@RequiredArgsConstructor
public class InterviewMemoryProvider {

    private final InterviewMemoryMapper memoryMapper;
    private final ObjectMapper objectMapper;

    /**
     * L1 内存缓存：sessionId → SessionMemories
     * <p>
     * 确保不同会话的记忆在内存中完全隔离
     * </p>
     */
    private final Map<String, SessionMemories> memoryCache = new ConcurrentHashMap<>();

    /**
     * 获取指定会话和 Agent 类型的 ChatMemory
     * <p>
     * 关键方法：确保 (sessionId, agentType) 组合有独立的 ChatMemory 实例
     * </p>
     *
     * @param sessionId 会话ID（会话隔离键）
     * @param agentType Agent类型（Agent隔离键）
     * @return 独立的 ChatMemory 实例
     */
    public ChatMemory getChatMemory(String sessionId, AgentType agentType) {
        // 1. 获取或创建会话的 SessionMemories（会话隔离）
        SessionMemories sessionMemories = memoryCache.computeIfAbsent(sessionId, id -> {
            log.info("创建新会话记忆容器，会话ID: {}", id);
            return loadOrCreateSessionMemories(id);
        });

        // 2. 获取或创建指定 Agent 的 ChatMemory（Agent 隔离）
        ChatMemory memory = sessionMemories.getMemory(agentType);
        if (memory == null) {
            memory = createChatMemory(sessionId, agentType);
            sessionMemories.setMemory(agentType, memory);
            log.info("创建 {} ChatMemory，会话ID: {}", agentType.getDisplayName(), sessionId);
        }

        return memory;
    }

    /**
     * 从数据库加载或创建新的会话记忆
     */
    private SessionMemories loadOrCreateSessionMemories(String sessionId) {
        // 先从数据库加载
        InterviewMemory dbMemory = memoryMapper.selectBySessionId(sessionId);

        if (dbMemory == null) {
            // 不存在则创建新记录
            dbMemory = new InterviewMemory();
            dbMemory.setSessionId(sessionId);
            memoryMapper.insert(dbMemory);
            log.info("创建数据库记忆记录，会话ID: {}", sessionId);
        } else {
            log.info("从数据库加载记忆记录，会话ID: {}", sessionId);
        }

        // 创建 SessionMemories 容器
        SessionMemories sessionMemories = new SessionMemories(sessionId);

        // 从数据库恢复三个 Agent 的 ChatMemory
        sessionMemories.setMainInterviewerMemory(
                restoreChatMemory(dbMemory.getMainInterviewerMemory())
        );
        sessionMemories.setViceInterviewerMemory(
                restoreChatMemory(dbMemory.getViceInterviewerMemory())
        );
        sessionMemories.setEvaluatorMemory(
                restoreChatMemory(dbMemory.getEvaluatorMemory())
        );

        return sessionMemories;
    }

    /**
     * 创建新的 ChatMemory 实例
     * <p>
     * 每个调用都返回全新的实例，确保完全隔离
     * </p>
     */
    private ChatMemory createChatMemory(String sessionId, AgentType agentType) {
        return MessageWindowChatMemory.builder()
                .maxMessages(50)  // 保留最近 50 条消息
                .id(sessionId + ":" + agentType.getCode())  // 唯一 ID
                .build();
    }

    /**
     * 从 JSON 恢复 ChatMemory
     */
    private ChatMemory restoreChatMemory(String json) {
        if (json == null || json.isBlank()) {
            return null;
        }

        try {
            // 解析 JSON，恢复消息列表
            List<MessageDto> messages = objectMapper.readValue(
                    json,
                    objectMapper.getTypeFactory().constructCollectionType(List.class, MessageDto.class)
            );

            // 构建新的 ChatMemory
            ChatMemory memory = MessageWindowChatMemory.builder()
                    .maxMessages(50)
                    .build();

            for (MessageDto msg : messages) {
                if ("user".equals(msg.role)) {
                    memory.add(UserMessage.from(msg.content));
                } else if ("ai".equals(msg.role)) {
                    memory.add(AiMessage.from(msg.content));
                }
            }

            return memory;
        } catch (JsonProcessingException e) {
            log.error("恢复 ChatMemory 失败", e);
            return null;
        }
    }

    /**
     * 持久化指定 Agent 的记忆到数据库
     */
    public void persistMemory(String sessionId, AgentType agentType) {
        SessionMemories sessionMemories = memoryCache.get(sessionId);
        if (sessionMemories == null) {
            log.warn("会话记忆不存在，无法持久化: {}", sessionId);
            return;
        }

        ChatMemory memory = sessionMemories.getMemory(agentType);
        if (memory == null) {
            log.warn("{} 记忆不存在，无法持久化: {}", agentType.getDisplayName(), sessionId);
            return;
        }

        // 转换为 JSON
        String json = convertToJson(memory);

        // 更新数据库
        InterviewMemory dbMemory = memoryMapper.selectBySessionId(sessionId);
        if (dbMemory != null) {
            switch (agentType) {
                case MAIN_INTERVIEWER -> dbMemory.setMainInterviewerMemory(json);
                case VICE_INTERVIEWER -> dbMemory.setViceInterviewerMemory(json);
                case EVALUATOR -> dbMemory.setEvaluatorMemory(json);
            }
            memoryMapper.updateById(dbMemory);
            log.debug("持久化 {} 记忆成功，会话ID: {}", agentType.getDisplayName(), sessionId);
        }
    }

    /**
     * 持久化会话的所有 Agent 记忆
     */
    public void persistAll(String sessionId) {
        persistMemory(sessionId, AgentType.MAIN_INTERVIEWER);
        persistMemory(sessionId, AgentType.VICE_INTERVIEWER);
        persistMemory(sessionId, AgentType.EVALUATOR);
        log.info("持久化会话所有记忆成功，会话ID: {}", sessionId);
    }

    /**
     * 将 ChatMemory 转换为 JSON
     */
    private String convertToJson(ChatMemory memory) {
        try {
            List<MessageDto> messages = new ArrayList<>();
            for (ChatMessage msg : memory.messages()) {
                if (msg instanceof UserMessage) {
                    messages.add(new MessageDto("user", ((UserMessage) msg).singleText()));
                } else if (msg instanceof AiMessage) {
                    messages.add(new MessageDto("ai", ((AiMessage) msg).text()));
                }
            }
            return objectMapper.writeValueAsString(messages);
        } catch (JsonProcessingException e) {
            log.error("转换 ChatMemory 为 JSON 失败", e);
            return "[]";
        }
    }

    /**
     * 清除会话记忆（从内存缓存中移除）
     * <p>
     * 注意：此操作仅从内存中移除，数据库记录保留
     * </p>
     */
    public void clearMemory(String sessionId) {
        memoryCache.remove(sessionId);
        log.info("清除会话记忆缓存，会话ID: {}", sessionId);
    }

    /**
     * 完全删除会话记忆（包括数据库记录）
     */
    public void deleteMemory(String sessionId) {
        memoryCache.remove(sessionId);
        memoryMapper.deleteBySessionId(sessionId);
        log.info("删除会话记忆（包括数据库），会话ID: {}", sessionId);
    }

    /**
     * 消息 DTO（用于 JSON 序列化）
     */
    @lombok.Data
    @lombok.AllArgsConstructor
    private static class MessageDto {
        private String role;
        private String content;
    }
}
```

### 9.2 关键类职责

| 类 | 职责 | 隔离机制 |
|---|------|----------|
| `AgentType` | 定义三种面试官类型 | 枚举值：MAIN_INTERVIEWER, VICE_INTERVIEWER, EVALUATOR |
| `SessionMemories` | 单会话的三个 Agent 记忆容器 | 内部包含三个独立的 `ChatMemory` 引用 |
| `InterviewMemoryProvider` | 记忆管理核心 | `Map<String, SessionMemories>` 实现会话隔离 |
| `InterviewMemory` | 数据库实体 | `session_id UNIQUE` 约束实现物理隔离 |
| `InterviewAgentFactory` | 创建三个 Agent | 为每个 Agent 分配独立的 `ChatMemory` |
