### InterviewState 枚举详解

| 枚举值 | 中文 | 触发时机 | 下一步 | 说明 |
|-------|------|---------|--------|------|
| INIT | 初始化 | 调用 createSession() | 用户点击"开始面试" | 会话刚创建，Agent 组已初始化 |
| WELCOME | 欢迎环节 | 用户点击"开始面试" | 进入 TECHNICAL_QA | 面试官自我介绍，说明流程 |
| TECHNICAL_QA | 技术问答 | 用户回复欢迎消息 | 达到轮次或结束 | 主要面试阶段，三Agent协同工作 |
| HR_QA | HR问答 | 技术问答结束 | 进入 FINISHED | 询问职业规划、薪资等（暂未使用） |
| GENERATING_REPORT | 生成报告 | 用户点击"结束面试" | 进入 FINISHED | 三Agent协同生成完整报告 |
| FINISHED | 已完成 | 报告生成完成 | 无 | 用户可查看完整报告 |

| sessionId | String | 会话ID | - | "abc-123-def" |

| totalScore | BigDecimal | 总分 | 0-10 | 9.0 |

### 评分维度详解

| 维度 | 满分 | 评分标准 | 说明 |
|-----|------|---------|------|
| technicalScore | 4.0 | • 概念是否正确<br>• 有无明显错误<br>• 与简历声称的匹配度 | 技术准确性是最基础的维度 |
| logicScore | 3.0 | • 表达是否条理清晰<br>• 是否符合 STAR 法则<br>• 逻辑是否连贯 | 逻辑清晰度反映表达能力 |
| depthScore | 3.0 | • 是否触及底层原理<br>• 与工作年限的匹配度<br>• 知识广度 | 深度与广度反映真实水平 |
| **totalScore** | **10.0** | **technicalScore + logicScore + depthScore** | **总分 = 各维度之和** |

### 申请状态流转

```
WISHLIST（意向）
    ↓
APPLIED（已申请）
    ↓
SCREENING（筛选中）
    ↓
INTERVIEW（面试中）
    ↓
FINAL_ROUND（终面）
    ↓
OFFERED（已发Offer）
    ↓
ACCEPTED（已接受） / DECLINED（已拒绝） / EXPIRED（已过期）

任何状态都可转到：
- REJECTED（被拒绝）
- WITHDRAWN（已撤回）
```

| FAILED | 未通过 | 面试未通过 |
| CANCELLED | 已取消 | 面试被取消 |
| NO_SHOW | 缺席 | 候选人未参加 |

# 四、总结

## 枚举值速查表

### 面试状态相关
- **InterviewState**: INIT → WELCOME → TECHNICAL_QA → HR_QA → GENERATING_REPORT → FINISHED
- **MessageRole**: ASSISTANT（AI） / USER（用户）
- **InterviewType**: PHONE / VIDEO / ONSITE / TECHNICAL / HR
- **InterviewStatus**: SCHEDULED / IN_PROGRESS / COMPLETED / PASSED / FAILED / CANCELLED / NO_SHOW

### 简历与技能相关
- **TargetLevel**: JUNIOR（1-2年） / MIDDLE（3-5年） / SENIOR（5-8年） / LEAD（8年以上）
- **ParseStatus**: PENDING → PARSING → SUCCESS / FAILED
- **ProficiencyLevel**: BEGINNER（了解） / INTERMEDIATE（熟悉） / ADVANCED（精通） / EXPERT（权威）
- **Category**: LANGUAGE / FRAMEWORK / DATABASE / TOOL / ALGORITHM

### 评分相关
- **technicalScore**: 0-4分（技术准确性）
- **logicScore**: 0-3分（逻辑清晰度）
- **depthScore**: 0-3分（深度与广度）
- **totalScore**: 0-10分（总分 = 三个维度之和）

## 关键设计要点

1. **快照机制**：resumeSnapshot、jdSnapshot、skillsSnapshot 确保历史记录不受后续更新影响
2. **三层隔离记忆**：mainInterviewerMemory、viceInterviewerMemory、evaluatorMemory 完全独立
3. **状态机设计**：InterviewState 严格遵循状态流转规则
4. **枚举驱动**：大量使用枚举确保数据一致性
5. **逻辑删除**：deleted 字段实现软删除，保留历史数据
