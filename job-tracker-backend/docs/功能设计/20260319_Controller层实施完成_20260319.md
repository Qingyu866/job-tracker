
**具体改动**：
1. ✅ 删除副面试官动态决策逻辑（decideNextStep）

4. ✅ 添加新方法 `buildQuestionContextFromPlan()` - 根据计划生成问题上下文

## API 接口变更总结

### 修改的接口

#### POST /api/mock-interview/sessions/{sessionId}/message
**变更**：从动态决策改为按计划执行

**请求示例**：
```json
{
  "content": "用户回答内容",
  "roundNumber": 1
}
```

**响应**：
- 如果有下一个计划：返回生成的面试官问题
- 如果没有更多计划：返回错误提示"所有问题已完成，请点击结束面试按钮查看报告"

## 流程图

### 创建会话流程
```
用户创建会话
  ↓
createSession() - 生成快照
  ↓
createAgents() - 初始化 Agent
  ↓
startInterview() - 更新状态
  ↓
generateAndSaveQuestionPlans() - 生成 25 轮计划 ✨ 新增
  ↓
返回会话信息
```

### 发送消息流程
```
用户发送回答
  ↓
保存用户消息
  ↓
评估上一轮回答
  ↓
更新上一轮计划状态为 COMPLETED ✨ 新增
  ↓
获取下一个 PENDING 计划 ✨ 新增
  ↓
如果没有计划 → 返回提示 ✨ 修改
  ↓
如果有计划：
  - 更新状态为 IN_PROGRESS ✨ 新增
  - 更新会话的 currentPlanId ✨ 新增
  - 根据计划生成问题 ✨ 修改
  ↓
保存面试官问题
  ↓
进入下一轮
```

### 暂停/恢复流程
```
暂停：
  - 更新会话状态为 PAUSED
  - 更新 IN_PROGRESS 计划为 PAUSED
  - 持久化 Agent 记忆

恢复：
  - 恢复 Agent 记忆
  - 更新 PAUSED 计划为 PENDING
  - 更新会话状态为 TECHNICAL_QA
```

## 技术要点

### 1. LangChain4j 自动序列化
```java
// Service 层
List<QuestionPlanDTO> plans = agents.viceInterviewer().generateQuestionPlan(context);
// LangChain4j 自动将 AI JSON 返回反序列化为 List<QuestionPlanDTO>

// 直接保存，无需手动解析
for (QuestionPlanDTO plan : plans) {
    MockInterviewEvaluation evaluation = MockInterviewEvaluation.builder()...build();
    evaluationMapper.insert(evaluation);
}
```

### 2. 计划状态管理
```
PENDING → IN_PROGRESS → COMPLETED
    ↓
  PAUSED (暂停时)
```

### 3. 主键字段名
- ⚠️ `MockInterviewEvaluation.evaluationId` (不是 `id`)
- 使用 Lombok 的 `@Data` 自动生成 getter

```javascript

  method: 'POST',

});

if (result.code !== 200) {
  // 没有更多问题，显示"结束面试"按钮
  showFinishButton();
} else {
  // 显示面试官问题
  displayMessage(result.data);
}
```
