# 三面试官 Agent 职责边界与调用时机梳理

## 文档概述

**创建日期**: 2026-03-19
**目的**: 明确三个 Agent 的职责边界、调用时机、数据流转
**相关文件**:
- `MainInterviewerAgent.java` - 主面试官
- `ViceInterviewerAgent.java` - 副面试官
- `ExpertEvaluatorAgent.java` - 评审专家
- `MockInterviewController.java` - 调用流程

## 一、三个 Agent 的角色定位

### 1. MainInterviewerAgent（主面试官）

**角色定位**: 面试的主导者，负责与候选人直接对话

**核心职责**:
1. **提问**: 根据副面试官的决策，生成具体的问题
2. **欢迎**: 在面试开始时，进行欢迎和开场
3. **总结**: 在面试结束时，对整体表现进行总结

**不应该做的事**:
- ❌ 决定下一个话题（这是副面试官的职责）
- ❌ 评估候选人回答（这是评审专家的职责）
- ❌ 分析简历差距（这是评审专家的职责）

**能力边界**:
- ✅ 擅长：基于具体话题生成自然、专业的问题
- ✅ 擅长：根据候选人的回答进行追问
- ✅ 擅长：营造良好的面试氛围

---

### 2. ViceInterviewerAgent（副面试官）

**角色定位**: 面试的大脑，负责选题和流程控制

**核心职责**:
1. **选题**: 基于简历、JD、已考察内容，决定下一个话题
2. **去重**: 避免重复考察同一个知识点
3. **流程控制**: 决定是继续提问、结束面试、还是切换到 HR 问题
4. **改进建议**: 在面试结束时，提供针对性的改进建议

**不应该做的事**:
- ❌ 直接向候选人提问（这是主面试官的职责）
- ❌ 评估回答质量（这是评审专家的职责）
- ❌ 生成总结（这是主面试官的职责）

**能力边界**:
- ✅ 擅长：策略性思考，平衡 JD 要求 vs 简历内容
- ✅ 擅长：识别候选人回答中的线索，决定下一步
- ✅ 擅长：判断面试进度，控制节奏

---

### 3. ExpertEvaluatorAgent（评审专家）

**角色定位**: 面试的观察者，负责评估和分析

**核心职责**:
1. **评估**: 评估候选人的回答质量（技术准确性、逻辑清晰度、深度）
2. **差距分析**: 对比简历声称与实际表现，识别夸大成分
3. **可信度评分**: 计算简历可信度评分（0-1）
4. **技能分析**: 生成每个技能的可信度分析报告

**不应该做的事**:
- ❌ 决定下一个话题（这是副面试官的职责）
- ❌ 直接提问（这是主面试官的职责）
- ❌ 提供改进建议（这是副面试官的职责）

**能力边界**:
- ✅ 擅长：客观评估回答质量
- ✅ 擅长：识别技术深度和广度
- ✅ 擅长：对比简历声称与实际表现

## 三、当前实现的问题分析

### 问题 1：调用顺序混乱

**当前流程**：
```
1. 评审专家评估 (evaluate)
2. 副面试官决策 (decideNextStep)
3. 主面试官提问 (askQuestion)
```

**问题**：
- ❌ 副面试官决策时，还没有看到评审专家的评估结果
- ❌ 副面试官无法基于评估结果调整选题策略
- ❌ 如果评审专家发现候选人在某个技能上表现很差，副面试官应该立即切换话题

**正确的顺序应该是**：
```
方案 A（并行处理）：
1. 【评审专家】评估用户回答 (evaluate)
2. 【副面试官】基于评估结果决定下一步 (decideNextStep)
3. 【主面试官】生成问题 (askQuestion)

方案 B（顺序处理，当前实现）：
需要修改：副面试官的决策上下文应该包含评估结果
```

---

### 问题 2：副面试官的决策缺少评估结果

**当前实现**：
```java
// 副面试官决定下一步
String nextStepContext = buildNextStepContext(session, request.getContent());
// ❌ 问题：只包含用户回答，不包含评估结果
```

**应该改为**：
```java
// 副面试官决定下一步（应该包含评估结果）
String nextStepContext = buildNextStepContextWithEvaluation(
    session,
    request.getContent(),
    evaluationResult  // ← 新增：包含上一轮的评估结果
);
```

**副面试官需要知道**：
- 候选人上一轮回答的评分
- 哪些技能已经验证过，表现如何
- 是否需要立即切换话题（连续2次"不知道"）

---

### 问题 3：评审专家评估时机过晚

**当前实现**：
```java
// 在主面试官提问之前评估
agents.evaluator().evaluate(evaluationContext);
```

**问题**：
- ✅ 时机是对的（在决策之前评估）
- ❌ 但评估结果没有传递给副面试官
- ❌ 副面试官无法基于评估结果做决策

---

### 问题 4：三个 Agent 的记忆没有共享

**当前实现**：
```java
// 三个 Agent 使用独立的 ChatMemory
ChatMemory mainMemory = memoryProvider.getChatMemory(sessionId, AgentType.MAIN_INTERVIEWER);
ChatMemory viceMemory = memoryProvider.getChatMemory(sessionId, AgentType.VICE_INTERVIEWER);
ChatMemory evalMemory = memoryProvider.getChatMemory(sessionId, AgentType.EVALUATOR);
```

**问题**：
- ✅ 记忆隔离是对的（避免互相干扰）
- ❌ 但缺少跨 Agent 的信息共享机制
- ❌ 副面试官不知道评审专家的评估结果

**解决方案**：
1. 保持记忆隔离（每个 Agent 有自己的对话历史）
2. 但在上下文中共享关键信息（评估结果、决策结果）

---

### 问题 5：评估结果没有反馈给副面试官

**当前流程**：
```
评审专家评估 → 保存到数据库
                      ↓
              副面试官决策（看不到评估结果）
```

**应该的流程**：
```
评审专家评估 → 保存到数据库
      ↓
      └─► 传递给副面试官 → 副面试官决策
```

## 四、优化方案设计

### 方案 A：在上下文中传递评估结果（推荐）

**修改位置**：`MockInterviewController.sendMessage()`

**修改前**：
```java
// 1. 评估用户回答
EvaluationResult evaluationResult = agents.evaluator().evaluate(context);

// 2. 保存评分（但不传递给副面试官）
createEvaluationFromResult(...);

// 3. 副面试官决策（看不到评估结果）
NextStepDecision nextStepDecision = agents.viceInterviewer().decideNextStep(context);
```

**修改后**：
```java
// 1. 评估用户回答
EvaluationResult evaluationResult = agents.evaluator().evaluate(context);

// 2. 保存评分
createEvaluationFromResult(...);

// 3. 副面试官决策（包含评估结果）
NextStepDecision nextStepDecision = agents.viceInterviewer().decideNextStep(
    buildNextStepContextWithEvaluation(session, request.getContent(), evaluationResult)
);
```

**新增方法**：
```java
/**
 * 构建下一步决策上下文（包含评估结果）
 */
private String buildNextStepContextWithEvaluation(
    MockInterviewSession session,
    String userAnswer,
    EvaluationResult lastEvaluation  // ← 新增：上一轮的评估结果
) {
    return String.format("""
        用户最新回答: %s

        # 上一轮评估结果
        - 总分: %.1f/10
        - 技术准确性: %.1f/4
        - 逻辑清晰度: %.1f/3
        - 深度与广度: %.1f/3
        - 反馈: %s

        # 已考察知识点
        %s

        # 待考察知识点
        %s

        # 最近 5 轮对话历史
        %s

        # 统计信息
        - 当前轮次: %d
        """,
        userAnswer,
        lastEvaluation.getTotalScore(),
        lastEvaluation.getScores().getTechnical(),
        lastEvaluation.getScores().getLogic(),
        lastEvaluation.getScores().getDepth(),
        lastEvaluation.getFeedback(),
        session.getSkillsCovered(),
        session.getSkillsPending(),
        getRecentConversationHistory(sessionId),
        session.getCurrentRound()
    );
}
```

## 六、职责边界总结

### 清晰的职责划分

| Agent | 主要职责 | 输入 | 输出 | 时机 |
|-------|---------|------|------|------|
| **评审专家** | 评估回答质量 | 问题、回答、简历、JD | EvaluationResult | 每轮对话后 |
| **副面试官** | 决定下一步 | 用户回答、评估结果、已考察知识点 | NextStepDecision | 每轮对话后 |
| **主面试官** | 生成问题 | NextStepDecision、简历、JD | String (问题) | 每轮对话后 |

### 面试结束时的职责

| Agent | 主要职责 | 输入 | 输出 | 时机 |
|-------|---------|------|------|------|
| **评审专家** | 可信度分析 | 完整面试上下文、所有评分 | List&lt;SkillCredibility&gt; | 面试结束时 |
| **评审专家** | 总体可信度评分 | 完整面试上下文、技能分析 | CredibilityScoreResult | 面试结束时 |
| **副面试官** | 改进建议 | 完整面试上下文、所有评分 | List&lt;ImprovementSuggestion&gt; | 面试结束时 |
| **主面试官** | 面试总结 | 完整面试上下文、所有评分 | String (总结) | 面试结束时 |

## 七、需要修改的代码
