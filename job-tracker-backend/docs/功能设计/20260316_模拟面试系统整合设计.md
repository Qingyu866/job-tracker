## 1. 设计目标

### 1.1 核心场景
```
用户添加求职申请（如：字节跳动 Java 后端工程师）
         ↓
上传/录入简历 → 系统解析简历（项目经历、技能、工作年限）
         ↓
系统匹配：岗位 JD + 用户简历
         ↓
用户发起"模拟面试"，系统针对该岗位 + 简历进行特训
         ↓
面试官基于简历提问：
  - "看到你简历上写了电商秒杀项目，能详细讲讲高并发怎么处理的吗？"
  - "你简历上写着精通 Redis，那 Redis 的持久化机制了解吗？"
         ↓
评分时对比：简历中的声称技能 vs 实际回答水平
         ↓
生成评估报告，指出简历与实际水平的差距，指导改进
```

### 1.2 设计原则
1. **双轮驱动**：面试问题基于**岗位 JD + 用户简历**
2. **深度挖掘**：针对简历中的项目经历和声称技能深入提问
3. **真实模拟**：还原真实面试中"根据简历提问"的场景
4. **差距分析**：对比简历声称和实际表现，指出"水分"

### 1.3 核心数据流

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│  岗位 JD     │  +   │  用户简历    │  =   │  面试问题    │
│             │      │             │      │             │
│ - 技能要求   │      │ - 项目经历   │      │ - 项目深挖   │
│ - 岗位级别   │      │ - 工作年限   │      │ - 技能验证   │
│ - 职责描述   │      │ - 声称技能   │      │ - 简历质疑   │
└─────────────┘      └─────────────┘      └─────────────┘
```

### 1.4 面试官提问策略

| 简历内容 | 提问示例 | 目的 |
|---------|---------|------|
| 项目经历 | "看到你做了电商秒杀项目，QPS 是多少？怎么解决的？" | 验证项目真实性 |
| 声称技能 | "你简历上写着精通 Redis，那 Redis 集群模式了解吗？" | 验证技能深度 |
| 技术栈 | "你项目中用了 Kafka，为什么选它而不是 RocketMQ？" | 考察技术选型能力 |
| 工作年限 | "3 年后端经验，那你说说 JVM 调优的实际案例？" | 评估经验匹配度 |

## 2. 核心架构与流程（开发参考）

### 2.1 面试流程三步走

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 副面试官决定：下一个考察哪个知识点？                       │
│                                                             │
│    待考察知识点：[Java, HashMap, Redis, Spring Boot, Kafka] │
│    已考察：[ArrayList, LinkedList]                          │
│                                                             │
│    决策逻辑：                                                │
│    - 优先深挖简历中的项目                                    │
│    - 其次验证简历声称的技能                                  │
│    - 最后覆盖 JD 要求但简历未提的技能                         │
│                                                             │
│    决策：下一个考察 "HashMap"                                │
└─────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 主面试官 Agent 动态生成问题（不是查询数据库）               │
│                                                             │
│    输入：                                                    │
│    - 知识点：HashMap                                        │
│    - JD 要求：掌握集合框架                                  │
│    - 简历内容：3 年经验，做过电商项目                         │
│    - 已考察：ArrayList、LinkedList                          │
│                                                             │
│    AI 生成问题（动态）：                                      │
│    "看你简历 3 年 Java 经验，集合应该很熟悉。               │
│     HashMap 在高并发场景下会有什么问题？如何解决？"          │
│                                                             │
│    注意：题目不是从数据库查的，是 AI 现场生成的！            │
└─────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. 评审专家 Agent 评分 + 差距分析                            │
│                                                             │
│    分析维度：                                                │
│    - 技术准确性（4分）                                       │
│    - 逻辑清晰度（3分）                                       │
│    - 深度与广度（3分）                                       │
│                                                             │
│    重点：简历真实性分析                                      │
│    - 简历声称："精通 Redis"                                 │
│    - 实际表现：只了解基础                                    │
│    - 差距评估："部分夸大"（exaggeration_score: 0.7）       │
└─────────────────────────────────────────────────────────────┘
```

### 2.5 为什么需要技能快照？

```
┌─────────────────────────────────────────────────────────────┐
│ 问题场景：skill_tags 表更新后，历史面试记录会受影响吗？        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│ 某天，管理员更新了 skill_tags 表：                          │
│   - 将 "HashMap" 改名为 "哈希表"                               │
│   - 将 "Redis" 的难度从 3 改为 2                              │
│                                                             │
│ 如果没有快照：                                                │
│   ❌ 历史面试记录中的 skill_name 可能显示不一致              │
│   ❌ 历史报告中的技能名称可能混乱                              │
│   ❌ 用户查看历史面试时感到困惑                               │
│                                                             │
│ 如果有快照：                                                  │
│   ✅ 历史面试记录使用快照数据                                │
│   ✅ skills_snapshot 保存了当时的技能名称："HashMap"          │
│   ✅ 即使 skill_tags 表改了名，历史记录仍然显示旧名称            │
└─────────────────────────────────────────────────────────────┘
```

#### skills_snapshot 字段示例

```json
{
  "skills_snapshot": {
    "1": {"skill_id": 1, "skill_name": "Java", "category": "LANGUAGE"},
    "1001": {"skill_id": 1001, "skill_name": "HashMap", "category": "ALGORITHM"},
    "3": {"skill_id": 3, "skill_name": "Redis", "category": "DATABASE"}
  }
}
```

**保存时机**：面试会话创建时（`startInterview` 方法中）

**作用**：
1. 存档当时技能的名称、分类等信息
2. 历史面试报告始终使用快照数据
3. 避免 `skill_tags` 表更新影响历史记录

### 2.2 用户简历管理（新增）

**重要说明**：模拟面试只关注技术能力，不需要隐私信息。

#### 简历表 (user_resumes) - 极简版

```sql
CREATE TABLE user_resumes (
    resume_id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '简历ID',
    user_id BIGINT NOT NULL COMMENT '用户ID',
    resume_name VARCHAR(100) COMMENT '简历名称（如"Java后端-3年经验"）',
    is_default BOOLEAN DEFAULT FALSE COMMENT '是否为默认简历',

    -- ⚠️ 模拟面试只需要这些：

    -- 工作信息（影响面试问题难度）
    work_years INT COMMENT '工作年限（年）- 影响：3年问JVM，1年问基础',
    current_position VARCHAR(100) COMMENT '当前职位 - 如：Java后端工程师',
    target_level VARCHAR(20) DEFAULT 'MIDDLE' COMMENT '目标岗位级别：JUNIOR/MIDDLE/SENIOR',

    -- ⚠️ 不需要：
    -- ❌ 真实姓名
    -- ❌ 手机号、邮箱
    -- ❌ 地址
    -- ❌ 学校、学历、专业（面试不问这些）
    -- ❌ 期望薪资（模拟不谈钱）

    -- 自我介绍（AI 用于了解技术背景）
    summary TEXT COMMENT '自我介绍（突出技术栈、项目经验，如"3年Java后端，熟悉Spring Cloud..."）',

    -- 原始文件（用于 AI 解析）
    original_file_url VARCHAR(255) COMMENT '原始简历文件URL',
    original_file_type VARCHAR(20) COMMENT '文件类型：PDF/DOCX/IMG',

    -- 解析状态
    parse_status VARCHAR(20) DEFAULT 'PENDING' COMMENT '解析状态：PENDING/PARSING/SUCCESS/FAILED',
    parsed_at DATETIME COMMENT '解析完成时间',

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_user_id (user_id),
    INDEX idx_is_default (user_id, is_default)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户简历表（极简版，仅保留面试必需）';
```

#### 字段保留说明

| 字段 | 是否保留 | 原因 |
|-----|---------|------|
| `resume_id`, `user_id`, `resume_name` | ✅ 保留 | 基础字段 |
| `is_default` | ✅ 保留 | 支持多简历 |
| `work_years` | ✅ 保留 | 影响问题难度 |
| `current_position` | ✅ 保留 | 了解技术方向 |
| `target_level` | ✅ 保留 | 决定问题深度 |
| `education_level` | ❌ 删除 | 面试不问学历 |
| `school_name`, `major`, `graduation_year` | ❌ 删除 | 无关信息 |
| `summary` | ✅ 保留 | AI 了解技术背景 |

```sql
CREATE TABLE resume_skills (
```

### 2.3 知识点库设计（重要说明）

**核心理念**：
- `skill_tags` 表存储的是**知识点**（如：HashMap、Redis 集群、Spring Boot 自动装配）
- **不存储预定义的题目**
- 面试题目由 **AI 主面试官动态生成**，基于：
  - 要考察的知识点（由副面试官决定）
  - 岗位 JD 要求
  - 用户简历内容

**面试流程**：
```
副面试官决定：下一个考察 "HashMap" 知识点
         ↓
主面试官 Agent 根据以下信息动态生成问题：
         ↓
    - 知识点：HashMap
    - JD 要求：掌握集合框架
    - 简历内容：3 年经验，做过电商项目
    - 已考察：ArrayList、LinkedList
         ↓
生成个性化问题：
"看你简历上 3 年 Java 经验，应该对集合很熟悉。
 HashMap 在高并发场景下会有什么问题？如何解决？"
```

**为什么不用预定义题库？**
1. 我们没有题库数据，从零创建工作量巨大
2. 预定义题目不够灵活，无法根据简历定制
3. AI 可以生成无限的问题，更贴近真实面试

**skill_tags 表的作用**：
- 作为"待考察知识点清单"
- 支持层级结构（Java → Collection → HashMap）
- 可与 JD 和简历关联

### 3.2 三 Agent 详细设计

#### 3.2.1 主面试官 Agent (MainInterviewerAgent)

**核心职责**：根据岗位 JD 和用户简历进行自然对话，针对性提问

```java
/**
 * 主面试官 Agent
 * <p>
 * 职责：基于 JD + 简历进行提问，深度挖掘项目经历和声称技能
 * </p>
 */
public interface MainInterviewerAgent {

    @SystemMessage("""
        # 角色设定
        你是 JobTracker 模拟面试系统的主面试官，正在对候选人进行专业面试。

        # ⏰ 当前时间
        - 当前日期：{{current_date}}
        - 当前时间：{{current_time}}
        - 星期：{{day_of_week}}

        # 面试上下文

        ## 岗位信息
        - 公司：{{company_name}}
        - 岗位：{{job_title}}
        - 级别：{{seniority_level}}
        - 岗位技能要求：{{jd_skills}}

        ## 用户简历摘要
        - 工作年限：{{resume_work_years}} 年
        - 当前职位：{{current_position}}
        - 学历：{{education}}

        ## 简历中声称的核心技能
        {{resume_skills}}

        ## 简历中的项目经历
        {{resume_projects}}

        ## 已考察的知识点
        {{covered_topics}}

        # 你的职责

        ## 1. 提问策略（优先级从高到低）
        1. **项目深挖**（最高优先级）
           - "看到你简历上写了{{project_name}}项目，能详细讲讲吗？"
           - "在这个项目中，你负责的模块遇到了什么技术挑战？"
           - "项目的{{metric}}是多少？是如何优化的？"

        2. **技能验证**
           - "你简历上写着精通{{skill}}，那{{advanced_question}}？"
           - 针对"精通"、"熟悉"等词汇，提出有深度的问题
           - 验证简历中的技能声称是否真实

        3. **JD 要求覆盖**
           - 基于 JD 中的技能要求，补充提问简历未提及的部分

        4. **经验匹配度评估**
           - 基于{{work_years}}年经验，提出相应难度的问题
           - 避免"过度提问"或"提问过浅"

        ## 2. 提问示例

        ❌ 错误：
        "请讲讲 HashMap 的原理。"
        （太泛泛，没有结合简历）

        ✅ 正确（项目深挖）：
        "看到你简历上写了电商秒杀项目，能详细讲讲在高并发场景下，
        Redis 和 Kafka 是如何配合使用的吗？"

        ✅ 正确（技能验证）：
        "你简历上写着精通 Redis，那我想问一下，Redis 的集群模式中，
        Slot 的迁移机制是怎样的？"

        ✅ 正确（追问）：
        "你提到了用 Kafka 做消息队列，那为什么选择 Kafka 而不是 RocketMQ？
        主要是考虑哪些因素？"

        # 严格约束
        1. **严禁评分**：不要提及分数、等级
        2. **严禁重复**：不要问已考察过的知识点
```

```java
@Service
@RequiredArgsConstructor

     * 为求职申请启动模拟面试
     *
     * @param applicationId 申请ID
     * @return 面试会话信息
     */
    public MockInterviewSessionDTO startInterview(Long applicationId) {
        // 1. 获取申请信息
        JobApplication application = applicationService.getById(applicationId);

        // 2. 检查是否有简历
        if (application.getResumeId() == null) {
            throw new BusinessException("请先上传简历");
        }

        // 3. 获取 JD 技能要求
        List<Long> jdSkillIds = skillService.getApplicationSkills(applicationId);

        if (jdSkillIds.isEmpty()) {
            throw new BusinessException("请先完善岗位技能要求（JD解析）");
        }

        // 4. 获取简历信息
        UserResume resume = resumeService.getById(application.getResumeId());

        // 5. 获取待考察的知识点列表（JD要求 + 简历声称技能）
        List<Long> pendingSkillIds = getPendingSkills(jdSkillIds, resume.getId());

        // 6. 创建面试会话（保存快照）
        MockInterviewSession session = MockInterviewSession.builder()
            .sessionId(generateSessionId())
            .applicationId(applicationId)
            .userId(application.getUserId())
            .resumeId(application.getResumeId())
            .companyId(application.getCompanyId())
            .jobTitle(application.getJobTitle())
            .seniorityLevel(application.getSeniorityLevel())

            // 保存快照（避免后续更新影响历史记录）
            .resumeSnapshot(buildResumeSnapshot(resume))
            .jdSnapshot(buildJDSnapshot(application))

            .state(InterviewState.INIT)
            .currentRound(0)
            .totalRounds(5)

            // 待考察知识点
            .skillsCovered(new ArrayList<>())
            .skillsPending(buildPendingSkills(pendingSkillIds))

            .build();

        sessionRepository.save(session);

        // 7. 初始化编排器（传入待考察知识点，不是预定义题目）
        orchestrator.initialize(session, pendingSkillIds);

        return toDTO(session);
    }

    /**
     * 获取待考察的知识点（JD要求 + 简历声称技能，去重）
     */
    private List<Long> getPendingSkills(List<Long> jdSkillIds, Long resumeId) {
        // 获取简历中声称的技能
        List<Long> resumeSkillIds = resumeSkillRepository.findSkillIdsByResumeId(resumeId);

        // 合并并去重
        Set<Long> allSkills = new LinkedHashSet<>();
        allSkills.addAll(jdSkillIds);    // JD 要求的技能
        allSkills.addAll(resumeSkillIds); // 简历中声称的技能

        return new ArrayList<>(allSkills);
    }

    /**
     * 获取申请的历史面试记录
     */
    public List<MockInterviewSessionDTO> getInterviewHistory(Long applicationId) {
        return sessionRepository.findByApplicationId(applicationId)
            .stream()
            .map(this::toDTO)
            .toList();
    }

    /**
     * 获取面试报告
     */
    public MockInterviewReportDTO getInterviewReport(String sessionId) {
        MockInterviewSession session = sessionRepository.findById(sessionId)
            .orElseThrow(() -> new ResourceNotFoundException("面试会话不存在"));

        List<EvaluationResult> evaluations = evaluationRepository.findBySessionId(sessionId);

        return MockInterviewReportDTO.builder()
            .session(session)
            .evaluations(evaluations)
            .strengths(analyzeStrengths(evaluations))
            .weaknesses(analyzeWeaknesses(evaluations))
            .improvementPlan(generateImprovementPlan(evaluations))
            .build();
    }

    /**
     * 面试结束后的处理
     */
    @Transactional
    public void finishInterview(String sessionId) {
        MockInterviewSession session = sessionRepository.findById(sessionId)
            .orElseThrow(() -> new ResourceNotFoundException("面试会话不存在"));

        // 计算总分
        List<EvaluationResult> evaluations = evaluationRepository.findBySessionId(sessionId);
        double totalScore = evaluations.stream()
            .mapToDouble(EvaluationResult::getTotalScore)
            .average()
            .orElse(0.0);

        session.setTotalScore(totalScore);
        session.setState(InterviewState.FINISHED);
        session.setFinishedAt(LocalDateTime.now());
        session.setDurationSeconds(
            (int) Duration.between(session.getStartedAt(), session.getFinishedAt()).getSeconds()
        );

        sessionRepository.updateById(session);

        // 更新申请记录
        JobApplication application = applicationService.getById(session.getApplicationId());
        application.setMockInterviewCount((application.getMockInterviewCount() ?? 0) + 1);

        // 更新最佳成绩
        if (application.getBestMockScore() == null || totalScore > application.getBestMockScore()) {
            application.setBestMockScore(totalScore);
        }

        applicationService.updateById(application);
    }
}
```

```java
     * @return 匹配的题目列表（按难度排序）
```

```typescript
// 客户端发送
interface ClientMessage {
  type: 'START' | 'USER_MESSAGE' | 'STOP';
  sessionId: string;
  content?: string;
}

// 服务端推送
interface ServerMessage {
  type:
    | 'WELCOME'           // 欢迎消息
    | 'ASSISTANT_STREAM'  // AI 流式回复
    | 'EVALUATION_UPDATE' // 评分更新
    | 'STATE_UPDATE'      // 状态更新
    | 'ROUND_COMPLETE'    // 单轮完成
    | 'INTERVIEW_END'     // 面试结束
    | 'REPORT_READY';     // 报告就绪

  sessionId: string;
  content?: string;
  data?: {
    // EVALUATION_UPDATE
    round_number?: number;
    technical_score?: number;
    logic_score?: number;
    depth_score?: number;
    total_score?: number;

    // STATE_UPDATE
    state?: string;
    current_round?: number;

    // ROUND_COMPLETE
    summary?: string;
    next_topic?: string;
  };
}
```
