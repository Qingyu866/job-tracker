# 考察计划持久化与暂停恢复机制设计

## 文档概述

**创建日期**: 2026-03-19
**目的**: 补充考察计划持久化设计和暂停/恢复机制
**用户建议**: 考察计划必须持久化，支持暂停后恢复进度

---

## 一、核心需求

### 1.1 考察计划持久化

**用户原话**：
> "还要将这个副面试官生成的要考察的知识点和已经考察完来的知识点持久化到数据库里面，这样用在后续暂停对话然后又回来的时候副面试管就知道当前考察的知识点的进度"

**核心需求**：
1. ✅ 考察计划必须持久化到数据库
2. ✅ 记录每个知识点的考察状态（待考察/已完成/跳过）
3. ✅ 支持暂停后恢复，副面试官知道当前进度
4. ✅ 避免重复考察或遗漏

---

### 1.2 暂停/恢复场景

**典型场景**：
```
用户场景 1：面试中途中断
- 第 3 轮面试时，用户有事离开
- 点击"暂停"按钮
- 系统保存当前进度
- 2 天后回来，点击"继续"
- 副面试官从第 4 轮开始

用户场景 2：分次完成面试
- 第 1 天：完成第 1-5 轮
- 点击"暂停"
- 第 2 天：继续第 6-10 轮
- ...
- 第 5 天：完成第 21-25 轮
- 点击"结束"
```

## 二、表结构设计

### 2.1 interview_question_plans 表（考察计划表）

**表结构**：
```sql
CREATE TABLE interview_question_plans (
  plan_id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '考察计划ID',
  session_id VARCHAR(50) NOT NULL COMMENT '面试会话ID',

  -- 轮次信息
  round_number INT NOT NULL COMMENT '轮次（1-25）',
  sequence_number INT NOT NULL COMMENT '该轮的序号（从1开始）',

  -- 技能信息
  skill_name VARCHAR(100) NOT NULL COMMENT '要考察的技能名称',
  skill_id BIGINT COMMENT '关联的技能ID（可能为NULL，动态补全）',

  -- 考察策略
  topic_source VARCHAR(50) COMMENT '选题来源：PROJECT_DEEP_DIVE, SKILL_VERIFICATION, JD_REQUIREMENT, GENERAL',
  question_type VARCHAR(50) COMMENT '问题类型：PROJECT_DEEP_DIVE, SKILL_VERIFICATION, JD_REQUIREMENT, GENERAL, OPEN_ENDED',
  difficulty INT COMMENT '难度等级（1-5）',

  -- 上下文信息
  context_info TEXT COMMENT '上下文信息（例如：简历声称精通Redis）',
  reason TEXT COMMENT '选择该技能的原因',

  -- 状态管理 ⭐ 重要
  status VARCHAR(20) DEFAULT 'PENDING' COMMENT '状态：PENDING(待考察), COMPLETED(已完成), SKIPPED(跳过), PAUSED(暂停中)',

  -- 执行结果
  actual_question TEXT COMMENT '实际生成的问题（保存后可回溯）',
  actual_difficulty INT COMMENT '实际使用的难度（可能与计划不同）',

  -- 时间戳
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  -- 索引
  INDEX idx_session_status (session_id, status),
  INDEX idx_round_sequence (round_number, sequence_number),
  INDEX idx_session_round (session_id, round_number)
) COMMENT='模拟面试考察计划表';
```

**字段说明**：

| 字段 | 类型 | 说明 |
|-----|------|------|
| `session_id` | VARCHAR(50) | 关联到 `mock_interview_sessions.session_id` |
| `round_number` | INT | 第几轮（1-25） |
| `sequence_number` | INT | 该轮的第几个问题（支持一轮多个问题） |
| `skill_name` | VARCHAR(100) | 技能名称（如："Redis"、"Spring Boot"） |
| `skill_id` | BIGINT | 关联 `skill_tags.skill_id`（可能为 NULL） |
| `topic_source` | VARCHAR(50) | 选题来源 |
| `question_type` | VARCHAR(50) | 问题类型 |
| `difficulty` | INT | 难度等级（1-5） |
| `context_info` | TEXT | 上下文（如："简历声称精通 Redis"） |
| `reason` | TEXT | 选择原因 |
| `status` | VARCHAR(20) | **状态管理（重要）** |
| `actual_question` | TEXT | 实际生成的问题（用于暂停后恢复上下文） |
| `actual_difficulty` | INT | 实际难度（可能与计划不同） |

---

### 2.2 状态枚举定义

**status 字段的值**：

| 状态 | 说明 | 下一步操作 |
|-----|------|----------|
| `PENDING` | 待考察 | 等待执行 |
| `COMPLETED` | 已完成 | 不再执行 |
| `SKIPPED` | 跳过 | 不再执行 |
| `PAUSED` | 暂停中 | 等待恢复 |

## 三、核心流程设计

### 3.1 创建会话流程

```
用户点击"开始面试"
    ↓
┌─────────────────────────────────────────────────────────┐
│ MockInterviewService.createSession()                   │
└─────────────────────────────────────────────────────────┘
    │
    ├─► 1. 解析 JD 和简历
    │   jdParsingService.parseJD()
    │   resumeParsingService.parseResume()
    │
    ├─► 2. 调用副面试官生成考察计划
    │   agents.viceInterviewer().generateQuestionPlan(context)
    │   ↓
    │   返回：List<QuestionPlan>（25 轮的计划）
    │   [
    │     {round: 1, seq: 1, skill: "项目深挖", status: "PENDING"},
    │     {round: 1, seq: 2, skill: "Redis", status: "PENDING"},
    │     {round: 2, seq: 1, skill: "Spring Boot", status: "PENDING"},
    │     ...
    │   ]
    │
    ├─► 3. 批量插入考察计划 ⭐
    │   interviewQuestionPlanService.batchInsert(plans)
    │   ↓
    │   INSERT INTO interview_question_plans (...) VALUES (...)
    │   INSERT INTO interview_question_plans (...) VALUES (...)
    │   ...
    │
    ├─► 4. 更新会话信息
    │   session.setTotalPlans(plans.size())

    │   interviewService.updateSession(session)
    │
    └─► 5. 返回会话

```

### 3.2 每轮对话流程

```
用户发送回答
    ↓
┌─────────────────────────────────────────────────────────┐
│ MockInterviewController.sendMessage()                     │
└─────────────────────────────────────────────────────────┘
    │
    ├─► 1. 获取会话当前进度
    │   Long currentPlanId = session.getCurrentPlanId()
    │   Integer currentRound = session.getCurrentRound()
    │
    ├─► 2. 查询当前应该执行的计划
    │   QuestionPlan plan = questionPlanService.getNextPlan(sessionId, currentRound)
    │   ↓
    │   SELECT * FROM interview_question_plans
    │   WHERE session_id = ? AND status = 'PENDING'
    │   ORDER BY round_number, sequence_number
    │   LIMIT 1
    │
    ├─► 3. 如果没有计划了，结束面试
    │   if (plan == null) {
    │       return finishInterview(sessionId);
    │   }
    │
    ├─► 4. 确保技能存在
    │   Map<String, Long> skillMap = ensureSkillsExist([plan.skillName])
    │   ↓
    │   查询 skill_tags
    │   如果缺少，AI 生成并批量插入
    │
    ├─► 5. 更新计划状态为"执行中"
    │   plan.setStatus("IN_PROGRESS")
    │   questionPlanService.updatePlan(plan)
    │
    ├─► 6. 评估用户回答（上一轮）
    │   agents.evaluator().evaluate(context)
    │
    ├─► 7. 主面试官生成问题
    │   agents.mainInterviewer().askQuestion(context)
    │
    ├─► 8. 保存问题和评分
    │   messageService.addQuestion()
    │   evaluationService.createEvaluation()
    │
    ├─► 9. 更新计划状态为"已完成" ⭐
    │   plan.setStatus("COMPLETED")
    │   plan.setActualQuestion(question)
    │   questionPlanService.updatePlan(plan)
    │
    ├─► 10. 更新会话进度
    │   session.setCompletedPlans(session.getCompletedPlans() + 1)
    │   session.setCurrentPlanId(getNextPlanId(sessionId))
    │   interviewService.updateSession(session)
    │
    ├─► 11. 进入下一轮
    │   interviewService.nextRound()
    │
    └─► 12. 返回给用户
        return Result.success(message)
```

### 3.3 暂停流程

```
用户点击"暂停"按钮
    ↓
┌─────────────────────────────────────────────────────────┐
│ MockInterviewController.pauseInterview()               │
│                                                         │
│ POST /api/mock-interview/sessions/{sessionId}/pause  │
└─────────────────────────────────────────────────────────┘
    │
    ├─► 1. 更新会话状态
    │   session.setState("PAUSED")
    │   session.setPausedAt(LocalDateTime.now())
    │   interviewService.updateSession(session)
    │
    ├─► 2. 更新当前执行中的计划状态
    │   QuestionPlan currentPlan = questionPlanService.getCurrentPlan(sessionId)
    │   if (currentPlan != null && currentPlan.getStatus().equals("IN_PROGRESS")) {
    │       currentPlan.setStatus("PAUSED")
    │       questionPlanService.updatePlan(currentPlan)
    │   }
    │
    ├─► 3. 持久化所有 Agent 记忆
    │   agentFactory.persistMemories(sessionId)
    │
    └─► 4. 返回成功
        return Result.success("面试已暂停")
```

### 3.4 恢复流程

```
用户点击"继续"按钮
    ↓
┌─────────────────────────────────────────────────────────┐
│ MockInterviewController.resumeInterview()              │
│                                                         │
│ POST /api/mock-interview/sessions/{sessionId}/resume │
└─────────────────────────────────────────────────────────┘
    │
    ├─► 1. 检查会话状态
    │   if (!session.getState().equals("PAUSED")) {
    │       return Result.error("会话未暂停，无法继续");
    │   }
    │
    ├─► 2. 恢复所有 Agent 记忆
    │   agentFactory.restoreMemories(sessionId)
    │
    ├─► 3. 获取当前进度
    │   Long currentPlanId = session.getCurrentPlanId()
    │   Integer currentRound = session.getCurrentRound()
    │
    ├─► 4. 查询下一个待执行的计划
    │   QuestionPlan plan = questionPlanService.getNextPlan(sessionId, currentRound)
    │
    ├─► 5. 如果没有计划了，提示结束面试
    │   if (plan == null) {
    │       return Result.error("所有考察计划已完成，请结束面试");
    │   }
    │
    ├─► 6. 恢复计划状态
    │   if (plan.getStatus().equals("PAUSED")) {
    │       plan.setStatus("PENDING")
    │       questionPlanService.updatePlan(plan)
    │   }
    │
    ├─► 7. 更新会话状态
    │   session.setState("IN_PROGRESS")
    │   session.setResumedAt(LocalDateTime.now())
    │   interviewService.updateSession(session)
    │
    ├─► 8. 生成恢复提示（可选）
    │   String resumeMessage = buildResumeMessage(session, plan)
    │
    └─► 9. 返回成功
        return Result.success("面试已恢复，准备开始第 " + plan.getRoundNumber() + " 轮")
```

## 五、数据一致性保证

### 5.1 并发控制

**问题**：用户快速点击"暂停"和"继续"按钮

**解决方案**：
```java
@Transactional
public Result<Void> pauseInterview(String sessionId) {
    // 使用乐观锁
    MockInterviewSession session = sessionMapper.selectById(sessionId);

    if (session.getState().equals("PAUSED")) {
        return Result.error("会话已暂停");
    }

    // 更新状态
    session.setState("PAUSED");
    session.setPausedAt(LocalDateTime.now());
    sessionMapper.updateById(session);

    return Result.success("面试已暂停");
}
```

---

### 5.2 状态转换规则

```
INIT → IN_PROGRESS（开始面试）
IN_PROGRESS → PAUSED（暂停）
PAUSED → IN_PROGRESS（恢复）
PAUSED → FINISHED（直接结束）
IN_PROGRESS → FINISHED（正常结束）
```

## 七、核心优势

### 7.1 可恢复性

**场景**：面试中断
```
第 5 轮 → 用户离开 → 暂停 → 2天后回来 → 恢复 → 从第 6 轮继续
```

**数据库记录**：
```sql
-- 暂停前
plan_id | session_id | round_number | status
--------|-----------|--------------|--------
1       | abc123     | 1            | COMPLETED
2       | abc123     | 1            | COMPLETED
...
10      | abc123     | 5            | COMPLETED
11      | abc123     | 6            | PENDING
12      | abc123     | 6            | PENDING
```

**恢复后**：
- 副面试官查询 `status = 'PENDING'` 的第一个计划
- 从第 6 轮继续执行
- 不会遗漏或重复

## 八、总结

### 核心改进

1. **考察计划持久化** ⭐
   - 创建 `interview_question_plans` 表
   - 记录每轮的考察计划
   - 支持暂停/恢复

2. **状态管理**
   - PENDING（待考察）
   - COMPLETED（已完成）
   - SKIPPED（跳过）
   - PAUSED（暂停中）

3. **进度追踪**
   - `current_plan_id`：当前执行位置
   - `completed_plans`：已完成数量
   - `total_plans`：总数量

4. **暂停/恢复机制**
   - 暂停：保存状态和 Agent 记忆
   - 恢复：读取状态和 Agent 记忆
   - 继续执行下一个计划

### 数据流

```
创建会话
    ↓
生成考察计划（25 轮）
    ↓
持久化到数据库 ⭐
    ↓
按照计划执行
    ↓
每轮更新状态
    ↓
支持暂停/恢复 ⭐
    ↓
进度可视化
```
