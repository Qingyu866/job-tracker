# 考察计划与评分表关系分析 - 避免冗余设计

## 文档概述

**创建日期**: 2026-03-19
**问题**: `interview_question_plans` 表是否与 `mock_interview_evaluations` 表重叠？
**目的**: 分析两个表的关系，优化设计，避免冗余

---

## 一、现有表结构分析

### 1.1 MockInterviewEvaluation（评分记录表）

**用途**: 存储**每轮对话后的评分结果**

**表结构**：
```sql
CREATE TABLE mock_interview_evaluations (
  evaluation_id BIGINT PRIMARY KEY,
  session_id VARCHAR(50),      -- 面试会话ID
  round_number INT,           -- 第几轮
  skill_id BIGINT,             -- 技能ID
  skill_name VARCHAR(100),     -- 技能名称
  question_text TEXT,          -- 问题内容
  user_answer TEXT,            -- 用户回答
  technical_score DECIMAL,     -- 技术准确性（0-4）
  logic_score DECIMAL,         -- 逻辑清晰度（0-3）
  depth_score DECIMAL,         -- 深度与广度（0-3）
  total_score DECIMAL,        -- 总分（0-10）
  feedback TEXT,               -- 详细反馈
  suggestion TEXT,            -- 改进建议
  keywords_matched VARCHAR,   -- 匹配的关键词
  keywords_missing VARCHAR,    -- 缺失的关键词
  evaluated_at TIMESTAMP       -- 评分时间
);
```

**关键特点**：
- ✅ **执行后的记录**：每轮对话后创建
- ✅ **结果数据**：包含评分、反馈、建议
- ✅ **历史追溯**：可以查询历史评分记录

## 三、是否真的重叠？

### 3.1 类比关系

**教育考试类比**：
- **InterviewQuestionPlan** = 考试大纲
  - 考试前就知道要考什么
  - 例如："第一章：Java基础，第二章：集合框架，第三章：多线程"

- **MockInterviewEvaluation** = 试卷评分
  - 考试后才有的分数
  - 例如："第一章得分：85，第二章得分：72"

**结论**：**不重叠！一个是规划，一个是结果**

---

### 3.2 数据流关系

```
创建会话时
    ↓
┌─────────────────────────────────────────────┐
│ InterviewQuestionPlan（考察计划）         │
│                                             │
│ round=1, skill="Redis", status=PENDING    │ → 计划考察 Redis
│ round=2, skill="Spring Boot", status=PENDING │ → 计划考察 Spring Boot
│ round=3, skill="MySQL", status=PENDING     │ → 计划考察 MySQL
└─────────────────────────────────────────────┘
    ↓
执行面试（按计划）
    ↓
┌─────────────────────────────────────────────┐
│ MockInterviewEvaluation（评分记录）          │
│                                             │
│ round=1, skill="Redis", total_score=6/10    │ → 已完成
│ round=2, skill="Spring Boot", total_score=8/10│ → 已完成
│ round=3, skill="MySQL", total_score=4/10      │ → 已完成
└─────────────────────────────────────────────┘
```

**关系**：
- InterviewQuestionPlan **指导** MockInterviewEvaluation
- MockInterviewEvaluation **验证** InterviewQuestionPlan
- 两者是 **一对多** 的关系：一个计划 → 一个评分记录

## 四、优化方案设计

### 方案 A：合并表设计（推荐）

**核心思想**：直接在 `MockInterviewEvaluation` 表中添加状态字段

#### 修改后的表结构

**新增字段**：
```sql
ALTER TABLE mock_interview_evaluations ADD COLUMN plan_status VARCHAR(20) DEFAULT 'PENDING' COMMENT '计划状态：PENDING(待考察), COMPLETED(已完成), SKIPPED(跳过), PAUSED(暂停中)';
ALTER TABLE mock_interview_evaluations ADD COLUMN topic_source VARCHAR(50) COMMENT '选题来源：PROJECT_DEEP_DIVE, SKILL_VERIFICATION, JD_REQUIREMENT, GENERAL';
ALTER TABLE mock_interview_evaluations ADD COLUMN question_type VARCHAR(50) COMMENT '问题类型';
ALTER TABLE mock_interview_evaluations ADD COLUMN planned_difficulty INT COMMENT '计划难度（1-5）';
ALTER TABLE mock_interview_evaluations ADD COLUMN context_info TEXT COMMENT '上下文信息';
ALTER TABLE mock_int_evaluations ADD COLUMN reason TEXT COMMENT '选择该技能的原因';
ALTER TABLE mock_interview_evaluations ADD COLUMN actual_difficulty INT COMMENT '实际难度（可能与计划不同）';
ALTER TABLE mock_interview_evaluations ADD COLUMN created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '计划创建时间';
```

#### 完整的表结构

```sql
CREATE TABLE mock_interview_evaluations (
  evaluation_id BIGINT PRIMARY KEY,
  session_id VARCHAR(50) NOT NULL,
  round_number INT NOT NULL,
  skill_id BIGINT,
  skill_name VARCHAR(100) NOT NULL,

  -- 考察计划字段（新增）⭐
  plan_status VARCHAR(20) DEFAULT 'PENDING' COMMENT '计划状态：PENDING(待考察), COMPLETED(已完成), SKIPPED(跳过), PAUSED(暂停中)',
  topic_source VARCHAR(50) COMMENT '选题来源',
  question_type VARCHAR(50) COMMENT '问题类型',
  planned_difficulty INT COMMENT '计划难度（1-5）',
  context_info TEXT COMMENT '上下文信息',
  reason TEXT COMMENT '选择原因',
  actual_difficulty INT COMMENT '实际难度',

  -- 面试执行字段
  question_text TEXT COMMENT '问题内容',
  user_answer TEXT COMMENT '用户回答',

  -- 评分字段
  technical_score DECIMAL(4,1),
  logic_score DECIMAL(3,1),
  depth_score DECIMAL(3,1),
  total_score DECIMAL(4,1),
  feedback TEXT,
  suggestion TEXT,
  keywords_matched VARCHAR,
  keywords_missing VARCHAR,
  evaluated_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  INDEX idx_session_status (session_id, plan_status),
  INDEX idx_round_skill (round_number, skill_name)
);
```

#### 使用方式

**创建会话时**（生成计划）：
```java
// 副面试官生成考察计划
List<QuestionPlan> plans = agents.viceInterviewer().generateQuestionPlan(context);

// 保存到 mock_interview_evaluations 表（状态为 PENDING）
for (QuestionPlan plan : plans) {
    MockInterviewEvaluation evaluation = MockInterviewEvaluation.builder()
        .sessionId(sessionId)
        .roundNumber(plan.getRoundNumber())
        .skillId(plan.getSkillId())
        .skillName(plan.getSkillName())
        .planStatus("PENDING")  // ⭐ 新增：计划状态
        .topicSource(plan.getTopicSource())
        .questionType(plan.getQuestionType())
        .plannedDifficulty(plan.getDifficulty())
        .contextInfo(plan.getContextInfo())
        .reason(plan.getReason())
        .build();

    evaluationMapper.insert(evaluation);
}
```

**执行面试时**（按照计划执行）：
```java
// 1. 获取当前轮次的计划（PENDING 状态）
MockInterviewEvaluation plan = evaluationMapper.selectOne(
    new LambdaQueryWrapper<MockInterviewEvaluation>()
        .eq(MockInterviewEvaluation::getSessionId, sessionId)
        .eq(MockInterviewEvaluation::getRoundNumber, currentRound)
        .eq(MockInterviewEvaluation::getPlanStatus, "PENDING")
);

// 2. 评估用户回答
agents.evaluator().evaluate(context);

// 3. 更新计划状态为已完成
plan.setPlanStatus("COMPLETED");
plan.setActualDifficulty(actualDifficulty);
plan.setQuestionText(question);
plan.setUserAnswer(userAnswer);
plan.setTechnicalScore(technicalScore);
plan.setTotalScore(totalScore);
evaluationMapper.updateById(plan);
```

### 方案 B：保留两表设计（原设计）

**核心思想**：保持两个表独立，通过外键关联

#### InterviewQuestionPlan 表
```sql
CREATE TABLE interview_question_plans (
  plan_id BIGINT PRIMARY KEY,
  session_id VARCHAR(50),
  round_number INT,
  skill_name VARCHAR(100),
  skill_id BIGINT,
  status VARCHAR(20),
  topic_source VARCHAR(50),
  question_type VARCHAR(50),
  difficulty INT,
  context_info TEXT,
  reason TEXT
);
```

#### MockInterviewEvaluation 表
```sql
CREATE TABLE mock_interview_evaluations (
  evaluation_id BIGINT PRIMARY KEY,
  session_id VARCHAR(50),
  round_number INT,
  skill_id BIGINT,
  skill_name VARCHAR(100),
  question_text TEXT,
  user_answer TEXT,
  technical_score DECIMAL,
  logic_score DECIMAL,
  depth_score DECIMAL,
  total_score DECIMAL,
  feedback TEXT,
  suggestion TEXT,
  evaluated_at TIMESTAMP,

  -- 新增：关联到计划表
  plan_id BIGINT COMMENT '关联的考察计划ID'
);
```

#### 使用方式

**创建会话时**：

## 五、方案对比

### 对比表

| 方面 | 方案 A（合并表） | 方案 B（两表） |
|-----|-------------|-----------|
| **表数量** | 1 个表 | 2 个表 |
| **字段数量** | 约 25 个字段 | 约 15 个字段（每个表） |
| **复杂度** | 较简单（单表查询） | 较复杂（需要 JOIN） |
| **冗余度** | 低（一个表包含所有信息） | 较高（两个字段重复） |
| **灵活性** | 较低（结构固定） | 较高（计划表可独立扩展） |
| **可维护性** | 较好（字段都在一个表） | 较复杂（需要维护两个表） |
| **性能** | 较好（单表查询） | 较差（需要 JOIN） |

## 六、推荐方案

### 推荐：方案 A（合并表设计）

**理由**：
1. ✅ **简化设计**：一个表包含所有信息，不需要 JOIN
2. ✅ **避免冗余**：不需要在两个表之间同步 `session_id` 和 `round_number`
3. ✅ **状态管理简单**：直接更新 `plan_status` 字段
4. ✅ **查询简单**：获取进度只需要查询一个表
5. ✅ **暂停/恢复简单**：只需要查询 `plan_status = 'PAUSED'` 的记录

---

### 实施细节

#### 修改后的数据流

**创建会话时**：
```
副面试官生成考察计划（25 轮）
    ↓
批量插入 mock_interview_evaluations（plan_status = 'PENDING'）
    ↓
session.setTotalPlans(25)
session.setCompletedPlans(0)
```

**每轮对话时**：
```
1. 查询：SELECT * FROM mock_interview_evaluations
   WHERE session_id = ? AND round_number = ? AND plan_status = 'PENDING'
   ORDER BY evaluation_id

2. 评估用户回答

3. 更新：UPDATE mock_interview_evaluations SET
   plan_status = 'COMPLETED',
   question_text = ?,
   user_answer = ?,
   technical_score = ?,
   ...
   WHERE evaluation_id = ?
```

**暂停/恢复时**：
```
暂停：UPDATE mock_interview_evaluations SET
   plan_status = 'PAUSED'
   WHERE session_id = ? AND round_number = ? AND plan_status = 'IN_PROGRESS'

恢复：SELECT * FROM mock_interview_evaluations
   WHERE session_id = ? AND plan_status = 'PENDING'
   ORDER BY round_number, evaluation_id
   LIMIT 1
```

## 七、总结

### 核心结论

1. **两个表不重叠**：
   - InterviewQuestionPlan = 考试大纲（规划）
   - MockInterviewEvaluation = 试卷评分（结果）

2. **推荐方案 A（合并表）**：
   - 在 `mock_interview_evaluations` 表中添加状态字段
   - 一个表包含所有信息
   - 简化设计，避免冗余

3. **字段调整**：
   - 新增 `plan_status`：计划状态（PENDING/COMPLETED/SKIPPED/PAUSED）
   - 新增 `topic_source`：选题来源
   - 新增 `question_type`：问题类型
   - 新增 `planned_difficulty`：计划难度
   - 新增 `actual_difficulty`：实际难度
   - 新增 `context_info`：上下文信息
   - 新增 `reason`：选择原因
