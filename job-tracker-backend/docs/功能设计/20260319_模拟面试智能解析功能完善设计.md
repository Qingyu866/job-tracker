## 核心问题

### 1. 缺少智能解析层

**问题**：
- ❌ 没有实现 `ApplicationSkillService.parseSkillsFromJD()`
- ❌ 没有实现 `ResumeParsingService`（我刚创建的，但还没集成）
- ❌ 没有实现 `JDParsingService`（我刚创建的，但还没集成）

**影响**：
- Agent 无法获得结构化的技能列表
- Agent 无法获得结构化的项目列表
- 面试提问缺乏针对性

### 2. MockInterviewService 快照生成逻辑有问题

**当前逻辑**（`MockInterviewService.java`）：

```java
// 3. 生成简历快照
String resumeSnapshot = generateResumeSnapshot(resumeId);
// ❌ 问题：只是从数据库表中读取已有数据
// ❌ 如果用户没有手动录入，返回的就是空对象

// 4. 生成 JD 快照
String jdSnapshot = generateJdSnapshot(applicationId);
// ❌ 问题：只是从 application 表中读取
// ❌ 如果 skills_required 字段为空，返回的就是空数组
```

**问题根源**：
- `generateResumeSnapshot()` 依赖 `resume_skills`、`resume_projects` 表有数据
- `generateJdSnapshot()` 依赖 `application.skills_required` 字段有值
- 但这些数据都需要用户**手动录入**，没有 AI 解析！

### 3. Agent 缺少关键上下文

**副面试官应该有的上下文**：
```json
{
  "skills_covered": ["Redis", "Spring Boot"],  // 已考察的技能
  "skills_pending": ["MySQL", "Kafka"]        // 待考察的技能（从 JD 解析）
}
```

**当前实际情况**：
```json
{
  "skills_covered": "无",
  "skills_pending": "无"
}
```

## 详细设计

### 1. ResumeParsingService（简历解析服务）

**文件位置**：`src/main/java/com/jobtracker/service/ResumeParsingService.java`

**职责**：
- 从简历文本（纯文本或 OCR 结果）中提取结构化信息
- 返回 `ResumeParseResult` 对象

**输入**：
```java
String resumeText = """
工作年限：3 年
当前职位：Java后端工程师

技能列表：
- Redis（精通）
- Spring Boot（精通）
- MySQL（熟练掌握）

项目经验：
项目名称：电商秒杀系统
角色：后端开发工程师
技术栈：Java, Spring Boot, Redis, Kafka, MySQL
描述：一个支持高并发的商品秒杀系统
""";
```

**输出**：
```java
ResumeParseResult {
  workYears: 3,
  currentPosition: "Java后端工程师",
  skills: [
    SkillInfo {
      skillName: "Redis",
      level: "精通",
      years: 2,
      projectsUsed: ["电商秒杀系统"]
    },
    ...
  ],
  projects: [
    ProjectInfo {
      projectName: "电商秒杀系统",
      role: "后端开发工程师",
      techStack: ["Java", "Spring Boot", "Redis", "Kafka", "MySQL"],
      description: "一个支持高并发的商品秒杀系统",
      ...
    }
  ],
  workExperiences: [...]
}
```

**提示词设计**：
```
# 任务
你是一个专业的简历解析专家。请从以下简历文本中提取结构化信息。

# 简历文本
{resumeText}

# 提取要求
1. 基本信息：work_years, current_position
2. 技能列表：skill_name, level (精通/熟练掌握/了解), years, projects_used
3. 项目经验：project_name, role, tech_stack, description
4. 工作经历：company, position, duration, description

# ⚠️ 输出格式要求
严禁使用 Markdown 代码块！
直接输出纯 JSON
```

## 风险评估

### 风险 1：AI 解析可能失败

**缓解措施**：
- 添加 try-catch 错误处理
- 如果 AI 解析失败，降级到空快照（不影响现有功能）
- 记录详细日志，便于调试

### 风险 2：AI 解析耗时较长

**缓解措施**：
- 只在快照为空时才调用 AI 解析
- 解析结果会保存到数据库（下次直接使用）
- 考虑添加缓存（如果频繁创建会话）

### 风险 3：LLM 返回 Markdown 代码块

**缓解措施**：
- 已实现 `cleanMarkdownCodeBlocks()` 方法
- 在解析前自动清理 Markdown 标记
- 降低 temperature（0.1）

1. ❌ 缺少智能解析层
2. ❌ 快照生成依赖手动录入
3. ❌ Agent 缺少结构化上下文

### 解决方案

1. ✅ 创建 `ResumeParsingService` 和 `JDParsingService`
2. ✅ 修改 `MockInterviewService` 集成 AI 解析

1. **自动化**：用户不需要手动录入技能和项目

3. **准确性**：面试提问更有针对性
