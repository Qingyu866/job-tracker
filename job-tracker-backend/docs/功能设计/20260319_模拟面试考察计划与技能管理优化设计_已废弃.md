# 模拟面试考察计划与技能管理优化设计

## 文档概述

**创建日期**: 2026-03-19
**目的**: 优化考察计划生成和技能管理机制
**核心建议**: 用户提出的关键建议

---

## 一、用户建议的核心思想

### 建议 1：预生成考察计划

**用户原话**：
> "既然这个面试的问题是由岗位的 JD 和用户的简历来生成的，那么要考察的知识点是不是就可以让副面试官先根据这个内容和面试的轮次来生成这个要考察的知识点和对应难度"

**核心思想**：
1. **在创建会话时**，副面试官就生成完整的考察计划
2. **考察计划包含**：每轮考察什么技能、难度等级、问题类型
3. **副面试官按照计划执行**，而不是每轮都动态决策
4. **需要修改**：`skill_tags` 表结构，支持考察计划

**优势**：
- ✅ 确保 JD 要求的所有技能都被覆盖
- ✅ 避免重复考察或遗漏
- ✅ 面试流程可控、可预测

---

### 建议 2：动态补全技能标签

**用户原话**：
> "agent已经知道他当前考察的知识点是什么那么他就可以，先去将他这一次要考察的知识点的名称转为list去查询数据库看数据库了里面缺少哪些，将缺少的在返回给ai让其生成这个skill_tags"

**核心思想**：
1. AI 评估时，已经知道当前考察的技能名称
2. **在评估前**，先查询数据库 `skill_tags`，看缺少哪些
3. 将缺少的技能列表返回给 AI
4. 让 AI 生成新的 `skill_tags` 记录（批量插入）

**优势**：
- ✅ 确保 `skill_tags` 表包含所有评估过的技能
- ✅ 避免所有记录都是"通用"
- ✅ 数据库记录有意义、可统计

---

### 建议 3：确保简历摘要字段

**用户原话**：
> "确保其至少要有这个 user_resumes 表里面的这个 summary 字段，因为确实有人没有工作经历和这个拿得出收的项目嘛"

**核心思想**：
1. 有些人没有工作经历和项目经验（应届生、转行者）
2. **必须确保 `user_resumes.summary` 字段不为空**
3. 如果用户没有填写，使用 AI 生成默认摘要

**优势**：
- ✅ 即使没有项目经验，也能生成面试问题
- ✅ 基于技能和目标职位生成问题
- ✅ 不会因为缺少内容而无法开始面试

## 二、当前实现的问题

### 问题 1：缺少考察计划

**当前流程**：
```
创建会话 → skillsPending = "" → 副面试官每轮动态决策 → 容易遗漏或重复
```

**日志显示**：
```
已考察知识点: 无
待考察知识点: 无
```

**问题**：
- ❌ 副面试官不知道应该考察什么
- ❌ 没有明确的考察计划
- ❌ 无法确保 JD 要求的技能都被覆盖

---

### 问题 2：skill_tags 表结构不完善

**当前表结构**（推测）：
```sql
CREATE TABLE skill_tags (
  skill_id BIGINT PRIMARY KEY,
  skill_name VARCHAR(100),
  category VARCHAR(50),
  description TEXT,
  ...
);
```

**问题**：
- ❌ 没有字段标记哪些技能是"JD 要求的"
- ❌ 没有字段标记技能的考察顺序
- ❌ 没有字段关联到具体的面试会话

---

### 问题 3：缺少技能预生成机制

**当前流程**：
```
AI 评估返回 "Redis" → 查询 skill_tags → 如果不存在，返回 0（通用）→ 用户体验差
```

**数据库记录**：
```
skill_id | skill_name | category
---------|-----------|----------
1        | Java      | 编程语言
2        | Redis     | 数据库
...
```

**问题**：
- ❌ AI 评估提到 "Elasticsearch"，但数据库中没有
- ❌ 只能使用默认值 0（通用）
- ❌ 所有新技能都显示为"通用"，用户看不懂

## 三、优化方案设计

### 方案 1：考察计划表（新增）

#### 3.1.1 创建考察计划表

**表名**：`interview_question_plans`

**表结构**：
```sql
CREATE TABLE interview_question_plans (
  plan_id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '考察计划ID',
  session_id VARCHAR(50) NOT NULL COMMENT '面试会话ID',
  round_number INT NOT NULL COMMENT '轮次（从1开始）',
  sequence_number INT NOT NULL COMMENT '该轮的序号（从1开始）',

  -- 技能信息
  skill_name VARCHAR(100) NOT NULL COMMENT '要考察的技能名称',
  skill_id BIGINT COMMENT '关联的技能ID（可能为NULL，动态补全）',

  -- 考察策略
  topic_source VARCHAR(50) COMMENT '选题来源：PROJECT_DEEP_DIVE, SKILL_VERIFICATION, JD_REQUIREMENT, GENERAL',
  question_type VARCHAR(50) COMMENT '问题类型：PROJECT_DEEP_DIVE, SKILL_VERIFICATION, JD_REQUIREMENT, GENERAL, OPEN_ENDED',
  difficulty INT COMMENT '难度等级（1-5）',

  -- 上下文信息
  context_info TEXT COMMENT '上下文信息（例如：简历声称精通Redis）',
  reason TEXT COMMENT '选择该技能的原因',

  -- 状态
  status VARCHAR(20) DEFAULT 'PENDING' COMMENT '状态：PENDING(待考察), COMPLETED(已完成), SKIPPED(跳过)',

  -- 时间戳
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  INDEX idx_session_round (session_id, round_number),
  INDEX idx_status (status)
) COMMENT='模拟面试考察计划表';
```

**字段说明**：
- `skill_id`：可能为 NULL（如果技能不在 skill_tags 表中）
- `status`：标记该计划是否已执行
- `round_number` + `sequence_number`：支持一轮考察多个技能

### 方案 2：动态补全技能标签

#### 3.2.1 新增方法：ensureSkillsExist()

**时机**：`MockInterviewController.sendMessage()` 评估前

**流程**：
```
副面试官决定考察 "Redis"
    ↓
查询 skill_tags 表，看是否存在 "Redis"
    ↓
如果存在 → 使用现有的 skill_id
    ↓
如果不存在 → 调用 AI 生成新的 skill_tags 记录
    ↓
AI 返回技能标签列表（包含分类、描述）
    ↓
批量插入 skill_tags 表
    ↓
返回 skill_id 给评估使用
```

**代码实现**：
```java
/**
 * 确保技能存在，如果不存在则让 AI 生成
 *
 * @param skillNames 技能名称列表
 * @return 技能名称 → ID 的映射
 */
private Map<String, Long> ensureSkillsExist(List<String> skillNames) {
    Map<String, Long> result = new HashMap<>();
    List<String> missingSkills = new ArrayList<>();

    // 1. 查询已存在的技能
    for (String skillName : skillNames) {
        SkillTag skill = skillTagMapper.selectOne(
            new LambdaQueryWrapper<SkillTag>()
                .eq(SkillTag::getSkillName, skillName)
        );

        if (skill != null) {
            result.put(skillName, skill.getSkillId());
        } else {
            missingSkills.add(skillName);
        }
    }

    // 2. 如果有缺失的技能，调用 AI 生成
    if (!missingSkills.isEmpty()) {
        log.info("发现缺失的技能：{}，调用 AI 生成", missingSkills);

        List<SkillTag> generatedSkills = generateSkillTagsWithAI(missingSkills);

        // 3. 批量插入
        for (SkillTag skill : generatedSkills) {
            skillTagMapper.insert(skill);
            result.put(skill.getSkillName(), skill.getSkillId());
        }
    }

    return result;
}

/**
 * 使用 AI 生成技能标签
 *
 * @param skillNames 缺失的技能名称
 * @return 生成的技能标签列表
 */
private List<SkillTag> generateSkillTagsWithAI(List<String> skillNames) {
    String prompt = buildSkillGenerationPrompt(skillNames);

    // 调用 AI
    String aiResponse = chatModel.generate(prompt);

    // 清理 Markdown 标记
    String cleanedResponse = cleanMarkdownCodeBlocks(aiResponse);

    // 解析 JSON
    ObjectMapper mapper = new ObjectMapper();
    try {
        return mapper.readValue(
            cleanedResponse,
            new TypeReference<List<SkillTag>>(){}
        );
    } catch (Exception e) {
        log.error("AI 生成技能标签失败，使用默认值", e);
        // 返回默认技能
        return skillNames.stream()
            .map(name -> SkillTag.builder()
                .skillName(name)
                .category("技术栈")
                .description("AI 自动生成")
                .build())
            .toList();
    }
}

/**
 * 构建技能生成提示词
 */
private String buildSkillGenerationPrompt(List<String> skillNames) {
    return String.format("""
        # 任务
        为以下技能生成技能标签记录（包含分类和描述）。

        # 技能列表
        %s

        # 生成要求

        ## 1. 技能分类
        请根据技能的常见用途，分配一个分类：
        - 编程语言：Java, Python, Go, JavaScript, TypeScript, C++
        - 框架：Spring Boot, Vue, React, Django, Express
        - 数据库：MySQL, PostgreSQL, MongoDB, Redis
        - 消息队列：Kafka, RabbitMQ, RocketMQ
        - 大数据：Hadoop, Spark, Flink
        - 容器化：Docker, Kubernetes
        - 其他：根据实际情况选择

        ## 2. 技能描述
        为每个技能生成一个简短的描述（20-50字），说明：
        - 这是什么技术
        - 主要用途是什么
        - 常见应用场景

        # 输出格式
        返回 JSON 数组：
        [
          {
            "skill_name": "Redis",
            "category": "数据库",
            "description": "Redis 是一个开源的内存数据库，常用作缓存、消息队列和分布式锁。"
          },
          ...
        ]

        # ⚠️ 输出格式要求
        严禁使用 Markdown 代码块！
        直接输出纯 JSON 数组
        """,
        String.join(", ", skillNames)
    );
}
```

### 方案 3：确保简历摘要字段

#### 3.3.1 生成默认摘要

**时机**：`MockInterviewService.createSession()`

**流程**：
```
获取简历
    ↓
检查 summary 字段
    ↓
如果为空 → 调用 AI 生成默认摘要
    ↓
保存到数据库
```

**代码实现**：
```java
private void ensureResumeSummaryNotEmpty(Long resumeId) {
    UserResume resume = resumeMapper.selectById(resumeId);

    if (resume == null) {
        throw new IllegalArgumentException("简历不存在: " + resumeId);
    }

    // 如果摘要不为空，直接返回
    if (resume.getSummary() != null && !resume.getSummary().trim().isEmpty()) {
        return;
    }

    // 摘要为空，生成默认摘要
    log.info("简历摘要为空，生成默认摘要，简历ID: {}", resumeId);

    String defaultSummary = generateDefaultResumeSummary(resume);

    // 更新数据库
    resume.setSummary(defaultSummary);
    resumeMapper.updateById(resume);
}

/**
 * 生成默认简历摘要
 */
private String generateDefaultResumeSummary(UserResume resume) {
    // 查询简历技能
    List<ResumeSkill> skills = resumeSkillMapper.selectList(
        new LambdaQueryWrapper<ResumeSkill>()
            .eq(ResumeSkill::getResumeId, resume.getId())
    );

    // 查询项目经验
    List<ResumeProject> projects = resumeProjectMapper.selectList(
        new LambdaQueryWrapper<ResumeProject>()
            .eq(ResumeProject::getResumeId, resume.getId())
    );

    // 查询工作经历
    List<ResumeWorkExperience> workExps = resumeWorkExperienceMapper.selectList(
        new LambdaQueryWrapper<ResumeWorkExperience>()
            .eq(ResumeWorkExperience::getResumeId, resume.getId())
    );

    // 构建摘要文本
    StringBuilder summary = new StringBuilder();

    if (!workExps.isEmpty()) {
        summary.append("工作经验：\n");
        for (ResumeWorkExperience work : workExps) {
            summary.append(String.format("- %s：%s\n",
                work.getCompanyName(),
                work.getPosition()
            ));
        }
        summary.append("\n");
    }

    if (!projects.isEmpty()) {
        summary.append("项目经验：\n");
        for (ResumeProject project : projects) {
            summary.append(String.format("- %s（%s）\n",
                project.getProjectName(),
                project.getRole()
            ));
        }
        summary.append("\n");
    }

    if (!skills.isEmpty()) {
        summary.append("技能列表：\n");
        for (ResumeSkill skill : skills) {
            summary.append(String.format("- %s（%s）\n",
                skill.getSkillName(),
                skill.getProficiencyLevel()
            ));
        }
    }

    // 如果全部为空，生成极简摘要
    if (summary.length() == 0) {
        summary.append("求职者，目标职位：")
            .append(resume.getTargetLevel() != null ? resume.getTargetLevel() : "未指定")
            .append("\n");
    }

    return summary.toString();
}
```

## 七、总结

### 三个方案的关系

1. **方案 1（考察计划）**：
   - 在创建会话时预生成
   - 副面试官按照计划执行
   - 确保所有技能都被覆盖

2. **方案 2（动态补全技能）**：
   - 评估前确保技能存在
   - 如果不存在，AI 生成并插入
   - 避免所有记录都是"通用"

3. **方案 3（确保简历摘要）**：
   - 创建会话时检查
   - 如果为空，AI 生成默认摘要
   - 即使没有项目经验也能面试

### 预期效果

| 方面 | 优化前 | 优化后 |
|-----|-------|-------|
| **考察计划** | ❌ 无计划，每轮动态决策 | ✅ 创建会话时预生成完整计划 |
| **技能覆盖** | ❌ 可能遗漏 JD 要求的技能 | ✅ 确保所有 JD 技能都被覆盖 |
| **技能记录** | ❌ 都是"通用"（skillId=0） | ✅ 真实的技能名称和 ID |
| **新技能处理** | ❌ 无法处理新技能 | ✅ AI 自动生成并插入数据库 |
| **简历摘要** | ❌ 可能为空 | ✅ 确保不为空，AI 生成默认值 |
