## 基础信息

### Base URL

```
开发环境: http://localhost:8080/api
生产环境: {待配置}
```

### 统一响应格式

所有接口返回统一的 JSON 格式：

```json
{
  "code": 200,
  "message": "成功",
  "data": { ... },
  "timestamp": 1234567890,
  "success": true
}
```

### 认证方式

使用 **Sa-Token** 进行身份认证：

```
Header: Authorization: Bearer {token}
```

## 求职申请接口

### 1. 获取所有求职申请

```
GET /api/applications
```

**响应示例**:
```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    {
      "id": 1,
      "companyId": 10,
      "jobTitle": "前端工程师",
      "status": "APPLIED",
      "applicationDate": "2026-03-01",
      "priority": "HIGH",
      "notes": "已投递简历"
    }
  ]
}
```

---

### 2. 根据ID获取求职申请

```
GET /api/applications/{id}
```

**路径参数**:
- `id` (Long): 申请ID

---

### 3. 根据状态获取求职申请

```
GET /api/applications/status/{status}
```

**路径参数**:
- `status` (String): 申请状态码 (如: APPLIED, INTERVIEW, OFFERED)

---

### 4. 分页查询求职申请

```
GET /api/applications/page?pageNum=1&pageSize=10&status=APPLIED
```

**查询参数**:
- `pageNum` (Integer): 当前页，默认 1
- `pageSize` (Integer): 每页大小，默认 10
- `status` (String, 可选): 申请状态

---

### 5. 获取高优先级申请

```
GET /api/applications/high-priority
```

---

### 6. 更新申请状态

```
PUT /api/applications/{id}/status?status={newStatus}
```

**路径参数**:
- `id` (Long): 申请ID

**查询参数**:
- `status` (String): 新状态码

**响应示例**:
```json
{
  "code": 200,
  "message": "状态更新成功"
}
```

---

### 7. 创建求职申请

```
POST /api/applications
```

**请求体**:
```json
{
  "companyId": 10,
  "jobTitle": "后端工程师",
  "status": "WISHLIST",
  "applicationDate": "2026-03-18",
  "priority": "MEDIUM",
  "notes": "准备投递"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "创建成功",
  "data": 123
}
```

---

### 8. 更新求职申请

```
PUT /api/applications/{id}
```

**路径参数**:
- `id` (Long): 申请ID

**请求体**: 同创建申请

---

### 9. 删除求职申请

```
DELETE /api/applications/{id}
```

**路径参数**:
- `id` (Long): 申请ID

**注意**: 如果申请下有关联的面试记录，将阻止删除

## OCR识别接口

```
POST /api/ocr/recognize
```

**请求体**:
```json

  "imageData": "data:image/png;base64,iVBORw0KG...",
  "tableOnly": false
}
```

**响应示例**:
```json
{
  "code": 200,
  "data": {

    "text": "识别出的文本内容",

  }
}
```

---

### 2. 上传并识别简历


**Content-Type**: `multipart/form-data`

## 🔒 认证相关

### Token 获取

通过登录接口获取：

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}'
```

### Token 使用

在需要认证的接口中添加 Header：

```bash
curl -X GET http://localhost:8080/api/applications \
  -H "Authorization: Bearer {token}"
```
