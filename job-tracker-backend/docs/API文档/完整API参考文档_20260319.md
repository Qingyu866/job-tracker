## 基础信息

### 服务器配置
- **开发环境地址**: `http://localhost:8080`
- **Context Path**: `/api`
- **完整URL**: `http://localhost:8080/api`
- **字符编码**: `UTF-8`
- **认证方式**: JWT Token (Bearer Token)

### 请求头
```http
Content-Type: application/json
Authorization: Bearer {token}
```

---

## 统一响应格式

所有API接口使用统一的响应格式 `Result<T>`：

### 成功响应
```json
{
  "code": 200,
  "message": "成功",
  "data": { /* 实际数据 */ },
  "timestamp": 1234567890,
  "success": true
}
```

### 错误响应
```json
{
  "code": 400,
  "message": "错误描述",
  "data": null,
  "timestamp": 1234567890,
  "success": false
}
```

### 常见HTTP状态码
- `200 OK` - 请求成功
- `400 Bad Request` - 请求参数错误
- `401 Unauthorized` - 未授权（Token无效或过期）
- `403 Forbidden` - 无权限访问
- `404 Not Found` - 资源不存在
- `500 Internal Server Error` - 服务器内部错误

## 2. 求职申请模块

**路径前缀**: `/applications`

### 2.1 获取所有申请
```http
GET /applications
```

**响应**:
```json
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "companyId": 1,
      "jobTitle": "Java开发工程师",
      "status": "APPLIED",
      "applicationDate": "2026-03-01",
      "priority": 8,
      "notes": "备注信息",
      "createdAt": "2026-03-01T10:00:00"
    }
  ]
}
```

---

### 2.2 根据ID获取申请详情
```http
GET /applications/{id}
```

**路径参数**:
- `id` (Long) - 申请ID

**响应**:
```json
{
  "code": 200,
  "data": {
    "id": 1,
    "companyId": 1,
    "jobTitle": "Java开发工程师",
    "jobDescription": "职位描述",
    "status": "APPLIED",
    "applicationDate": "2026-03-01",
    "priority": 8,
    "salaryMin": 20000,
    "salaryMax": 35000,
    "notes": "备注信息"
  }
}
```

---

### 2.3 根据状态获取申请
```http
GET /applications/status/{status}
```

**路径参数**:
- `status` (String) - 申请状态
  - 可选值: `WISHLIST`, `APPLIED`, `SCREENING`, `INTERVIEW`, `FINAL_ROUND`, `OFFERED`, `ACCEPTED`, `REJECTED`, `WITHDRAWN`, `EXPIRED`

**响应**: 同 2.1

---

### 2.4 分页查询申请
```http
GET /applications/page?pageNum=1&pageSize=10&status=APPLIED
```

**查询参数**:
- `pageNum` (Integer, 默认: 1) - 页码
- `pageSize` (Integer, 默认: 10) - 每页数量
- `status` (String, 可选) - 状态筛选

**响应**:
```json
{
  "code": 200,
  "data": {
    "records": [ /* 申请列表 */ ],
    "total": 100,
    "size": 10,
    "current": 1,
    "pages": 10
  }
}
```

---

### 2.5 获取高优先级申请
```http
GET /applications/high-priority
```

**说明**: 返回优先级 >= 7 的申请

**响应**: 同 2.1

---

### 2.6 更新申请状态
```http
PUT /applications/{id}/status?status=INTERVIEW
```

**路径参数**:
- `id` (Long) - 申请ID

**查询参数**:
- `status` (String) - 新状态

**说明**:
- 使用状态转换服务验证合法性
- 自动处理面试记录的状态联动

**响应**:
```json
{
  "code": 200,
  "data": "状态更新成功"
}
```

---

### 2.7 创建申请
```http
POST /applications
```

**请求体**:
```json
{
  "companyId": 1,
  "jobTitle": "Java开发工程师",

  "workLocation": "北京",
```

```json
{
  "code": 200,

```

### 2.8 更新申请

```http
PUT /applications/{id}
```

**路径参数**:
- `id` (Long) - 申请ID

**请求体**: 同 2.7

**响应**:
```json
{
  "code": 200,
  "data": true
```

## 15. 数据查询模块

**路径前缀**: `/data`

**说明**: 此模块提供与专用模块相同的接口，路径前缀为 `/data` 而不是模块专用前缀。

### 接口映射

| 功能 | 专用路径 | 备用路径 |
|------|---------|---------|
| 求职申请 | `/applications/*` | `/data/applications/*` |
| 面试记录 | `/interviews/*` | `/data/interviews/*` |
| 公司信息 | `/companies/*` | `/data/companies/*` |
| 日志 | `/logs/*` | `/data/logs/*` |
| 统计 | `/statistics/*` | `/data/statistics/*` |
| 导出 | `/export/*` | `/data/export/*` |
| 搜索 | `/search/*` | `/data/search/*` |

**示例**:
- 专用: `GET /applications` → 备用: `GET /data/applications`
- 专用: `GET /interviews` → 备用: `GET /data/interviews`
