## 1. 求职申请接口

### 1.1 更新求职申请（完整更新）

**接口描述**：更新指定申请的所有可修改字段

**请求信息**：

| 项目 | 内容 |
|------|------|
| URL | `PUT /api/data/applications/{id}` |
| Content-Type | `application/json` |

**路径参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Long | 是 | 申请ID |

**请求体**：

```json
{
  "companyId": 1,
  "jobTitle": "高级Java工程师",
  "jobDescription": "负责核心系统开发...",
  "jobType": "FULL_TIME",
  "workLocation": "北京",
  "salaryMin": 25000,
  "salaryMax": 40000,
  "salaryCurrency": "CNY",
  "jobUrl": "https://example.com/job/123",
  "status": "INTERVIEW",
  "applicationDate": "2026-03-10",
  "priority": 8,
  "notes": "重点跟进"
}
```

**请求体字段说明**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| companyId | Long | 否 | 公司ID |
| jobTitle | String | 否 | 职位名称 |
| jobDescription | String | 否 | 职位描述 |
| jobType | String | 否 | 工作类型（FULL_TIME/PART_TIME/CONTRACT/INTERNSHIP） |
| workLocation | String | 否 | 工作地点 |
| salaryMin | BigDecimal | 否 | 最低薪资 |
| salaryMax | BigDecimal | 否 | 最高薪资 |
| salaryCurrency | String | 否 | 薪资货币（默认CNY） |
| jobUrl | String | 否 | 职位链接 |
| status | String | 否 | 状态（WISHLIST/APPLIED/INTERVIEW/OFFER/REJECTED/WITHDRAWN） |
| applicationDate | LocalDate | 否 | 申请日期 |
| priority | Integer | 否 | 优先级（1-10） |
| notes | String | 否 | 备注 |

**响应示例（成功）**：

```json
{
  "code": 200,
  "message": "更新成功",
  "data": true
}
```

**响应示例（失败）**：

```json
{
  "code": 400,
  "message": "申请不存在",
  "data": null
}
```

**接口描述**：删除指定申请。如果该申请下有关联的面试记录，则阻止删除并提示用户。

**请求信息**：

| 项目 | 内容 |
|------|------|
| URL | `DELETE /api/data/applications/{id}` |
| Content-Type | 无 |

**路径参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Long | 是 | 申请ID |

**响应示例（成功）**：

```json
{
  "code": 200,
  "message": "删除成功",
  "data": true
}
```

**响应示例（保护性阻止）**：

```json
{
  "code": 400,
  "message": "该申请下有 2 条面试记录，请先删除面试记录",
  "data": null
}
```

```json
{
  "code": 404,
  "message": "申请不存在",
  "data": null

```

## 4. 错误码说明

| 错误码 | 说明 | 处理建议 |
|--------|------|---------|
| 200 | 操作成功 | - |
| 400 | 请求参数错误或保护性阻止 | 检查参数格式或先删除关联数据 |
| 404 | 资源不存在 | 确认ID是否正确 |
| 500 | 服务器内部错误 | 联系管理员或查看日志 |

## 5. 调用示例

### cURL 示例

#### 更新申请

```bash
curl -X PUT http://localhost:8080/api/data/applications/1 \
  -H "Content-Type: application/json" \
  -d '{
    "jobTitle": "高级Java工程师",
    "status": "INTERVIEW",
    "priority": 9
  }'
```

#### 删除申请

```bash
curl -X DELETE http://localhost:8080/api/data/applications/1
```

#### 更新面试记录

```bash
curl -X PUT http://localhost:8080/api/data/interviews/1 \
  -H "Content-Type: application/json" \
  -d '{
    "interviewDate": "2026-03-20T15:00:00",
    "status": "SCHEDULED"
  }'
```

#### 删除面试记录

```bash
curl -X DELETE http://localhost:8080/api/data/interviews/1
```

```bash

  -H "Content-Type: application/json" \
```

```javascript

// 更新申请

  return response.data;

// 删除申请
async function deleteApplication(id) {
  const response = await axios.delete(`${API_BASE}/applications/${id}`);
```

## 6. 注意事项

### 6.1 保护性删除机制

为防止误删重要数据，系统采用**保护性删除**策略：

1. **删除申请时**：如果该申请下有面试记录，系统会阻止删除并返回错误提示
2. **删除公司时**：如果该公司下有申请记录，系统会阻止删除并返回错误提示
3. **删除面试记录时**：直接删除，无前置检查

### 6.2 逻辑删除

所有删除操作均为**逻辑删除**（将 `deleted` 字段设为 1），数据仍保留在数据库中，可恢复。

### 6.3 时间戳保留

更新操作会自动保留原记录的 `createdAt` 创建时间，仅更新 `updatedAt` 更新时间。
