# 定时提醒系统设计方案

## 1. 需求背景

### 1.1 现状分析
- `InterviewRecord` 实体已有 `followUpRequired` 字段，标记需要跟进的面试
- 但后端没有定时任务来检查和发送提醒
- 用户需要主动查看才能知道即将到来的面试或需要跟进的事项

### 1.2 目标
实现基于 Spring `@Scheduled` 的定时提醒系统，支持：
- 面试提醒（即将到来的面试）
- 跟进提醒（标记为需要跟进的面试）
- 状态变更提醒（如收到 Offer、被拒绝等）

## 3. 数据库设计

### 3.1 提醒记录表 (reminder_records)

```sql
CREATE TABLE reminder_records (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '用户ID（预留，当前为1）',
    reminder_type VARCHAR(50) NOT NULL COMMENT '提醒类型：INTERVIEW/FOLLOW_UP/STATUS_CHANGE',
    reference_type VARCHAR(50) NOT NULL COMMENT '关联类型：APPLICATION/INTERVIEW',
    reference_id BIGINT NOT NULL COMMENT '关联ID',
    title VARCHAR(200) NOT NULL COMMENT '提醒标题',
    content TEXT COMMENT '提醒内容',
    reminder_time DATETIME NOT NULL COMMENT '提醒时间',
    status VARCHAR(20) NOT NULL DEFAULT 'PENDING' COMMENT '状态：PENDING/SENT/READ/DISABLED',
    sent_at DATETIME COMMENT '发送时间',
    read_at DATETIME COMMENT '阅读时间',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_user_status (user_id, status),
    INDEX idx_reminder_time (reminder_time),
    INDEX idx_reference (reference_type, reference_id)
) COMMENT '提醒记录表';
```

### 3.2 用户提醒配置表 (user_reminder_configs)

```sql
CREATE TABLE user_reminder_configs (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '用户ID',

```

### 4.2 核心代码示例

#### 4.2.1 面试提醒调度器

```java
package com.jobtracker.scheduler;

import com.jobtracker.service.ReminderService;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.scheduling.annotation.Scheduled;
import org.springframework.stereotype.Component;

/**
 * 面试提醒调度器
 */
@Slf4j
@Component
@RequiredArgsConstructor
public class InterviewReminderScheduler {

    private final ReminderService reminderService;

    /**
     * 每天早上 8:00 检查当天的面试
     */
    @Scheduled(cron = "0 0 8 * * ?")
    public void checkTodayInterviews() {
        log.info("[定时任务] 开始检查今天的面试...");
        reminderService.checkAndSendInterviewReminders(0); // 当天
    }

    /**
     * 每天早上 8:00 检查明天的面试
     */
    @Scheduled(cron = "0 0 8 * * ?")
    public void checkTomorrowInterviews() {
        log.info("[定时任务] 开始检查明天的面试...");
        reminderService.checkAndSendInterviewReminders(1); // 明天
    }

    /**
     * 每小时检查接下来 2 小时内的面试
     */
    @Scheduled(cron = "0 0 * * * ?")
    public void checkUpcomingInterviews() {
        log.info("[定时任务] 检查接下来 2 小时内的面试...");
        reminderService.checkAndSendUrgentReminders(2);
    }
}
```

#### 4.2.4 提醒服务实现

```java
package com.jobtracker.service.impl;

import com.jobtracker.dto.ReminderDTO;
import com.jobtracker.entity.InterviewRecord;
import com.jobtracker.entity.ReminderRecord;
import com.jobtracker.mapper.InterviewRecordMapper;
import com.jobtracker.mapper.ReminderRecordMapper;
import com.jobtracker.service.ReminderService;
import com.jobtracker.websocket.WebSocketSessionManager;
import com.jobtracker.dto.WebSocketMessage;
import com.fasterxml.jackson.databind.ObjectMapper;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;
import org.springframework.web.socket.TextMessage;

import java.time.LocalDate;
import java.time.LocalDateTime;
import java.util.List;

@Slf4j
@Service
@RequiredArgsConstructor
public class ReminderServiceImpl implements ReminderService {

    private final InterviewRecordMapper interviewMapper;
    private final ReminderRecordMapper reminderMapper;
    private final WebSocketSessionManager sessionManager;
    private final ObjectMapper objectMapper;

    @Override
    public void checkAndSendInterviewReminders(int daysAhead) {
        LocalDate targetDate = LocalDate.now().plusDays(daysAhead);

        // 查询当天的面试
        List<InterviewRecord> interviews = interviewMapper.findByDate(targetDate);

        for (InterviewRecord interview : interviews) {
            if (interview.getStatus() == InterviewStatus.CANCELLED ||
                interview.getStatus() == InterviewStatus.NO_SHOW) {
                continue;
            }

            // 检查是否已发送过提醒
            if (hasReminder("INTERVIEW", interview.getId(), targetDate)) {
                continue;
            }

            // 创建并保存提醒
            ReminderRecord reminder = createInterviewReminder(interview, daysAhead);
            reminderMapper.insert(reminder);

            // 发送 WebSocket 通知
            sendWebSocketNotification(reminder);
        }
    }

    @Override
    public void checkAndSendUrgentReminders(int hoursAhead) {
        LocalDateTime now = LocalDateTime.now();
        LocalDateTime deadline = now.plusHours(hoursAhead);

        List<InterviewRecord> urgentInterviews =
            interviewMapper.findByDateTimeRange(now, deadline);

        for (InterviewRecord interview : urgentInterviews) {
            // 紧急提醒逻辑...
        }
    }

    @Override
    public void checkAndSendFollowUpReminders() {
        // 查询需要跟进且未完成的面试
        List<InterviewRecord> followUps =
            interviewMapper.findByFollowUpRequired(true);

        for (InterviewRecord interview : followUps) {
            // 检查是否已发送过今日提醒
            if (hasReminderToday("FOLLOW_UP", interview.getId())) {
                continue;
            }

            // 创建跟进提醒
            ReminderRecord reminder = createFollowUpReminder(interview);
            reminderMapper.insert(reminder);

            // 发送通知
            sendWebSocketNotification(reminder);
        }
    }

    private void sendWebSocketNotification(ReminderRecord reminder) {
        try {
            WebSocketMessage message = WebSocketMessage.builder()
                .type("REMINDER")
                .content(reminder.getTitle())
                .data(reminder)
                .build();

            String json = objectMapper.writeValueAsString(message);
            sessionManager.broadcast(new TextMessage(json));
        } catch (Exception e) {
            log.error("发送提醒通知失败", e);
        }
    }

    // ... 其他辅助方法
}
```

## 9. 注意事项

1. **时区处理**：确保所有时间使用同一时区，建议使用 `LocalDateTime` 并在配置中指定时区
2. **幂等性**：定时任务需要确保不会重复发送相同提醒
3. **性能**：大量提醒时考虑分批处理，避免阻塞
4. **日志**：关键操作需要记录详细日志，便于排查问题
5. **开关控制**：提供配置开关，方便在开发环境禁用定时任务
